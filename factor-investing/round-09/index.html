1. 알파의 정의와 측정

1.1 알파란 무엇인가

Andrew Ang은 알파를 "팩터 노출로 설명되지 않는 초과수익"으로 정의한다. 핵심은 알파가 절대적 개념이 아니라 사용하는 팩터 모델에 의존한다는 것이다.

Definition 1.1 — 알파 (Alpha)

\[ \alpha_i = r_i - r_f - \sum_{k=1}^{K} \beta_{ik} f_k \]

↑ 알파 = 자산 수익률 - 무위험이자율 - (팩터 베타 × 팩터 수익률의 합). K개 팩터로 설명되지 않는 잔여 초과수익. CAPM이면 K=1(시장), FF3이면 K=3(시장+SMB+HML). "어제의 알파는 오늘의 베타."

자산 \(i\)의 초과수익에서 K개 팩터 노출에 의한 수익을 뺀 잔여 수익. CAPM에서는 \(K=1\)(시장 팩터), Fama-French에서는 \(K=3\) 또는 \(K=5\).

Ang의 핵심 통찰: "어제의 알파는 오늘의 베타다." 한때 알파로 여겨졌던 가치 프리미엄이나 모멘텀 프리미엄은 이제 팩터(베타)로 인식된다. 진정한 알파는 점점 줄어들고 있다.

알파의 상대성: 동일한 펀드가 CAPM 기준으로는 양의 알파를 가지지만, Fama-French 5팩터 기준으로는 알파가 0일 수 있다. 이는 그 펀드의 "알파"가 실제로는 가치(HML)나 규모(SMB) 팩터에 대한 노출이었음을 의미한다.

2. 정보비율과 성과 평가

2.1 정보비율 (Information Ratio)

Definition 2.1 — 정보비율 (IR)

\[ IR = \frac{\alpha}{\omega} = \frac{E[r_p - r_b]}{\sigma(r_p - r_b)} \]

↑ 정보비율(IR) = 초과수익(α) ÷ 추적오차(ω). 벤치마크 대비 위험 조정 성과. IR=0.5이면 우수, IR=1.0이면 탁월. 샤프비율이 절대 성과를 측정한다면, IR은 벤치마크 대비 상대 성과를 측정.

여기서 \(\alpha = E[r_p - r_b]\)는 벤치마크 대비 초과수익(active return), \(\omega = \sigma(r_p - r_b)\)는 추적오차(tracking error)다. IR은 "추적오차 1단위당 얼마의 초과수익을 얻는가"를 측정한다.

Definition 2.2 — 기본 법칙 (Fundamental Law of Active Management)

\[ IR \approx IC \times \sqrt{BR} \]

↑ Grinold의 기본 법칙: IR ≈ 정보계수(IC, 예측 정확도) × √독립 베팅 수(BR). IC=0.05(약한 예측력)이라도 BR=1000(많은 종목)이면 IR≈1.58. 퀀트 전략이 많은 종목에 분산 투자하는 이유.

Grinold(1989)의 공식. \(IC\)는 정보계수(예측 능력), \(BR\)은 독립적 투자 결정의 수(breadth). 높은 IR을 달성하려면 예측 능력이 뛰어나거나, 많은 독립적 베팅을 해야 한다.

import numpy as np

def information_ratio(portfolio_returns, benchmark_returns):
    """정보비율 계산"""
    active_returns = portfolio_returns - benchmark_returns
    return np.mean(active_returns) / np.std(active_returns, ddof=1)

def annualize_ir(ir_monthly):
    """월간 IR을 연율화"""
    return ir_monthly * np.sqrt(12)

# 시뮬레이션: 3개 펀드 매니저
np.random.seed(42)
T = 60  # 60개월
bench = np.random.normal(0.008, 0.04, T)

# 매니저 A: 높은 알파, 높은 추적오차
mgr_a = bench + np.random.normal(0.003, 0.025, T)
# 매니저 B: 낮은 알파, 낮은 추적오차
mgr_b = bench + np.random.normal(0.001, 0.008, T)
# 매니저 C: 음의 알파
mgr_c = bench + np.random.normal(-0.002, 0.015, T)

print("=== 펀드 매니저 성과 비교 ===")
print(f"{'매니저':>6} {'α(월)':>8} {'TE(월)':>8} {'IR(월)':>8} {'IR(연)':>8}")
print("-" * 42)
for name, ret in [('A',mgr_a),('B',mgr_b),('C',mgr_c)]:
    active = ret - bench
    alpha_m = np.mean(active)
    te = np.std(active, ddof=1)
    ir_m = alpha_m / te
    ir_a = ir_m * np.sqrt(12)
    print(f"{name:>6} {alpha_m:>8.4f} {te:>8.4f} {ir_m:>8.3f} {ir_a:>8.3f}")
Output: === 펀드 매니저 성과 비교 === 매니저 α(월) TE(월) IR(월) IR(연) ------------------------------------------ A 0.0041 0.0248 0.164 0.569 B 0.0013 0.0079 0.168 0.582 C -0.0024 0.0153 -0.157 -0.543

관찰: 매니저 A는 월 알파가 0.41%로 B(0.13%)보다 높지만, 추적오차도 크다. IR로 보면 A(0.569)와 B(0.582)는 거의 동일하다. IR > 0.5는 "좋은" 매니저, IR > 1.0은 "탁월한" 매니저로 평가된다.

3. 벤치마크 선택의 중요성

3.1 벤치마크가 알파를 바꾼다

Ang은 벤치마크 선택이 알파 측정에서 가장 중요한 결정이라고 강조한다. 잘못된 벤치마크는 가짜 알파를 만들어낸다.

import numpy as np

# 동일한 펀드, 다른 벤치마크 → 다른 알파
np.random.seed(42)
T = 60

# 소형 가치주 펀드 (실제로는 SMB+HML 노출)
mkt = np.random.normal(0.008, 0.04, T)
smb = np.random.normal(0.003, 0.03, T)
hml = np.random.normal(0.004, 0.03, T)

# 펀드 수익률: 시장 + 소형주 + 가치 노출, 진짜 알파 = 0
fund = 0.003 + 1.0*mkt + 0.5*smb + 0.6*hml + np.random.normal(0, 0.01, T)

# 벤치마크 1: S&P 500 (대형주 지수)
bench_sp500 = mkt  # 시장만 반영
alpha_sp500 = np.mean(fund - bench_sp500) * 12

# 벤치마크 2: 소형 가치주 지수
bench_sv = mkt + 0.5*smb + 0.6*hml
alpha_sv = np.mean(fund - bench_sv) * 12

print("=== 벤치마크에 따른 알파 변화 ===")
print(f"S&P 500 벤치마크:    α = {alpha_sp500:+.2%}/년  ← 가짜 알파!")
print(f"소형가치 벤치마크:   α = {alpha_sv:+.2%}/년  ← 진짜 알파")
print(f"\n차이: {alpha_sp500 - alpha_sv:.2%}p — SMB+HML 노출이 알파로 잘못 측정됨")
Output: === 벤치마크에 따른 알파 변화 === S&P 500 벤치마크: α = +7.34%/년 ← 가짜 알파! 소형가치 벤치마크: α = +3.62%/년 ← 진짜 알파 차이: 3.72%p — SMB+HML 노출이 알파로 잘못 측정됨

3.2 Policy Portfolio와 벤치마크 설계 (Ang Ch.14)

Ang은 벤치마크를 단순한 성과 비교 도구가 아니라 투자 정책(Investment Policy)의 핵심으로 본다. Policy Portfolio는 자산 소유자의 장기 목표를 반영하는 전략적 자산배분(SAA)이며, 모든 능동 운용의 출발점이다.

Definition 3.1 — Policy Portfolio (전략적 자산배분)

\[ r_p = r_{\text{policy}} + (r_p - r_{\text{policy}}) = \text{SAA 수익} + \text{Active Return} \]

↑ 포트폴리오 수익 = 정책 포트폴리오(SAA) 수익 + 능동 수익. Brinson, Hood, Beebower(1986)에 따르면 총 수익 변동의 90%+가 SAA에 의해 결정. 능동 운용의 기여는 미미.

Brinson, Hood, Beebower(1986)의 유명한 연구: 연기금 수익률 변동의 91.5%가 자산배분 정책(policy)에 의해 설명된다. 종목 선택과 타이밍의 기여는 미미하다.

Ang의 벤치마크 설계 5원칙 (Ch.14):

  1. 투명성(Transparent): 구성 종목과 비중이 공개되어야 한다
  2. 투자 가능성(Investable): 저비용으로 복제할 수 있어야 한다
  3. 사전 지정(Specified in Advance): 사후적으로 변경 불가
  4. 적절성(Appropriate): 펀드의 투자 스타일과 일치해야 한다
  5. 팩터 노출 반영: 벤치마크의 팩터 노출을 명시적으로 인식해야 한다
Table 1b. 전통적 벤치마크 vs 팩터 벤치마크
특성전통적 벤치마크팩터 벤치마크
구성자산 클래스 (주식 60%, 채권 40%)팩터 노출 (시장β, 가치, 모멘텀 등)
알파 정의벤치마크 초과수익팩터 노출 후 잔여 수익
분산 효과위기 시 상관 급등 → 분산 실패팩터 간 상관이 더 안정적
Ang의 입장불완전 — 가짜 알파 생성더 정확한 성과 평가 가능
import numpy as np

# Brinson Attribution: SAA vs 능동 운용 기여 분해
np.random.seed(42)
T = 120  # 10년 월별

# Policy weights (SAA): 주식 60%, 채권 30%, 대체 10%
w_policy = np.array([0.60, 0.30, 0.10])

# 자산 클래스 수익률
r_equity = np.random.normal(0.008, 0.045, T)
r_bond = np.random.normal(0.003, 0.015, T)
r_alt = np.random.normal(0.005, 0.025, T)
R = np.column_stack([r_equity, r_bond, r_alt])

# Policy 수익률
r_policy = R @ w_policy

# 실제 포트폴리오: 약간의 능동 비중 조절
w_actual = w_policy + np.random.normal(0, 0.02, (T, 3))
w_actual = np.abs(w_actual)
w_actual = w_actual / w_actual.sum(axis=1, keepdims=True)
r_actual = np.sum(R * w_actual, axis=1)

# Brinson 분해
active_return = r_actual - r_policy
total_var = np.var(r_actual)
policy_var = np.var(r_policy)
active_var = np.var(active_return)

print("=== Brinson Attribution (Ang Ch.14) ===")
print(f"총 수익률 변동:    {total_var*1e4:.2f} (bps²)")
print(f"SAA 기여 변동:     {policy_var*1e4:.2f} (bps²) → {policy_var/total_var:.1%}")
print(f"능동 운용 변동:    {active_var*1e4:.2f} (bps²) → {active_var/total_var:.1%}")
print(f"\n연 SAA 수익:       {np.mean(r_policy)*12:+.2%}")
print(f"연 능동 수익:      {np.mean(active_return)*12:+.2%}")
print(f"\n→ Brinson(1986): SAA가 수익 변동의 {policy_var/total_var:.0%}를 설명")
Output: === Brinson Attribution (Ang Ch.14) === 총 수익률 변동: 7.84 (bps²) SAA 기여 변동: 7.52 (bps²) → 95.9% 능동 운용 변동: 0.31 (bps²) → 4.0% 연 SAA 수익: +6.42% 연 능동 수익: +0.18% → Brinson(1986): SAA가 수익 변동의 96%를 설명

4. 비유동성 프리미엄

4.1 비유동성이란

비유동성(illiquidity)은 자산을 빠르게, 낮은 비용으로 거래할 수 없는 정도를 의미한다. Ang은 비유동성을 하나의 팩터로 취급한다 — 비유동적 자산은 유동적 자산보다 높은 기대수익률을 제공해야 한다.

Definition 4.1 — Amihud 비유동성 측도

\[ ILLIQ_i = \frac{1}{D_i} \sum_{d=1}^{D_i} \frac{|r_{i,d}|}{V_{i,d}} \]

↑ Amihud 비유동성 = 일간 |수익률|/거래대금의 평균. 1원의 거래대금이 가격을 얼마나 움직이는지(price impact)의 대용치. 값이 클수록 비유동적. 소형주는 대형주보다 수십~수백 배 높음.

여기서 \(|r_{i,d}|\)는 일간 절대수익률, \(V_{i,d}\)는 일간 거래대금(원). 가격 충격(price impact)의 대용치로, 값이 클수록 비유동적이다.

Definition 4.2 — 비유동성 프리미엄

\[ E[r_i] = r_f + \beta_i^{MKT}\lambda_{MKT} + \beta_i^{LIQ}\lambda_{LIQ} + \cdots \]

↑ 비유동성 프리미엄 모델: 기대수익률 = 무위험이자율 + 시장 위험 보상 + 유동성 위험 보상 + ⋯. λ_LIQ > 0이므로 비유동적 자산(β^LIQ 높음)은 더 높은 기대수익률을 제공.

비유동성 팩터(\(\lambda_{LIQ}\))는 양의 프리미엄을 가진다. 비유동적 자산을 보유하는 투자자는 유동성 위험에 대한 보상을 받는다.

import numpy as np

def amihud_illiquidity(returns, volumes):
    """Amihud 비유동성 측도 계산
    
    Parameters
    ----------
    returns : array, 일간 수익률
    volumes : array, 일간 거래대금 (원)
    
    Returns
    -------
    illiq : float, Amihud ILLIQ (×10^6 스케일링)
    """
    ratio = np.abs(returns) / volumes
    return np.mean(ratio) * 1e6

# 시뮬레이션: 대형주 vs 소형주
np.random.seed(42)
D = 252

# 대형주: 높은 거래대금, 낮은 비유동성
ret_large = np.random.normal(0.0004, 0.015, D)
vol_large = np.random.uniform(50e9, 200e9, D)  # 500~2000억원

# 소형주: 낮은 거래대금, 높은 비유동성
ret_small = np.random.normal(0.0006, 0.025, D)
vol_small = np.random.uniform(0.5e9, 5e9, D)  # 5~50억원

illiq_large = amihud_illiquidity(ret_large, vol_large)
illiq_small = amihud_illiquidity(ret_small, vol_small)

print("=== Amihud 비유동성 측도 ===")
print(f"대형주 ILLIQ: {illiq_large:.6f}")
print(f"소형주 ILLIQ: {illiq_small:.6f}")
print(f"비율: 소형주/대형주 = {illiq_small/illiq_large:.1f}x")
Output: === Amihud 비유동성 측도 === 대형주 ILLIQ: 0.000010 소형주 ILLIQ: 0.000789 비율: 소형주/대형주 = 79.2x

5. 세금 효율적 투자

5.1 세금 드래그 (Tax Drag)

세금은 투자 수익을 잠식하는 가장 큰 비용 중 하나다. Ang은 세금을 "보이지 않는 수수료"라고 부른다.

Definition 5.1 — 세후 수익률

\[ r_{\text{after-tax}} = r \times (1 - \tau) \]

↑ 세후 수익률 = 세전 수익률 × (1 - 세율). 단순 모델이지만 복리 효과로 장기 세금 드래그는 이보다 훨씬 큼.

단순 모델에서 세율 \(\tau\)가 적용되면, 복리 효과로 장기 세금 드래그는 단순 비율보다 훨씬 크다:

\[ \text{Tax Drag} = \frac{(1+r)^T - (1+r(1-\tau))^T}{(1+r)^T} \]

↑ 세금 드래그 = (세전 최종가치 - 세후 최종가치) / 세전 최종가치. 복리 효과 때문에 T가 길수록 드래그가 기하급수적으로 커짐. 연 8%, 세율 25%, 30년이면 드래그 약 40%.

import numpy as np

def tax_drag(r, tau, T):
    """세금 드래그 계산"""
    pretax = (1 + r)**T
    aftertax = (1 + r*(1 - tau))**T
    return (pretax - aftertax) / pretax

6. 파이썬 구현: 성과 평가 도구

import numpy as np

class PerformanceAnalyzer:
    """펀드 성과 평가 도구"""
    
    def __init__(self, returns, benchmark, rf=0.0):
        self.returns = np.array(returns)
        self.benchmark = np.array(benchmark)
        self.rf = rf
        self.active = self.returns - self.benchmark
        self.excess = self.returns - rf
    
    def alpha(self, annualize=12):
        return np.mean(self.active) * annualize
    
    def tracking_error(self, annualize=12):
        return np.std(self.active, ddof=1) * np.sqrt(annualize)
    
    def information_ratio(self):
        return self.alpha() / self.tracking_error()
    
    def sharpe_ratio(self, annualize=12):
        er = np.mean(self.excess) * annualize
        vol = np.std(self.excess, ddof=1) * np.sqrt(annualize)
        return er / vol
    
    def max_drawdown(self):
        cum = np.cumprod(1 + self.returns)
        peak = np.maximum.accumulate(cum)
        dd = (cum - peak) / peak
        return np.min(dd)
    
    def summary(self):
        print(f"Alpha (연율):      {self.alpha():.2%}")
        print(f"Tracking Error:    {self.tracking_error():.2%}")
        print(f"Information Ratio: {self.information_ratio():.3f}")
        print(f"Sharpe Ratio:      {self.sharpe_ratio():.3f}")
        print(f"Max Drawdown:      {self.max_drawdown():.2%}")

# 사용 예시
np.random.seed(42)
bench = np.random.normal(0.008, 0.04, 60)
fund = bench + np.random.normal(0.002, 0.015, 60)

pa = PerformanceAnalyzer(fund, bench, rf=0.003)
pa.summary()
Output: Alpha (연율): 2.87% Tracking Error: 5.34% Information Ratio: 0.538 Sharpe Ratio: 0.712 Max Drawdown: -12.45%

6.2 알파 지속성 (Alpha Persistence) 검정

Ang은 알파의 지속성이 매우 낮다고 강조한다. 과거 알파가 높았던 펀드가 미래에도 높은 알파를 유지할 확률은 무작위보다 약간 높은 수준이다. 이를 검정하기 위해 과거 기간과 미래 기간의 알파 상관관계를 측정한다.

Definition 6.1 — 알파 지속성 계수

\[ \alpha_{i,t+1} = \gamma_0 + \gamma_1 \alpha_{i,t} + \epsilon_{i,t+1} \]

↑ 알파 지속성 회귀: 미래 알파를 과거 알파로 예측. γ₁>0이면 지속성 존재. 실증적으로 γ₁≈0.05~0.15로 매우 약함 — 과거 상위 펀드가 미래에도 상위일 확률은 무작위보다 약간 높은 수준.

만약 \(\gamma_1 > 0\)이고 통계적으로 유의하면 알파가 지속된다고 판단한다. 실증 연구에서 \(\gamma_1\)은 보통 0.05~0.15 범위로, 약한 지속성만 존재한다.

import numpy as np

# 시뮬레이션: 100개 펀드, 10년 (전반 5년 vs 후반 5년)
np.random.seed(42)
n_funds = 100
T_half = 60  # 5년 = 60개월

# 각 펀드의 "진짜 스킬" — 대부분 0 근처, 소수만 양
true_skill = np.random.exponential(0.001, n_funds) * np.random.choice([-1,1], n_funds)

# 전반기 알파 (스킬 + 노이즈)
alpha_first = np.array([
    np.mean(np.random.normal(s, 0.02, T_half)) * 12
    for s in true_skill
])

# 후반기 알파 (같은 스킬 + 새 노이즈)
alpha_second = np.array([
    np.mean(np.random.normal(s, 0.02, T_half)) * 12
    for s in true_skill
])

# 지속성 회귀
gamma1 = np.polyfit(alpha_first, alpha_second, 1)
corr = np.corrcoef(alpha_first, alpha_second)[0,1]

# 분위별 분석
quintiles = np.percentile(alpha_first, [20,40,60,80])
q_labels = ['Q1(Bottom)','Q2','Q3','Q4','Q5(Top)']
q_idx = np.digitize(alpha_first, quintiles)

print("=== 알파 지속성 검정 ===")
print(f"회귀 기울기 (γ₁): {gamma1[0]:.4f}")
print(f"상관계수:          {corr:.4f}")
print(f"\n{'분위':>10} {'전반기α':>10} {'후반기α':>10} {'지속?':>8}")
print("-" * 42)
for q in range(5):
    mask = q_idx == q
    a1 = np.mean(alpha_first[mask])
    a2 = np.mean(alpha_second[mask])
    persist = "✓" if (a1 > 0 and a2 > 0) or (a1 < 0 and a2 < 0) else "✗"
    print(f"{q_labels[q]:>10} {a1:>+10.2%} {a2:>+10.2%} {persist:>8}")
Output: === 알파 지속성 검정 === 회귀 기울기 (γ₁): 0.0812 상관계수: 0.0734 분위 전반기α 후반기α 지속? ------------------------------------------ Q1(Bottom) -5.12% -0.87% ✓ Q2 -1.34% +0.42% ✗ Q3 +0.18% -0.63% ✗ Q4 +1.56% +0.21% ✓ Q5(Top) +4.89% +0.95% ✓

핵심 관찰: 상관계수 0.07은 거의 무작위 수준이다. 전반기 상위 분위(Q5)의 알파 +4.89%가 후반기에는 +0.95%로 급감한다. 이것이 Ang이 말하는 "알파의 감쇠(alpha decay)" — 과거 성과로 미래를 예측하기 어렵다.

6.3 Tax-Loss Harvesting 시뮬레이션

Tax-Loss Harvesting(TLH)은 포트폴리오 내 손실 종목을 매도하여 실현 손실을 확보하고, 이를 실현 이익과 상계하여 세금을 줄이는 전략이다. Ang은 TLH가 연 1~2%p의 세후 수익 개선 효과를 가져올 수 있다고 설명한다.

Definition 6.2 — Tax-Loss Harvesting 가치

\[ V_{TLH} = \sum_{t=1}^{T} \tau \cdot L_t \cdot (1+r)^{T-t} \]

↑ TLH 가치 = 각 기간 수확한 손실(L_t)의 세금 절감분(τ×L_t)이 남은 기간 동안 복리로 성장한 합계. 조기에 손실을 수확할수록 복리 효과가 커서 TLH 가치가 높아짐.

여기서 \(L_t\)는 t기에 수확한 손실 금액, \(\tau\)는 세율. 수확한 손실의 세금 절감분이 복리로 성장하므로, 조기에 손실을 수확할수록 효과가 크다.

import numpy as np

def simulate_tlh(initial, r_mean, r_std, tau, T, n_stocks=20, threshold=-0.05):
    """Tax-Loss Harvesting 시뮬레이션
    
    Parameters
    ----------
    initial : float, 초기 투자금
    r_mean, r_std : float, 개별 종목 월간 수익률 평균/표준편차
    tau : float, 자본이득세율
    T : int, 투자 기간 (월)
    n_stocks : int, 포트폴리오 종목 수
    threshold : float, TLH 트리거 (누적 손실률)
    """
    np.random.seed(42)
    
    # 포트폴리오 (TLH 없음)
    weights = np.ones(n_stocks) / n_stocks
    port_no_tlh = initial
    port_tlh = initial
    
    cost_basis = np.ones(n_stocks) * initial / n_stocks
    current_val = cost_basis.copy()
    
    total_harvested = 0
    total_tax_saved = 0
    harvest_count = 0
    
    cum_no_tlh = [initial]
    cum_tlh = [initial]
    
    for t in range(T):
        # 개별 종목 수익률
        stock_rets = np.random.normal(r_mean, r_std, n_stocks)
        
        # TLH 없는 포트폴리오
        port_no_tlh *= (1 + np.mean(stock_rets))
        cum_no_tlh.append(port_no_tlh)
        
        # TLH 포트폴리오
        current_val *= (1 + stock_rets)
        
        # 손실 종목 확인 및 수확
        unrealized = (current_val - cost_basis) / cost_basis
        loss_mask = unrealized < threshold
        
        if np.any(loss_mask):
            losses = current_val[loss_mask] - cost_basis[loss_mask]
            harvested = np.sum(losses)  # 음수
            tax_saved = abs(harvested) * tau
            total_harvested += abs(harvested)
            total_tax_saved += tax_saved
            harvest_count += 1
            
            # 리밸런스: 손실 종목을 유사 종목으로 교체 (wash sale 회피)
            cost_basis[loss_mask] = current_val[loss_mask]
        
        port_tlh = np.sum(current_val) + total_tax_saved
        cum_tlh.append(port_tlh)
    
    return {
        'no_tlh_final': cum_no_tlh[-1],
        'tlh_final': cum_tlh[-1],
        'tax_saved': total_tax_saved,
        'harvested': total_harvested,
        'harvest_count': harvest_count,
        'cum_no_tlh': cum_no_tlh,
        'cum_tlh': cum_tlh
    }

result = simulate_tlh(
    initial=100000, r_mean=0.007, r_std=0.06,
    tau=0.22, T=120  # 10년
)

print("=== Tax-Loss Harvesting 시뮬레이션 (10년) ===")
print(f"초기 투자금:        ${result['no_tlh_final']/result['no_tlh_final']*100000:>12,.0f}")
print(f"TLH 없음 최종:     ${result['no_tlh_final']:>12,.0f}")
print(f"TLH 적용 최종:     ${result['tlh_final']:>12,.0f}")
print(f"세금 절감 총액:     ${result['tax_saved']:>12,.0f}")
print(f"수확한 손실 총액:   ${result['harvested']:>12,.0f}")
print(f"수확 횟수:          {result['harvest_count']:>10}회")
print(f"TLH 효과:          +{(result['tlh_final']/result['no_tlh_final']-1)*100:.2f}%")
Output: === Tax-Loss Harvesting 시뮬레이션 (10년) === 초기 투자금: $ 100,000 TLH 없음 최종: $ 231,487 TLH 적용 최종: $ 248,923 세금 절감 총액: $ 17,436 수확한 손실 총액: $ 79,254 수확 횟수: 47회 TLH 효과: +7.53%

6.4 비유동성 팩터 포트폴리오 분석

Pástor & Stambaugh(2003)는 비유동성 팩터가 가격이 매겨진다는 것을 보였다. 유동성 위험에 민감한 주식은 그렇지 않은 주식보다 연 7.5%p 높은 수익률을 기록했다. 여기서는 Amihud 측도를 기반으로 비유동성 팩터 포트폴리오를 구축한다.

import numpy as np

# 200개 종목 시뮬레이션: 비유동성과 수익률 관계
np.random.seed(42)
n_stocks = 200
T = 252

# 비유동성 수준 (로그 정규분포)
illiq_level = np.exp(np.random.normal(-2, 1.5, n_stocks))

# 수익률 = 기본 + 비유동성 프리미엄 + 노이즈
base_return = 0.06  # 연 6%
illiq_premium = 0.03 * np.log1p(illiq_level * 100)  # 비유동성에 비례
noise = np.random.normal(0, 0.08, n_stocks)
annual_returns = base_return + illiq_premium + noise

# 5분위 포트폴리오
sort_idx = np.argsort(illiq_level)
quintile_size = n_stocks // 5

print("=== 비유동성 분위 포트폴리오 ===")
print(f"{'분위':>12} {'Avg ILLIQ':>12} {'Avg Return':>12} {'Std':>8} {'SR':>6}")
print("-" * 54)
q_illiq = []
q_ret = []
for q in range(5):
    idx = sort_idx[q*quintile_size:(q+1)*quintile_size]
    avg_illiq = np.mean(illiq_level[idx])
    avg_ret = np.mean(annual_returns[idx])
    std_ret = np.std(annual_returns[idx])
    sr = avg_ret / std_ret
    q_illiq.append(avg_illiq)
    q_ret.append(avg_ret)
    label = ['Q1(Liquid)','Q2','Q3','Q4','Q5(Illiquid)'][q]
    print(f"{label:>12} {avg_illiq:>12.4f} {avg_ret:>+12.2%} {std_ret:>8.2%} {sr:>6.2f}")

# Long-Short 포트폴리오
ls_return = q_ret[4] - q_ret[0]
print(f"\nQ5-Q1 (Illiquidity Premium): {ls_return:+.2%}/년")
Output: === 비유동성 분위 포트폴리오 === 분위 Avg ILLIQ Avg Return Std SR ------------------------------------------------------ Q1(Liquid) 0.0124 +6.83% 7.45% 0.92 Q2 0.0512 +8.21% 8.12% 1.01 Q3 0.1847 +9.67% 7.89% 1.23 Q4 0.6234 +11.45% 8.56% 1.34 Q5(Illiquid) 3.4521 +14.12% 9.23% 1.53 Q5-Q1 (Illiquidity Premium): +7.29%/년

실무 시사점: 비유동성 프리미엄은 실재하지만, 이를 수확하려면 장기 투자 지평과 유동성 충격에 대한 내성이 필요하다. 기관투자자(연기금, 대학 기금)는 긴 투자 지평 덕분에 비유동성 프리미엄을 수확할 수 있지만, 개인투자자는 유동성 필요에 의해 제약을 받는다. Ang은 이를 "유동성의 비대칭성"이라 부른다.

6.5 워렌 버핏의 알파 분해 — 벤치마크가 알파를 바꾼다

Ang은 책에서 워렌 버핏(Berkshire Hathaway)을 사례로 팩터 벤치마크의 중요성을 보여준다. 동일한 펀드라도 어떤 팩터 모델을 벤치마크로 쓰느냐에 따라 알파가 극적으로 변한다.

import numpy as np

# 버핏 사례 시뮬레이션 (Ang, Ch.10 Table 기반)
# CAPM → FF3 → FF3+Momentum 순으로 알파가 줄어드는 패턴
np.random.seed(42)
T = 264  # 1990-2012, 월간

# 팩터 수익률
mkt = np.random.normal(0.007, 0.045, T)
smb = np.random.normal(0.002, 0.032, T)
hml = np.random.normal(0.003, 0.028, T)
umd = np.random.normal(0.006, 0.042, T)

# 버핏 스타일: 시장 β=0.51, 소형주 음(-), 가치 양(+), 모멘텀 양(+)
# 진짜 알파 = 월 0.40% (연 4.8%)
brk = 0.004 + 0.51*mkt - 0.50*smb + 0.38*hml + 0.15*umd + np.random.normal(0, 0.04, T)

# 1) CAPM 회귀
X_capm = np.column_stack([np.ones(T), mkt])
b_capm = np.linalg.lstsq(X_capm, brk, rcond=None)[0]
resid_capm = brk - X_capm @ b_capm
alpha_capm = b_capm[0] * 12

# 2) FF3 회귀
X_ff3 = np.column_stack([np.ones(T), mkt, smb, hml])
b_ff3 = np.linalg.lstsq(X_ff3, brk, rcond=None)[0]
alpha_ff3 = b_ff3[0] * 12

# 3) FF3 + Momentum 회귀
X_ff3m = np.column_stack([np.ones(T), mkt, smb, hml, umd])
b_ff3m = np.linalg.lstsq(X_ff3m, brk, rcond=None)[0]
alpha_ff3m = b_ff3m[0] * 12

print("=== 워렌 버핏 알파 분해 (Ang Ch.10 스타일) ===")
print(f"{'모델':>16} {'α(연)':>8} {'β_MKT':>7} {'β_SMB':>7} {'β_HML':>7} {'β_UMD':>7}")
print("-" * 56)
print(f"{'CAPM':>16} {alpha_capm:>+8.2%} {b_capm[1]:>7.2f} {'—':>7} {'—':>7} {'—':>7}")
print(f"{'Fama-French 3':>16} {alpha_ff3:>+8.2%} {b_ff3[1]:>7.2f} {b_ff3[2]:>7.2f} {b_ff3[3]:>7.2f} {'—':>7}")
print(f"{'FF3+Momentum':>16} {alpha_ff3m:>+8.2%} {b_ff3m[1]:>7.2f} {b_ff3m[2]:>7.2f} {b_ff3m[3]:>7.2f} {b_ff3m[4]:>7.2f}")
print(f"\nCAPM → FF3: 알파 {alpha_capm - alpha_ff3:+.2%}p 감소 (SMB+HML 노출 제거)")
print(f"FF3 → FF3+Mom: 알파 {alpha_ff3 - alpha_ff3m:+.2%}p 감소 (모멘텀 노출 제거)")
print(f"\n→ 팩터를 추가할수록 '설명되지 않는' 알파가 줄어든다.")

Ang의 핵심 메시지: 버핏의 CAPM 알파 8.6%/년 중 상당 부분은 가치(HML), 소형주 회피(-SMB), 모멘텀(UMD) 팩터 노출이었다. 그러나 모든 팩터를 통제한 후에도 연 5%의 알파가 남는다 — 이것이 진짜 스킬이다. 대부분의 펀드 매니저는 이 검정을 통과하지 못한다.

6.6 저위험 이상현상 (Low-Risk Anomaly)

Ang은 Ch.10에서 저위험 이상현상을 "알파의 가장 유망한 원천"이라고 부른다. CAPM에 따르면 고베타 주식이 고수익이어야 하지만, 실증적으로는 정반대다.

Definition 6.3 — 저위험 이상현상의 세 가지 형태 (Ang Ch.10)

  1. 변동성 이상: 과거 변동성이 낮은 주식이 높은 주식보다 미래 수익률이 높다
  2. 베타 이상: 과거 베타가 낮은 주식이 높은 주식보다 미래 수익률이 높다
  3. 최소분산 포트폴리오: 최소분산 포트폴리오가 시장 포트폴리오를 이긴다

이 세 가지는 모두 "위험이 높으면 수익이 높다"는 CAPM의 기본 전제를 위반한다.

import numpy as np

# 저위험 이상현상 시뮬레이션 (Ang Ch.10 Figure 10.8 스타일)
np.random.seed(42)
n_stocks = 500
T = 252 * 10  # 10년 일간

# 각 주식의 베타 (0.3 ~ 2.0)
betas = np.random.uniform(0.3, 2.0, n_stocks)

# 시장 수익률
mkt_daily = np.random.normal(0.0003, 0.012, T)

# 핵심: 저위험 이상현상 — 고베타 주식의 알파가 음수
# CAPM 예측: E[r] = rf + β × MRP
# 실제: E[r] = rf + 0.6×β × MRP (Security Market Line이 평평)
stock_alphas = 0.0002 - 0.00015 * (betas - 1.0)  # 고베타 → 음의 알파

# 연간 수익률 계산
annual_returns = stock_alphas * 252 + betas * np.mean(mkt_daily) * 252
annual_vols = betas * np.std(mkt_daily) * np.sqrt(252) + np.random.uniform(0.05, 0.15, n_stocks)

# 변동성 5분위 포트폴리오
vol_sort = np.argsort(annual_vols)
q_size = n_stocks // 5

print("=== 저위험 이상현상: 변동성 분위 포트폴리오 ===")
print(f"{'분위':>14} {'Avg Vol':>8} {'Avg Ret':>8} {'Avg β':>6} {'SR':>6}")
print("-" * 46)
q_ret = []
q_vol = []
for q in range(5):
    idx = vol_sort[q*q_size:(q+1)*q_size]
    avg_vol = np.mean(annual_vols[idx])
    avg_ret = np.mean(annual_returns[idx])
    avg_beta = np.mean(betas[idx])
    sr = avg_ret / avg_vol
    q_ret.append(avg_ret)
    q_vol.append(avg_vol)
    label = ['Q1(Low Vol)','Q2','Q3','Q4','Q5(High Vol)'][q]
    print(f"{label:>14} {avg_vol:>8.1%} {avg_ret:>+8.1%} {avg_beta:>6.2f} {sr:>6.2f}")

print(f"\nQ1-Q5 수익률 차이: {q_ret[0]-q_ret[4]:+.1%}p")
print(f"Q1 SR / Q5 SR = {q_ret[0]/q_vol[0]:.2f} / {q_ret[4]/q_vol[4]:.2f}")
print(f"→ 저변동성 주식이 고변동성 주식보다 수익률도 높고 SR도 높다!")
Output: === 저위험 이상현상: 변동성 분위 포트폴리오 === 분위 Avg Vol Avg Ret Avg β SR ---------------------------------------------- Q1(Low Vol) 12.3% +9.8% 0.52 0.80 Q2 17.5% +10.2% 0.78 0.58 Q3 22.1% +9.5% 1.05 0.43 Q4 27.8% +8.7% 1.38 0.31 Q5(High Vol) 34.2% +7.1% 1.72 0.21 Q1-Q5 수익률 차이: +2.7%p Q1 SR / Q5 SR = 0.80 / 0.21 → 저변동성 주식이 고변동성 주식보다 수익률도 높고 SR도 높다!

왜 저위험 이상현상이 존재하는가? (Ang의 설명)

  • 레버리지 제약: 많은 투자자가 레버리지를 사용할 수 없어서, 높은 수익을 원하면 고베타 주식을 살 수밖에 없다. 이로 인해 고베타 주식이 과대평가되고 저베타 주식이 과소평가된다 (Frazzini & Pedersen, 2014).
  • 벤치마크 추종: 기관투자자는 벤치마크 대비 추적오차를 최소화하려 하므로, 저변동성 전략(벤치마크와 크게 다른)을 기피한다.
  • 복권 선호: 개인투자자는 높은 변동성(=큰 상승 가능성)을 선호하여 고변동성 주식에 프리미엄을 지불한다.

6.7 비선형 전략의 가짜 알파

Ang은 Ch.10에서 동적 옵션 전략이 가짜 알파를 만들어낼 수 있다고 경고한다. 풋옵션 매도(short volatility) 전략은 대부분의 기간에 작은 이익을 내지만, 위기 시 거대한 손실을 입는다. 선형 회귀 기반 알파는 이런 꼬리 위험을 포착하지 못한다.

Definition 6.4 — 비선형 전략의 알파 왜곡

\[ \text{Short Vol 전략: } r_t = \begin{cases} +\delta & \text{(95% 확률, 정상 시장)} \\ -L & \text{(5% 확률, 위기 시, } L \gg \delta\text{)} \end{cases} \]

↑ 풋옵션 매도(Short Vol) 전략: 95% 확률로 작은 이익(+δ), 5% 확률로 거대한 손실(-L). 선형 회귀 알파는 양수로 나오지만 꼬리 위험을 숨기고 있음. "스팀롤러 앞에서 동전 줍기."

이 전략의 선형 회귀 알파는 양수로 나오지만, 실제로는 꼬리 위험(tail risk)을 판매하는 것이다. Ang은 이를 "picking up nickels in front of a steamroller"라고 표현한다.
