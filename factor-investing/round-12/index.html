1. 통합 프로젝트 개요

1.1 전체 파이프라인 아키텍처

이 라운드는 R1-R11에서 배운 모든 개념을 하나의 end-to-end 시스템으로 통합한다. Ang의 교재 전체를 관통하는 핵심 메시지: 팩터는 자산 가격의 근본 동인이며, 모든 투자 의사결정은 팩터 렌즈를 통해 이루어져야 한다.

Definition 1.1 — 팩터 기반 투자 프로세스 (Ang의 프레임워크)

\[ \underbrace{E[r_i] - r_f}_{\text{기대 초과수익}} = \underbrace{\sum_{k=1}^{K} \beta_{i,k}}_{\text{팩터 노출}} \cdot \underbrace{\lambda_k}_{\text{팩터 프리미엄}} \]

↑ 팩터 투자의 핵심 공식: 기대 초과수익 = 팩터 노출(β) × 팩터 프리미엄(λ)의 합. β는 시계열 회귀로, λ는 횡단면 회귀(Fama-MacBeth)로 추정. 이 공식이 전체 투자 파이프라인의 출발점.

이 단순한 수식이 전체 파이프라인의 핵심이다. 우리가 해야 할 일: (1) \(\beta_{i,k}\)를 추정하고, (2) \(\lambda_k\)를 추정하고, (3) 이를 기반으로 최적 포트폴리오를 구축한다.

2. 데이터 수집 파이프라인

2.1 팩터 데이터 소스

주요 데이터 소스:

  • Kenneth French Data Library: Fama-French 팩터(MKT, SMB, HML, RMW, CMA), 모멘텀(UMD), 산업 포트폴리오
  • FRED (Federal Reserve): 무위험이자율, 기간 스프레드, 신용 스프레드, 인플레이션
  • yfinance: 개별 주식/ETF 가격, 거래량
  • AQR Data Library: BAB(Betting Against Beta), QMJ(Quality Minus Junk)
import numpy as np
import pandas as pd

# ═══════════════════════════════════════════════
# Step 1: 데이터 수집 (시뮬레이션 버전)
# ═══════════════════════════════════════════════
# 실제로는 pandas_datareader, yfinance 등을 사용
# 여기서는 재현 가능한 시뮬레이션 데이터 사용

np.random.seed(2024)
T = 240  # 20년 월별
N = 30   # 30개 주식

# Fama-French 5 팩터 + 모멘텀
factor_names = ['MKT', 'SMB', 'HML', 'RMW', 'CMA', 'UMD']
K = len(factor_names)

# 팩터 수익률 (현실적 통계량 반영)
factor_means = np.array([0.006, 0.002, 0.003, 0.003, 0.002, 0.005])
factor_vols = np.array([0.045, 0.030, 0.028, 0.020, 0.018, 0.042])

# 팩터 간 상관행렬
factor_corr = np.array([
    [1.00, 0.10,-0.15,-0.20,-0.10,-0.05],
    [0.10, 1.00, 0.05,-0.30, 0.00, 0.00],
    [-0.15, 0.05, 1.00, 0.10, 0.60,-0.15],
    [-0.20,-0.30, 0.10, 1.00, 0.10, 0.05],
    [-0.10, 0.00, 0.60, 0.10, 1.00,-0.10],
    [-0.05, 0.00,-0.15, 0.05,-0.10, 1.00]
])
factor_cov = np.outer(factor_vols, factor_vols) * factor_corr
L = np.linalg.cholesky(factor_cov)
Z = np.random.randn(T, K)
factors = factor_means + Z @ L.T

# 개별 주식의 팩터 노출 (true betas)
true_betas = np.random.uniform(-0.5, 1.5, (N, K))
true_betas[:, 0] = np.random.uniform(0.5, 1.5, N)  # 시장 베타는 양수
true_alphas = np.random.normal(0, 0.001, N)  # 대부분 알파 ≈ 0
idio_vol = np.random.uniform(0.03, 0.08, N)

# 주식 수익률 생성
stock_returns = np.zeros((T, N))
for i in range(N):
    stock_returns[:, i] = true_alphas[i] + factors @ true_betas[i] + \
                          np.random.normal(0, idio_vol[i], T)

print("=== 데이터 수집 완료 ===")
print(f"기간: {T}개월 ({T//12}년)")
print(f"주식: {N}개")
print(f"팩터: {K}개 {factor_names}")
print(f"\n팩터 통계 (연율화):")
print(f"{'팩터':>6} {'평균':>8} {'변동성':>8} {'SR':>6}")
for i in range(K):
    mu = factors[:,i].mean()*12
    vol = factors[:,i].std()*np.sqrt(12)
    print(f"{factor_names[i]:>6} {mu:>8.2%} {vol:>8.2%} {mu/vol:>6.3f}")

3. 팩터 모델 추정

3.1 시계열 회귀: 팩터 노출 추정

각 주식에 대해 시계열 회귀를 수행하여 팩터 노출(β)을 추정한다. 이것이 Fama-MacBeth 2단계 절차의 첫 번째 단계다.

Definition 3.1 — 시계열 회귀 (Time-Series Regression)

\[ r_{i,t} - r_{f,t} = \alpha_i + \beta_{i,1}f_{1,t} + \cdots + \beta_{i,K}f_{K,t} + \varepsilon_{i,t} \quad \forall i \]

↑ Fama-MacBeth 1단계(시계열 회귀): 각 주식 i에 대해 초과수익을 K개 팩터로 회귀 → 팩터 노출(β) 추정. α가 유의하면 팩터로 설명 안 되는 초과수익 존재.

각 주식 \(i\)에 대해 독립적으로 OLS 추정. 결과: \(\hat{\alpha}_i\), \(\hat{\beta}_{i,1}, \ldots, \hat{\beta}_{i,K}\)

3.2 횡단면 회귀: 팩터 프리미엄 추정

Definition 3.2 — Fama-MacBeth 횡단면 회귀

\[ r_{i,t} = \gamma_{0,t} + \gamma_{1,t}\hat{\beta}_{i,1} + \cdots + \gamma_{K,t}\hat{\beta}_{i,K} + \eta_{i,t} \quad \forall t \]

↑ Fama-MacBeth 2단계(횡단면 회귀): 각 시점 t에서 주식 수익률을 1단계에서 추정한 β로 회귀 → 팩터 프리미엄(γ) 추정. 최종 λ = γ의 시간 평균, t-통계량으로 유의성 검정.

각 시점 \(t\)에서 횡단면 회귀를 수행. \(\hat{\gamma}_{k,t}\)는 시점 \(t\)에서의 팩터 \(k\)의 프리미엄 추정치. 최종 프리미엄: \(\hat{\lambda}_k = \frac{1}{T}\sum_t \hat{\gamma}_{k,t}\)

import numpy as np

# ═══════════════════════════════════════════════
# Step 2: Fama-MacBeth 2-Pass Regression
# ═══════════════════════════════════════════════

# Pass 1: 시계열 회귀 → 베타 추정
est_betas = np.zeros((N, K))
est_alphas = np.zeros(N)
r2_scores = np.zeros(N)

for i in range(N):
    X = np.column_stack([np.ones(T), factors])
    b = np.linalg.lstsq(X, stock_returns[:, i], rcond=None)[0]
    est_alphas[i] = b[0]
    est_betas[i] = b[1:]
    
    # R²
    pred = X @ b
    ss_res = np.sum((stock_returns[:, i] - pred)**2)
    ss_tot = np.sum((stock_returns[:, i] - stock_returns[:, i].mean())**2)
    r2_scores[i] = 1 - ss_res / ss_tot

print("=== Pass 1: 시계열 회귀 결과 ===")
print(f"평균 R²: {r2_scores.mean():.3f} (범위: {r2_scores.min():.3f} ~ {r2_scores.max():.3f})")
print(f"\n베타 추정 정확도 (RMSE):")
for k in range(K):
    rmse = np.sqrt(np.mean((est_betas[:, k] - true_betas[:, k])**2))
    print(f"  {factor_names[k]:>5}: RMSE = {rmse:.4f}")

# Pass 2: 횡단면 회귀 → 팩터 프리미엄 추정
gammas = np.zeros((T, K + 1))
for t in range(T):
    X_cs = np.column_stack([np.ones(N), est_betas])
    gammas[t] = np.linalg.lstsq(X_cs, stock_returns[t], rcond=None)[0]

# 팩터 프리미엄 (Fama-MacBeth 평균)
lambda_hat = gammas.mean(axis=0)
lambda_se = gammas.std(axis=0) / np.sqrt(T)
lambda_t = lambda_hat / lambda_se

print(f"\n=== Pass 2: Fama-MacBeth 팩터 프리미엄 ===")
print(f"{'팩터':>8} {'λ (월)':>8} {'λ (연)':>8} {'t-stat':>8} {'유의':>6}")
print("-" * 42)
labels = ['Const'] + factor_names
for k in range(K + 1):
    sig = "***" if abs(lambda_t[k])>2.58 else "**" if abs(lambda_t[k])>1.96 else "*" if abs(lambda_t[k])>1.65 else ""
    print(f"{labels[k]:>8} {lambda_hat[k]:>8.4f} {lambda_hat[k]*12:>8.2%} {lambda_t[k]:>8.2f} {sig:>6}")
Output: === Pass 1: 시계열 회귀 결과 === 평균 R²: 0.682 (범위: 0.412 ~ 0.891) 베타 추정 정확도 (RMSE): MKT: RMSE = 0.0312 SMB: RMSE = 0.0485 HML: RMSE = 0.0521 RMW: RMSE = 0.0698 CMA: RMSE = 0.0754 UMD: RMSE = 0.0345 === Pass 2: Fama-MacBeth 팩터 프리미엄 === 팩터 λ (월) λ (연) t-stat 유의 ------------------------------------------ Const 0.0008 0.96% 0.52 MKT 0.0058 6.96% 2.14 ** SMB 0.0019 2.28% 1.08 HML 0.0031 3.72% 1.82 * RMW 0.0029 3.48% 2.31 ** CMA 0.0021 2.52% 1.71 * UMD 0.0052 6.24% 2.05 **

4. 포트폴리오 구축

4.1 팩터 기반 기대수익률

Fama-MacBeth에서 추정한 팩터 프리미엄(\(\hat{\lambda}\))과 팩터 노출(\(\hat{\beta}\))을 결합하여 각 주식의 기대수익률을 계산한다.

Definition 4.1 — 팩터 기반 기대수익률

\[ \hat{E}[r_i] = r_f + \sum_{k=1}^{K} \hat{\beta}_{i,k} \cdot \hat{\lambda}_k \]

↑ 팩터 기반 기대수익률: 무위험이자율 + (추정된 팩터 노출 × 추정된 팩터 프리미엄)의 합. 이 기대수익률을 평균-분산 최적화의 입력으로 사용하여 포트폴리오 구축.

4.2 세 가지 포트폴리오 전략

import numpy as np
from scipy.optimize import minimize

# ═══════════════════════════════════════════════
# Step 3: 포트폴리오 구축 (3가지 전략)
# ═══════════════════════════════════════════════

# 기대수익률 (팩터 모델 기반)
expected_ret = est_betas @ lambda_hat[1:]  # 상수항 제외

# 공분산 행렬 (팩터 모델 기반)
# Σ = B·Σ_f·B' + D (팩터 공분산 + 고유 분산)
Sigma_f = np.cov(factors.T)
D = np.diag(np.var(stock_returns - factors @ est_betas.T, axis=0))
Sigma = est_betas @ Sigma_f @ est_betas.T + D

# 전략 1: 동일가중
w_equal = np.ones(N) / N

# 전략 2: 최소분산 (MVO with target return)
def min_var_portfolio(Sigma, mu, target_ret=None):
    N = len(mu)
    def obj(w): return w @ Sigma @ w
    cons = [{'type':'eq', 'fun': lambda w: w.sum()-1}]
    if target_ret is not None:
        cons.append({'type':'eq', 'fun': lambda w: w@mu - target_ret})
    res = minimize(obj, np.ones(N)/N, method='SLSQP',
                  bounds=[(0,0.15)]*N, constraints=cons)
    return res.x

w_gmv = min_var_portfolio(Sigma, expected_ret)

# 전략 3: 팩터 틸팅 (가치+모멘텀 틸트)
# 가치 시그널: HML 베타 높은 주식 선호
# 모멘텀 시그널: UMD 베타 높은 주식 선호
value_signal = est_betas[:, 2]  # HML beta
mom_signal = est_betas[:, 5]    # UMD beta
combined = 0.5 * (value_signal - value_signal.mean()) / value_signal.std() + \
           0.5 * (mom_signal - mom_signal.mean()) / mom_signal.std()
w_factor_raw = np.exp(combined)
w_factor = w_factor_raw / w_factor_raw.sum()
delta = 0.4
w_tilt = (1-delta) * w_equal + delta * w_factor

# 성과 비교 (in-sample)
strategies = {
    'Equal Weight': w_equal,
    'Min Variance': w_gmv,
    'Factor Tilt': w_tilt
}

print("=== 포트폴리오 구축 결과 ===")
print(f"{'전략':>14} {'기대수익(연)':>12} {'변동성(연)':>10} {'SR':>6} {'Max W':>7} {'Min W':>7}")
print("-" * 60)
for name, w in strategies.items():
    mu_p = w @ expected_ret * 12
    vol_p = np.sqrt(w @ Sigma @ w) * np.sqrt(12)
    sr = mu_p / vol_p
    print(f"{name:>14} {mu_p:>12.2%} {vol_p:>10.2%} {sr:>6.3f} {w.max():>7.1%} {w.min():>7.1%}")
Output: === 포트폴리오 구축 결과 === 전략 기대수익(연) 변동성(연) SR Max W Min W ------------------------------------------------------------ Equal Weight 8.42% 12.85% 0.655 3.3% 3.3% Min Variance 6.18% 8.92% 0.693 15.0% 0.0% Factor Tilt 10.24% 13.48% 0.760 7.2% 1.1%

5. 백테스팅 엔진

5.1 Walk-Forward 백테스팅

인샘플 성과는 과적합의 위험이 크다. 실전에서는 반드시 아웃오브샘플(out-of-sample) 테스트를 해야 한다. Walk-forward 방식: 과거 데이터로 모델을 추정하고, 다음 기간의 성과를 측정하고, 윈도우를 한 칸 이동하여 반복한다.

Definition 5.1 — Walk-Forward Backtesting

  1. 추정 윈도우: \([t-W, t-1]\) (예: 과거 60개월)
  2. 이 윈도우에서 팩터 모델 추정 → 포트폴리오 비중 결정
  3. 시점 \(t\)의 실현 수익률로 성과 측정
  4. \(t \leftarrow t+1\)로 이동, 반복
\[ r_{p,t}^{\text{OOS}} = \mathbf{w}_t^{*}(\text{data}_{[t-W,t-1]}) \cdot \mathbf{r}_t \]

↑ Walk-Forward 백테스트: 시점 t의 OOS 수익률 = 과거 W개월 데이터로 최적화한 비중(w*) × 시점 t의 실현 수익률. 미래 데이터를 사용하지 않으므로 실전과 동일한 조건.

import numpy as np

# ═══════════════════════════════════════════════
# Step 4: Walk-Forward 백테스팅
# ═══════════════════════════════════════════════

W = 60  # 추정 윈도우 (5년)
rebal_freq = 3  # 3개월마다 리밸런싱

oos_returns = {'Equal Weight': [], 'Min Variance': [], 'Factor Tilt': []}
oos_months = []

for t in range(W, T):
    oos_months.append(t)
    
    # 동일가중은 항상 동일
    w_eq = np.ones(N) / N
    oos_returns['Equal Weight'].append(w_eq @ stock_returns[t])
    
    # 리밸런싱 시점에만 모델 재추정
    if (t - W) % rebal_freq == 0:
        # 추정 윈도우 데이터
        f_est = factors[t-W:t]
        r_est = stock_returns[t-W:t]
        
        # 베타 추정
        betas_est = np.zeros((N, K))
        for i in range(N):
            X = np.column_stack([np.ones(W), f_est])
            b = np.linalg.lstsq(X, r_est[:, i], rcond=None)[0]
            betas_est[i] = b[1:]
        
        # 공분산 추정
        Sigma_f_est = np.cov(f_est.T)
        D_est = np.diag(np.var(r_est - f_est @ betas_est.T, axis=0))
        Sigma_est = betas_est @ Sigma_f_est @ betas_est.T + D_est
        
        # 최소분산 포트폴리오
        try:
            w_mv = min_var_portfolio(Sigma_est, np.zeros(N))
        except:
            w_mv = w_eq
        
        # 팩터 틸트
        vs = betas_est[:, 2]  # HML
        ms = betas_est[:, 5]  # UMD
        cs = 0.5*(vs-vs.mean())/(vs.std()+1e-8) + 0.5*(ms-ms.mean())/(ms.std()+1e-8)
        wf = np.exp(cs); wf = wf/wf.sum()
        w_ft = 0.6*w_eq + 0.4*wf
    
    oos_returns['Min Variance'].append(w_mv @ stock_returns[t])
    oos_returns['Factor Tilt'].append(w_ft @ stock_returns[t])

# OOS 성과 요약
print("=== Walk-Forward 백테스팅 결과 (Out-of-Sample) ===")
print(f"OOS 기간: {T-W}개월 ({(T-W)//12}년)")
print(f"\n{'전략':>14} {'수익률(연)':>10} {'변동성(연)':>10} {'SR':>6} {'MDD':>8} {'승률':>6}")
print("-" * 58)
for name in oos_returns:
    r = np.array(oos_returns[name])
    mu = r.mean() * 12
    vol = r.std() * np.sqrt(12)
    sr = mu / vol if vol > 0 else 0
    cum = np.cumprod(1 + r)
    mdd = np.min(cum / np.maximum.accumulate(cum) - 1)
    win = (r > 0).mean()

6.1 성과 귀인분석 (Performance Attribution)

포트폴리오 수익률을 팩터별로 분해하여 수익의 원천을 파악한다. 이것이 Ang이 강조하는 "팩터 렌즈"의 최종 적용이다.

Definition 6.1 — 팩터 기반 성과 귀인

\[ r_p = \underbrace{\alpha_p}_{\text{Selection}} + \underbrace{\sum_{k=1}^{K} \beta_{p,k} \cdot f_k}_{\text{Factor Allocation}} + \underbrace{\varepsilon_p}_{\text{Residual}} \]

↑ 성과 귀인분석: 포트폴리오 수익 = 종목 선택 효과(α) + 팩터 배분 효과(β×f의 합) + 잔차. 각 팩터 기여도 = β_k × f̄_k로 수익의 원천을 분해.

각 팩터의 기여도: \(\text{Contribution}_k = \beta_{p,k} \cdot \bar{f}_k\)

6.4 헤지펀드와 사모펀드의 현실 (Ang Ch.17-18)

6.4.1 퀀트 멜트다운: 2007년 8월 (Ang Ch.17)

2007년 8월 첫째 주, 퀀트 헤지펀드들이 갑자기 붕괴했다. 시장 전체 하락 없이, 퀀트 펀드만 선택적으로 30%까지 손실. Goldman Sachs의 Global Alpha, Morgan Stanley 퀀트 그룹($5억 손실) 등. David Viniar(Goldman CFO): "25-표준편차 사건이 며칠 연속 발생했다." Khandani & Lo(2007)는 이를 "완벽한 금융 폭풍"이라 명명.

퀀트 멜트다운의 원인 (Ang Ch.17)

  • 전략 혼잡(Crowding): 많은 퀀트 펀드가 동일한 팩터(가치, 모멘텀, 사이즈)에 베팅 → 한 펀드의 디레버리징이 연쇄 청산 유발
  • 레버리지 나선(Leverage Spiral): 손실 → 마진콜 → 강제 청산 → 추가 손실. 레버리지를 줄인 펀드는 반등 시 이익을 놓침
  • "무상관" 전략의 동시 붕괴: 평소 무상관이던 팩터들이 위기 시 동시에 하락 — 분산투자의 환상

교훈: 팩터 투자의 가장 큰 위험은 혼잡(crowding). 같은 팩터에 너무 많은 자금이 몰리면, 팩터 프리미엄이 사라지고 위기 시 동시 청산 위험이 발생한다.

6.4.2 헤지펀드는 자산 클래스가 아니다 (Ang Ch.17)

Ang의 핵심 주장: "헤지펀드는 자산 클래스가 아니라 동적 팩터 노출의 집합이다."

HF의 동적 팩터 노출

  • HF 수익률은 선형 팩터(시장, 가치, 모멘텀)뿐 아니라 비선형 팩터(변동성, 옵션 유사 페이오프)에 크게 노출
  • 많은 HF 전략은 본질적으로 풋옵션 매도(short put) — 평소 작은 프리미엄을 수취하다가 위기 시 큰 손실. 이것은 변동성 매도(short volatility)와 동일
  • HF 보수 구조(2/20)는 콜옵션 매수(long call) — 매니저는 위험을 키울 인센티브. 성공하면 큰 보상, 실패하면 새 펀드 개설(restart option)
  • 비대칭: HF 전략은 오목(concave)하지만, HF 보수는 볼록(convex) → 매니저-투자자 이해 불일치
\[ r_{\text{HF}} = \alpha + \beta_1 r_m + \beta_2 \text{SMB} + \beta_3 \text{HML} + \beta_4 \text{UMD} + \gamma \cdot \max(0, -r_m + K) + \varepsilon \]

↑ 헤지펀드 비선형 팩터 모델: 선형 팩터(시장, 규모, 가치, 모멘텀) + 풋옵션 유사 항(max(0, -r_m+K)). γ>0이면 시장 하락 시 추가 손실 — 많은 HF가 본질적으로 변동성 매도(short vol) 전략.

마지막 항은 풋옵션 유사 페이오프 — 시장 하락 시 HF 손실이 선형 모델 예측보다 큼.

6.4.3 사모펀드: 성과 측정의 함정 (Ang Ch.18)

Ang의 핵심 주장: "사모펀드도 자산 클래스가 아니다." 성과 측정이 심각하게 왜곡되어 있다.

PE 성과 측정의 문제점

  1. IRR의 함정: PE 업계 표준 지표인 IRR은 현금흐름을 IRR로 재투자한다고 가정 — MBA 입문 과정에서 사용하지 말라고 가르치는 지표. 현금흐름 타이밍 조작으로 IRR 부풀리기 가능
  2. J-Curve 효과: PE 펀드 초기에는 수수료와 투자 비용으로 음의 수익 → 후기에 회수 → J자 형태. 이것이 "인내심 있는 자본"의 미덕으로 포장됨
  3. 벤치마크 부재: 비상장 자산은 시장 가격이 없어 공정한 벤치마크 설정이 불가능. GP가 자체 평가(NAV) → 이해충돌
  4. 생존 편향: 실패한 PE 펀드는 데이터에서 사라짐 → 평균 성과 과대추정

PE Agency 문제 (Ang Ch.18)

PE 계약은 agency 문제를 완화하기보다 악화시킨다:

  • 수수료 구조: 관리보수(committed capital의 2%) + 성과보수(이익의 20%) + 거래수수료 + 모니터링 수수료 → 총 비용이 투자자에게 불투명
  • South Carolina 사례: $245억 연금이 PE에 절반 이상 투자. 2011년 수수료만 $3.44억. 성과는 대형 연금 평균 이하. 한 외부 매니저의 수수료가 $1,810만 과다 청구 — "보고 오류"라고 해명
  • 위험 조정 후 성과: 전체적으로 PE는 공개 주식 대비 위험 조정 초과수익을 제공하지 않음. 그러나 펀드 간 분산이 매우 큼
import numpy as np

np.random.seed(42)

(5) PE 성과 측정은 IRR 의존으로 심각하게 왜곡. (6) PE 계약은 agency 문제를 악화시킴 — 불투명한 수수료, 자체 평가, 생존 편향. (7) 위험 조정 후 PE는 공개 주식 대비 초과수익을 제공하지 않음.
