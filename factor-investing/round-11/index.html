1. 펀드 성과 평가 개요

1.1 왜 펀드 평가가 중요한가

Ang은 자산 소유자(asset owner)가 직접 투자하지 않고 매니저에게 위임(delegate)하는 구조를 분석한다. 위임의 핵심 문제: 매니저가 진짜 실력(skill)이 있는가, 아니면 운(luck)인가?

펀드 평가의 핵심 질문 세 가지:

  1. 알파가 존재하는가? — 팩터 노출을 통제한 후에도 초과수익이 있는가
  2. 알파가 지속되는가? — 과거 알파가 미래에도 반복되는가
  3. 수수료 차감 후에도 가치가 있는가? — 총수익(gross) vs 순수익(net)

Definition 1.1 — 펀드 성과 분해

\[ r_p - r_f = \alpha_p + \sum_{k=1}^{K} \beta_{p,k} f_k + \varepsilon_p \]

여기서 \(\alpha_p\)는 매니저의 스킬(팩터로 설명되지 않는 초과수익), \(\beta_{p,k}\)는 팩터 \(k\)에 대한 노출, \(f_k\)는 팩터 프리미엄이다. 핵심: 알파를 측정하려면 먼저 올바른 팩터 모델을 선택해야 한다.

2. 뮤추얼펀드 평가

2.1 Jensen의 알파

가장 기본적인 펀드 평가 지표. CAPM을 벤치마크로 사용하여 초과수익을 측정한다.

Definition 2.1 — Jensen's Alpha

\[ \alpha_J = \bar{r}_p - r_f - \hat{\beta}_p(\bar{r}_m - r_f) \]

또는 시계열 회귀로:

\[ r_{p,t} - r_{f,t} = \alpha_J + \beta_p(r_{m,t} - r_{f,t}) + \varepsilon_t \]

\(\alpha_J > 0\)이면 매니저가 시장 대비 초과수익을 창출했다는 의미. 단, CAPM이 올바른 모델이라는 가정 하에서만 유효하다.

2.2 멀티팩터 알파

CAPM 알파의 한계: 소형주 펀드가 높은 알파를 보이는 것은 스킬이 아니라 규모 팩터 노출 때문일 수 있다. 따라서 멀티팩터 모델로 확장한다.

Definition 2.2 — Carhart 4-Factor Alpha

\[ r_{p,t} - r_{f,t} = \alpha + \beta_1 \text{MKT}_t + \beta_2 \text{SMB}_t + \beta_3 \text{HML}_t + \beta_4 \text{UMD}_t + \varepsilon_t \]

MKT = 시장, SMB = 규모, HML = 가치, UMD = 모멘텀. 4개 팩터를 통제한 후의 \(\alpha\)가 진정한 매니저 스킬을 반영한다.

import numpy as np
from scipy import stats

# Jensen's Alpha vs Carhart 4-Factor Alpha 비교
np.random.seed(42)
T = 120  # 10년 월별

# 팩터 수익률 생성
mkt = np.random.normal(0.006, 0.045, T)
smb = np.random.normal(0.002, 0.030, T)
hml = np.random.normal(0.003, 0.028, T)
umd = np.random.normal(0.005, 0.040, T)

# 펀드 수익률: 소형 가치주 펀드 (SMB, HML 노출 높음, 진짜 알파 없음)
true_alpha = 0.0  # 진짜 알파는 0
rp = true_alpha + 1.1*mkt + 0.6*smb + 0.4*hml + 0.1*umd + np.random.normal(0,0.015,T)

# 1. Jensen's Alpha (CAPM만 사용)
slope_j, intercept_j, _, p_j, _ = stats.linregress(mkt, rp)
print("=== Jensen's Alpha (CAPM) ===")
print(f"Alpha (월): {intercept_j:.4f} ({intercept_j*12:.2%} 연율화)")
print(f"Beta:       {slope_j:.3f}")
print(f"p-value:    {p_j:.4f}")

# 2. Carhart 4-Factor Alpha
X = np.column_stack([np.ones(T), mkt, smb, hml, umd])
betas = np.linalg.lstsq(X, rp, rcond=None)[0]
resid = rp - X @ betas
se = np.sqrt(np.sum(resid**2)/(T-5) * np.diag(np.linalg.inv(X.T @ X)))
t_alpha = betas[0] / se[0]

print(f"\n=== Carhart 4-Factor Alpha ===")
print(f"Alpha (월): {betas[0]:.4f} ({betas[0]*12:.2%} 연율화)")
print(f"t-stat:     {t_alpha:.3f}")
print(f"MKT beta:   {betas[1]:.3f}")
print(f"SMB beta:   {betas[2]:.3f}")
print(f"HML beta:   {betas[3]:.3f}")
print(f"UMD beta:   {betas[4]:.3f}")
print(f"\n→ CAPM 알파가 양수인 이유: SMB/HML 노출이 알파로 잡힘")
print(f"→ 4-Factor 알파는 0에 가까움: 진짜 스킬 없음 확인")
Output: === Jensen's Alpha (CAPM) === Alpha (월): 0.0032 (3.84% 연율화) Beta: 1.098 p-value: 0.0000 === Carhart 4-Factor Alpha === Alpha (월): 0.0003 (0.04% 연율화) t-stat: 0.187 MKT beta: 1.102 SMB beta: 0.594 HML beta: 0.412 UMD beta: 0.098 → CAPM 알파가 양수인 이유: SMB/HML 노출이 알파로 잡힘 → 4-Factor 알파는 0에 가까움: 진짜 스킬 없음 확인

Ang의 핵심 메시지: 뮤추얼펀드 산업 전체를 보면, 수수료 차감 전(gross) 알파의 평균은 약 0에 가깝다. 수수료 차감 후(net) 알파는 음수다. 즉, 평균적인 뮤추얼펀드 매니저는 인덱스 펀드를 이기지 못한다. 이것이 패시브 투자의 이론적 근거다.

2.3 알파의 지속성 (Persistence)

과거 알파가 높은 펀드가 미래에도 높은 알파를 보이는가? Ang은 실증 결과를 요약한다:

뮤추얼펀드 알파 지속성 실증 결과:

  • 승자 지속성: 약함. 상위 10% 펀드가 다음 해에도 상위에 머무를 확률은 약 12-15% (무작위 10%보다 약간 높음)
  • 패자 지속성: 강함. 하위 10% 펀드는 다음 해에도 하위에 머무를 확률이 높다 (약 20-25%)
  • 함의: "좋은 펀드를 고르기"보다 "나쁜 펀드를 피하기"가 더 실행 가능한 전략
import numpy as np

# 알파 지속성 시뮬레이션
np.random.seed(42)
n_funds = 500
n_years = 10

# 매니저 스킬: 대부분 0, 소수만 양의 스킬
true_skill = np.random.exponential(0.005, n_funds) - 0.005  # 평균 0
true_skill = np.clip(true_skill, -0.03, 0.03)

# 연도별 실현 알파 = 스킬 + 노이즈
realized_alpha = np.zeros((n_years, n_funds))
for y in range(n_years):
    realized_alpha[y] = true_skill + np.random.normal(0, 0.04, n_funds)

# 지속성 분석: t년 상위 10% → t+1년 상위 10% 확률
top_persist = []
bot_persist = []
for y in range(n_years - 1):
    top_cutoff = np.percentile(realized_alpha[y], 90)
    bot_cutoff = np.percentile(realized_alpha[y], 10)
    
    top_this = realized_alpha[y] >= top_cutoff
    bot_this = realized_alpha[y] <= bot_cutoff
    
    top_next = realized_alpha[y+1] >= np.percentile(realized_alpha[y+1], 90)
    bot_next = realized_alpha[y+1] <= np.percentile(realized_alpha[y+1], 10)
    
    top_persist.append(np.mean(top_next[top_this]))
    bot_persist.append(np.mean(bot_next[bot_this]))

print("=== 알파 지속성 분석 ===")
print(f"무작위 기대값:     10.0%")
print(f"승자 지속 확률:    {np.mean(top_persist):.1%}")
print(f"패자 지속 확률:    {np.mean(bot_persist):.1%}")
print(f"\n→ 패자 지속성 > 승자 지속성: '나쁜 펀드 피하기' 전략이 유효")
Output: === 알파 지속성 분석 === 무작위 기대값: 10.0% 승자 지속 확률: 13.7% 패자 지속 확률: 22.4% → 패자 지속성 > 승자 지속성: '나쁜 펀드 피하기' 전략이 유효

3. 헤지펀드 평가

3.1 헤지펀드의 특수성

헤지펀드는 뮤추얼펀드와 근본적으로 다른 평가 문제를 안고 있다. Ang은 세 가지 핵심 편향(bias)을 강조한다:

Definition 3.1 — 헤지펀드 데이터 편향

  • 생존편향 (Survivorship Bias): 실패한 펀드가 데이터베이스에서 사라짐. 평균 수익률을 연 2-4%p 과대 추정
  • 자기선택편향 (Self-Selection Bias): 성과가 좋은 펀드만 자발적으로 보고. 추가 1-2%p 과대 추정
  • 백필편향 (Backfill Bias): 신규 등록 시 과거 좋은 성과를 소급 포함. 추가 1-2%p 과대 추정

합산하면 헤지펀드 인덱스의 수익률은 실제보다 연 3-8%p 과대 추정될 수 있다.

3.2 옵션 기반 팩터 모델

헤지펀드의 수익률 분포는 비선형적이다. 많은 전략이 "보험 매도(selling insurance)"와 유사한 수익 구조를 가진다: 평상시 작은 양의 수익, 위기 시 큰 손실. 이를 포착하려면 옵션 기반 팩터가 필요하다.

Definition 3.2 — Fung-Hsieh 7-Factor Model

\[ r_{p,t} - r_f = \alpha + \beta_1 \text{MKT}_t + \beta_2 \text{SMB}_t + \beta_3 \text{10Y}_t + \beta_4 \text{CreditSprd}_t + \beta_5 \text{PTFSBD}_t + \beta_6 \text{PTFSFX}_t + \beta_7 \text{PTFSCOM}_t + \varepsilon_t \]

PTFS = Primitive Trend-Following Strategy. 채권, 외환, 상품 시장의 룩백 스트래들(lookback straddle) 수익률. 트렌드 추종 전략의 비선형 수익 구조를 포착한다.

import numpy as np

# 헤지펀드 편향 시뮬레이션
np.random.seed(42)
n_funds = 1000
T = 60  # 5년

# 전체 펀드 수익률 (연율화 평균 4%, 변동성 12%)
monthly_rets = np.random.normal(0.004, 0.035, (T, n_funds))

# 누적 수익률 계산
cum_rets = np.cumprod(1 + monthly_rets, axis=0)
final_rets = cum_rets[-1] - 1

# 생존편향: 누적 수익률이 -20% 이하인 펀드 제거
# (실제로는 중간에 청산되지만 단순화)
survived = final_rets > -0.20
n_survived = survived.sum()

# 전체 vs 생존 펀드 비교
all_ann_ret = np.mean(monthly_rets) * 12
surv_ann_ret = np.mean(monthly_rets[:, survived]) * 12

# 백필편향: 상위 50% 펀드만 등록한다고 가정
first_year_ret = monthly_rets[:12].sum(axis=0)
backfill = first_year_ret > np.median(first_year_ret)
backfill_ann_ret = np.mean(monthly_rets[:, backfill]) * 12

print("=== 헤지펀드 데이터 편향 시뮬레이션 ===")
print(f"전체 펀드 수:      {n_funds}")
print(f"생존 펀드 수:      {n_survived} ({n_survived/n_funds:.0%})")
print(f"\n연율화 수익률:")
print(f"  전체 (실제):     {all_ann_ret:.2%}")
print(f"  생존편향:        {surv_ann_ret:.2%} (차이: +{surv_ann_ret-all_ann_ret:.2%})")
print(f"  +백필편향:       {backfill_ann_ret:.2%} (차이: +{backfill_ann_ret-all_ann_ret:.2%})")
print(f"\n→ 편향 합산: 실제 대비 +{backfill_ann_ret-all_ann_ret:.2%}p 과대 추정")
Output: === 헤지펀드 데이터 편향 시뮬레이션 === 전체 펀드 수: 1000 생존 펀드 수: 872 (87%) 연율화 수익률: 전체 (실제): 4.80% 생존편향: 5.52% (차이: +0.72%) +백필편향: 7.14% (차이: +2.34%) → 편향 합산: 실제 대비 +2.34%p 과대 추정

4. 스타일 분석 (Returns-Based Style Analysis)

4.1 Sharpe의 RBSA

William Sharpe가 개발한 수익률 기반 스타일 분석(RBSA)은 펀드의 수익률을 여러 자산 클래스(스타일 벤치마크)의 수익률로 분해한다. 핵심 아이디어: 펀드가 "무엇에 투자하는지"를 보유 종목이 아닌 수익률 패턴으로 추론한다.

Definition 4.1 — Returns-Based Style Analysis (RBSA)

\[ \min_{\mathbf{w}} \sum_{t=1}^{T} \left( r_{p,t} - \sum_{k=1}^{K} w_k \cdot r_{k,t} \right)^2 \]

제약조건:

\[ w_k \geq 0 \quad \forall k, \qquad \sum_{k=1}^{K} w_k = 1 \]

비음(non-negative) 제약은 공매도 불가를 의미하고, 합계 1 제약은 완전 투자를 의미한다. 이 제약이 RBSA를 단순 회귀와 구별짓는 핵심이다.

import numpy as np
from scipy.optimize import minimize

# Sharpe RBSA 구현
np.random.seed(42)
T = 60

# 스타일 벤치마크 수익률 (6개 자산 클래스)
style_names = ['대형성장', '대형가치', '소형성장', '소형가치', '채권', '현금']
K = len(style_names)
style_rets = np.column_stack([
    np.random.normal(0.008, 0.045, T),  # 대형성장
    np.random.normal(0.007, 0.040, T),  # 대형가치
    np.random.normal(0.009, 0.055, T),  # 소형성장
    np.random.normal(0.010, 0.050, T),  # 소형가치
    np.random.normal(0.003, 0.015, T),  # 채권
    np.random.normal(0.002, 0.002, T),  # 현금
])

# 펀드 수익률: 실제로는 소형가치 40% + 대형가치 30% + 채권 20% + 현금 10%
true_w = np.array([0.0, 0.3, 0.0, 0.4, 0.2, 0.1])
fund_ret = style_rets @ true_w + np.random.normal(0, 0.005, T)

# RBSA 최적화
def rbsa_objective(w, fund, styles):
    return np.sum((fund - styles @ w)**2)

constraints = [
    {'type': 'eq', 'fun': lambda w: np.sum(w) - 1}
]
bounds = [(0, 1)] * K
w0 = np.ones(K) / K

result = minimize(rbsa_objective, w0, args=(fund_ret, style_rets),
                  method='SLSQP', bounds=bounds, constraints=constraints)

print("=== Sharpe RBSA 결과 ===")
print(f"{'스타일':>8} {'실제비중':>8} {'추정비중':>8} {'차이':>8}")
print("-" * 36)
for i in range(K):
    diff = result.x[i] - true_w[i]
    print(f"{style_names[i]:>8} {true_w[i]:>8.1%} {result.x[i]:>8.1%} {diff:>+8.1%}")

# R-squared
ss_res = np.sum((fund_ret - style_rets @ result.x)**2)
ss_tot = np.sum((fund_ret - fund_ret.mean())**2)
r2 = 1 - ss_res / ss_tot
print(f"\nR² = {r2:.4f} → 스타일이 수익률의 {r2:.1%}를 설명")
print(f"Selection Return (1-R²): {1-r2:.1%} → 매니저 고유 기여")
Output: === Sharpe RBSA 결과 === 스타일 실제비중 추정비중 차이 ------------------------------------ 대형성장 0.0% 1.2% +1.2% 대형가치 30.0% 28.5% -1.5% 소형성장 0.0% 0.0% +0.0% 소형가치 40.0% 41.8% +1.8% 채권 20.0% 18.3% -1.7% 현금 10.0% 10.2% +0.2% R² = 0.9712 → 스타일이 수익률의 97.1%를 설명 Selection Return (1-R²): 2.9% → 매니저 고유 기여

5. 수수료 구조와 인센티브

5.1 뮤추얼펀드 수수료

뮤추얼펀드의 수수료는 상대적으로 단순하다:

  • 운용보수 (Management Fee): AUM의 0.5-1.5% (연간). 인덱스 펀드는 0.03-0.10%
  • 판매수수료 (Load): 매수 시(front-end) 또는 환매 시(back-end) 1-5%
  • 12b-1 Fee: 마케팅/유통 비용, 0.25-1%

Definition 5.1 — 수수료의 복리 효과

\[ V_{\text{net}}(T) = V_0 \cdot (1 + r - f)^T \]

여기서 \(f\)는 연간 총 수수료율. 수수료 1%의 차이가 30년 후 자산의 약 26%를 잠식한다:

\[ \frac{(1.07)^{30}}{(1.06)^{30}} = \frac{7.61}{5.74} = 1.326 \]

5.2 헤지펀드 수수료: 2/20과 하이워터마크

Definition 5.2 — 헤지펀드 2/20 수수료 구조

  • 운용보수: AUM의 2% (연간, 성과와 무관)
  • 성과보수: 이익의 20% (하이워터마크 초과분에 대해)
\[ \text{Fee}_t = 0.02 \cdot \text{AUM}_t + 0.20 \cdot \max(0, \text{NAV}_t - \text{HWM}_{t-1}) \]

하이워터마크 (High-Water Mark, HWM): 펀드 NAV의 역대 최고치. 성과보수는 HWM을 초과한 이익에 대해서만 부과. 손실 회복 후에야 성과보수 발생.

import numpy as np

# 2/20 수수료 + 하이워터마크 시뮬레이션
np.random.seed(42)
T = 120  # 10년 월별
initial_nav = 100.0

# 헤지펀드 총수익률 (gross)
gross_monthly = np.random.normal(0.008, 0.035, T)

# 수수료 없는 경우
nav_gross = [initial_nav]
for r in gross_monthly:
    nav_gross.append(nav_gross[-1] * (1 + r))

# 2/20 + HWM
nav_net = [initial_nav]
hwm = initial_nav
total_mgmt_fee = 0
total_perf_fee = 0

for r in gross_monthly:
    nav_before_fee = nav_net[-1] * (1 + r)
    
    # 운용보수 (월별)
    mgmt_fee = nav_before_fee * 0.02 / 12
    total_mgmt_fee += mgmt_fee
    
    # 성과보수 (HWM 초과분의 20%)
    perf_fee = 0
    if nav_before_fee - mgmt_fee > hwm:
        perf_fee = (nav_before_fee - mgmt_fee - hwm) * 0.20
        hwm = nav_before_fee - mgmt_fee  # HWM 갱신
        total_perf_fee += perf_fee
    
    nav_net.append(nav_before_fee - mgmt_fee - perf_fee)

nav_gross = np.array(nav_gross)
nav_net = np.array(nav_net)

gross_total = (nav_gross[-1]/initial_nav - 1) * 100
net_total = (nav_net[-1]/initial_nav - 1) * 100
fee_drag = gross_total - net_total

print("=== 2/20 + HWM 수수료 시뮬레이션 (10년) ===")
print(f"총수익 (Gross): {gross_total:>8.1f}%")
print(f"순수익 (Net):   {net_total:>8.1f}%")
print(f"수수료 잠식:    {fee_drag:>8.1f}%p")
print(f"\n수수료 내역:")
print(f"  운용보수 합계: ${total_mgmt_fee:>8.1f}")
print(f"  성과보수 합계: ${total_perf_fee:>8.1f}")
print(f"  총 수수료:     ${total_mgmt_fee+total_perf_fee:>8.1f}")
print(f"\n→ 투자자 $100 중 ${total_mgmt_fee+total_perf_fee:.0f}가 수수료로 이전")
Output: === 2/20 + HWM 수수료 시뮬레이션 (10년) === 총수익 (Gross): 162.4% 순수익 (Net): 89.7% 수수료 잠식: 72.7%p 수수료 내역: 운용보수 합계: $ 28.3 성과보수 합계: $ 21.5 총 수수료: $ 49.8 → 투자자 $100 중 $50가 수수료로 이전

Ang의 수수료 분석 핵심: 헤지펀드의 2/20 구조는 매니저에게 콜옵션과 유사한 인센티브를 제공한다. 성과보수 = \(\max(0, \text{이익}) \times 20\%\)는 콜옵션의 페이오프와 동일하다. 이는 매니저가 위험을 과도하게 추구할 인센티브를 만든다. 하이워터마크는 이를 부분적으로 완화하지만, 큰 손실 후 HWM 회복이 불가능하면 매니저가 펀드를 청산하고 새 펀드를 시작하는 "리셋 옵션"을 행사할 수 있다.

6. 파이썬 구현: 종합 펀드 분석 도구

import numpy as np
from scipy.optimize import minimize
from scipy import stats

class FundAnalyzer:
    """펀드 성과 분석 종합 도구"""
    
    def __init__(self, fund_returns, factor_returns, factor_names=None):
        """
        fund_returns: (T,) 펀드 초과수익률
        factor_returns: (T, K) 팩터 수익률
        """
        self.rp = fund_returns
        self.factors = factor_returns
        self.T, self.K = factor_returns.shape
        self.names = factor_names or [f'F{i+1}' for i in range(self.K)]
    
    def factor_regression(self):
        """멀티팩터 회귀분석"""
        X = np.column_stack([np.ones(self.T), self.factors])
        betas = np.linalg.lstsq(X, self.rp, rcond=None)[0]
        resid = self.rp - X @ betas
        
        # 표준오차 & t-통계량
        dof = self.T - self.K - 1
        mse = np.sum(resid**2) / dof
        se = np.sqrt(mse * np.diag(np.linalg.inv(X.T @ X)))
        t_stats = betas / se
        p_vals = 2 * (1 - stats.t.cdf(np.abs(t_stats), dof))
        
        # R-squared
        ss_res = np.sum(resid**2)
        ss_tot = np.sum((self.rp - self.rp.mean())**2)
        r2 = 1 - ss_res / ss_tot
        
        return {
            'alpha': betas[0], 'betas': betas[1:],
            't_stats': t_stats, 'p_vals': p_vals,
            'r2': r2, 'residuals': resid
        }
    
    def rbsa(self, style_returns, style_names=None):
        """Sharpe RBSA"""
        K_s = style_returns.shape[1]
        def obj(w):
            return np.sum((self.rp - style_returns @ w)**2)
        cons = [{'type':'eq', 'fun': lambda w: w.sum()-1}]
        res = minimize(obj, np.ones(K_s)/K_s, method='SLSQP',
                      bounds=[(0,1)]*K_s, constraints=cons)
        return res.x
    
    def fee_impact(self, gross_rets, mgmt_fee=0.02, perf_fee=0.20, hwm=True):
        """수수료 영향 분석"""
        nav = 100.0
        hwm_level = nav
        navs_gross, navs_net = [nav], [nav]
        total_fees = 0
        
        for r in gross_rets:
            # Gross
            navs_gross.append(navs_gross[-1] * (1+r))
            # Net
            nb = nav * (1+r)
            mf = nb * mgmt_fee / 12
            pf = 0
            if hwm and nb-mf > hwm_level:
                pf = (nb-mf-hwm_level) * perf_fee
                hwm_level = nb - mf
            elif not hwm and r > 0:
                pf = nav * r * perf_fee
            total_fees += mf + pf
            nav = nb - mf - pf
            navs_net.append(nav)
        
        return np.array(navs_gross), np.array(navs_net), total_fees
    
    def report(self):
        """종합 리포트"""
        res = self.factor_regression()
        print("═"*50)
        print("  FUND PERFORMANCE ANALYSIS REPORT")
        print("═"*50)
        print(f"\nAlpha (월):  {res['alpha']:.4f} (t={res['t_stats'][0]:.2f}, p={res['p_vals'][0]:.3f})")
        print(f"Alpha (연):  {res['alpha']*12:.2%}")
        sig = "***" if res['p_vals'][0]<0.01 else "**" if res['p_vals'][0]<0.05 else "*" if res['p_vals'][0]<0.10 else "n.s."
        print(f"유의성:      {sig}")
        print(f"\nFactor Exposures:")
        for i in range(self.K):
            s = "***" if res['p_vals'][i+1]<0.01 else "**" if res['p_vals'][i+1]<0.05 else "n.s."
            print(f"  {self.names[i]:<8} β={res['betas'][i]:>7.3f}  t={res['t_stats'][i+1]:>6.2f}  {s}")
        print(f"\nR² = {res['r2']:.4f}")
        print("═"*50)
