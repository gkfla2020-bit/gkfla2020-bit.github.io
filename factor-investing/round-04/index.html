1. 팩터란 무엇인가

1.1 Ang의 팩터 정의

Andrew Ang은 팩터(factor)를 다음과 같이 정의한다: 팩터란 자산 수익률의 횡단면적 차이를 설명하는 체계적 위험의 원천(systematic source of risk)이며, 그 위험을 감수한 대가로 프리미엄(premium)을 제공하는 것이다.

Definition 1.1 — 팩터 (Factor)

팩터는 다음 세 가지 조건을 만족하는 변수다:

  1. 체계적(Systematic): 분산투자로 제거할 수 없는 위험이다
  2. 프리미엄(Premium): 장기적으로 양(+)의 기대수익률을 제공한다
  3. 횡단면 설명력: 자산 간 수익률 차이를 설명한다

핵심 구분은 체계적 위험(systematic risk)과 고유 위험(idiosyncratic risk)이다. 고유 위험은 분산투자로 제거 가능하므로 프리미엄이 없다. 오직 체계적 위험만이 보상받는다.

왜 팩터인가? Ang은 자산 클래스(asset class) 대신 팩터로 사고해야 한다고 주장한다. 주식, 채권, 부동산이라는 자산 클래스 분류는 표면적이다. 그 이면에는 시장(market), 가치(value), 규모(size), 모멘텀(momentum) 등의 팩터가 수익률을 구동한다. 자산은 팩터의 묶음(bundle of factors)이다.

1.2 팩터 모델의 일반 형태

임의의 자산 \(i\)의 초과수익률은 \(K\)개 팩터의 선형결합으로 표현된다:

\[ r_i - r_f = \alpha_i + \beta_{i1}f_1 + \beta_{i2}f_2 + \cdots + \beta_{iK}f_K + \epsilon_i \]

여기서 \(f_k\)는 팩터 수익률, \(\beta_{ik}\)는 팩터 노출도(factor loading), \(\alpha_i\)는 팩터로 설명되지 않는 초과수익(알파), \(\epsilon_i\)는 고유 위험이다. CAPM은 \(K=1\)이고 유일한 팩터가 시장 포트폴리오인 가장 단순한 팩터 모델이다.

2. CAPM 도출

2.1 평균-분산에서 CAPM으로

Round 2에서 우리는 접선 포트폴리오(tangency portfolio)가 샤프비율을 극대화하는 최적 위험 포트폴리오임을 보았다. CAPM의 핵심 통찰은 다음과 같다: 모든 투자자가 동일한 기대수익률과 공분산을 사용한다면(동질적 기대, homogeneous expectations), 모든 투자자의 접선 포트폴리오는 동일하다.

시장 균형에서 모든 투자자가 동일한 위험 포트폴리오를 보유하므로, 그 포트폴리오는 시장에 존재하는 모든 자산을 시가총액 비중으로 포함해야 한다. 이것이 시장 포트폴리오(market portfolio)다.

Definition 2.1 — CAPM의 가정

  1. 투자자는 평균-분산 최적화를 수행한다 (Round 2)
  2. 동질적 기대: 모든 투자자가 동일한 \(\boldsymbol{\mu}\)와 \(\boldsymbol{\Sigma}\)를 사용한다
  3. 무위험자산이 존재하며, 무제한 차입/대출이 가능하다
  4. 세금, 거래비용이 없다

2.2 베타의 도출

시장 포트폴리오가 접선 포트폴리오이므로, Round 2의 접선 포트폴리오 공식에서:

\[ \mathbf{w}_m = \frac{\boldsymbol{\Sigma}^{-1}(\boldsymbol{\mu} - r_f \mathbf{1})}{\mathbf{1}^T \boldsymbol{\Sigma}^{-1}(\boldsymbol{\mu} - r_f \mathbf{1})} \]

이를 정리하면 개별 자산 \(i\)의 기대수익률은:

\[ E[r_i] = r_f + \beta_i \big(E[r_m] - r_f\big) \]

여기서 베타는 자산 \(i\)의 수익률이 시장 수익률에 얼마나 민감한지를 측정한다:

Definition 2.2 — 베타 (Beta)

\[ \beta_i = \frac{\text{Cov}(r_i, r_m)}{\text{Var}(r_m)} = \frac{\sigma_{im}}{\sigma_m^2} \]

베타는 시장 수익률 1% 변동 시 자산 \(i\)의 수익률이 평균적으로 몇 % 변동하는지를 나타낸다.

  • \(\beta = 1\): 시장과 동일한 민감도 (시장 포트폴리오 자체)
  • \(\beta > 1\): 시장보다 공격적 (예: 기술주, 레버리지 ETF)
  • \(\beta < 1\): 시장보다 방어적 (예: 유틸리티, 필수소비재)
  • \(\beta = 0\): 시장과 무관 (무위험자산)
import numpy as np

# 베타 계산 예시
# 시장 수익률과 개별 주식 수익률 (월간, 12개월)
np.random.seed(42)
r_m = np.random.normal(0.01, 0.04, 60)   # 시장: 월 1%, 변동성 4%
r_i = 1.3 * r_m + np.random.normal(0.002, 0.02, 60)  # 베타=1.3인 주식

# 베타 = Cov(ri, rm) / Var(rm)
beta = np.cov(r_i, r_m)[0, 1] / np.var(r_m, ddof=1)
print(f"추정 베타: {beta:.4f}")
print(f"이론 베타: 1.3000")
Output: 추정 베타: 1.2847 이론 베타: 1.3000

3. Security Market Line (SML)

3.1 SML의 정의

CAPM에 따르면 모든 자산의 기대수익률은 베타의 선형함수다. 이 관계를 \(\beta\)-\(E[r]\) 평면에 그린 것이 증권시장선(Security Market Line, SML)이다.

Definition 3.1 — 증권시장선 (SML)

\[ E[r_i] = r_f + \beta_i \big(E[r_m] - r_f\big) \]

SML은 \(\beta=0\)에서 \(r_f\)를 지나고, \(\beta=1\)에서 \(E[r_m]\)을 지나는 직선이다. 기울기는 시장 리스크 프리미엄 \(E[r_m] - r_f\)이다.

3.2 알파의 정의

SML 위에 있는 자산은 CAPM이 예측하는 수익률을 정확히 제공한다. SML 위쪽에 있는 자산은 양의 알파(positive alpha)를 가지며, 위험 대비 초과수익을 제공한다.

Definition 3.2 — 젠센의 알파 (Jensen's Alpha)

\[ \alpha_i = (r_i - r_f) - \beta_i(r_m - r_f) \]

실현된 초과수익에서 CAPM이 예측하는 초과수익을 뺀 값. \(\alpha > 0\)이면 위험 조정 후에도 초과수익이 존재한다.

SML vs CML: 자본시장선(CML)은 효율적 포트폴리오만 위에 놓이며 x축이 총위험(\(\sigma\))이다. 반면 SML은 모든 자산과 포트폴리오에 적용되며 x축이 체계적 위험(\(\beta\))이다. CML은 포트폴리오 구성 도구이고, SML은 자산 가격결정 도구다.

3.3 R² 해석

CAPM 회귀에서 \(R^2\)는 자산 수익률 변동 중 시장 팩터로 설명되는 비율이다:

\[ R^2 = \frac{\text{체계적 위험}}{\text{총 위험}} = \frac{\beta_i^2 \sigma_m^2}{\sigma_i^2} \]

\(R^2\)가 높으면 시장 팩터 하나로 해당 자산의 움직임을 잘 설명할 수 있다. 대형주는 \(R^2 \approx 0.5\text{-}0.7\), 소형주는 \(R^2 \approx 0.1\text{-}0.3\)이 일반적이다.

4. 베타 추정

4.1 OLS 회귀

베타는 초과수익률의 시계열 회귀로 추정한다. 자산 \(i\)의 초과수익률을 시장 초과수익률에 회귀:

\[ r_{i,t} - r_{f,t} = \alpha_i + \beta_i(r_{m,t} - r_{f,t}) + \epsilon_{i,t} \]

OLS 추정량 \(\hat{\beta}_i\)는 표본 공분산과 표본 분산의 비율이다:

\[ \hat{\beta}_i = \frac{\sum_{t=1}^{T}(r_{i,t}^e - \bar{r}_i^e)(r_{m,t}^e - \bar{r}_m^e)}{\sum_{t=1}^{T}(r_{m,t}^e - \bar{r}_m^e)^2} \]

4.2 statsmodels 구현

import numpy as np
import statsmodels.api as sm

# 시뮬레이션 데이터: 60개월
np.random.seed(42)
T = 60
rf = 0.003  # 월 무위험이자율 0.3%
r_m = np.random.normal(0.01, 0.045, T)  # 시장 수익률

# 주식 A: 공격적 (β=1.4, α=0.2%)
r_a = 0.002 + 1.4 * (r_m - rf) + np.random.normal(0, 0.03, T) + rf

# 초과수익률
excess_m = r_m - rf
excess_a = r_a - rf

# OLS 회귀
X = sm.add_constant(excess_m)  # 상수항 추가 (α 추정)
model = sm.OLS(excess_a, X).fit()

print(model.summary().tables[1])
print(f"\n추정 α: {model.params[0]:.4f} (p-value: {model.pvalues[0]:.4f})")
print(f"추정 β: {model.params[1]:.4f} (p-value: {model.pvalues[1]:.6f})")
print(f"R²:     {model.rsquared:.4f}")
Output: ============================================================================== coef std err t P>|t| [0.025 0.975] ------------------------------------------------------------------------------ const 0.0024 0.004 0.601 0.550 -0.006 0.011 x1 1.3712 0.089 15.412 0.000 1.193 1.549 ============================================================================== 추정 α: 0.0024 (p-value: 0.5503) 추정 β: 1.3712 (p-value: 0.000000) R²: 0.8037

베타 추정 시 주의사항:

  • 데이터 빈도: 일간 데이터는 비동기 거래(nonsynchronous trading) 문제가 있다. 월간 데이터가 표준이다.
  • 추정 기간: 보통 60개월(5년)을 사용한다. 너무 짧으면 추정 오차가 크고, 너무 길면 구조 변화를 반영하지 못한다.
  • 시장 지수: KOSPI, S&P 500 등 시장 대용치(proxy)를 사용한다. Roll(1977)의 비판: 진정한 시장 포트폴리오는 관측 불가능하다.

5. CAPM의 한계

5.1 이상현상 (Anomalies)

CAPM이 유일한 가격결정 모델이라면, 베타만으로 모든 자산의 기대수익률을 설명할 수 있어야 한다. 그러나 실증 연구는 CAPM이 설명하지 못하는 체계적 패턴을 발견했다.

주요 이상현상 (Anomalies)

이상현상발견내용CAPM 문제
규모 효과 (Size)Banz (1981)소형주가 대형주보다 높은 수익률소형주의 높은 수익률이 β로 설명 안 됨
가치 효과 (Value)Fama-French (1992)높은 B/M 주식이 낮은 B/M보다 고수익가치주의 β가 특별히 높지 않음
모멘텀 (Momentum)Jegadeesh-Titman (1993)과거 승자가 계속 이기는 경향모멘텀 수익률과 β 무관
저변동성 (Low Vol)Ang et al. (2006)저변동성 주식이 고변동성보다 고수익CAPM과 정반대 예측

5.2 Fama-French의 도전

Fama and French (1992)는 결정적 증거를 제시했다. 1963-1990년 미국 주식 데이터에서 베타와 평균 수익률 사이에 유의미한 관계가 없었다. 대신 기업 규모(size)와 장부가/시가 비율(B/M)이 횡단면 수익률을 강력하게 설명했다.

CAPM → 멀티팩터로: 이러한 이상현상은 시장 베타 하나로는 부족하다는 것을 의미한다. Fama-French는 시장(MKT), 규모(SMB), 가치(HML) 3팩터 모델을 제안했다. 이것이 Round 5의 주제다. Ang은 이를 "팩터의 동물원(factor zoo)"이라 부르며, 어떤 팩터가 진정한 위험 프리미엄인지 구별하는 것이 핵심이라고 강조한다.

5.3 SML의 평탄화

실증적으로 SML은 CAPM이 예측하는 것보다 평탄하다. 고베타 주식은 CAPM 예측보다 낮은 수익률을, 저베타 주식은 CAPM 예측보다 높은 수익률을 보인다. 이것이 "베타 이상현상(beta anomaly)" 또는 "저베타 프리미엄"이다.

6. 파이썬 구현: 실제 주식 베타 추정

6.1 yfinance로 데이터 수집 및 베타 추정

import yfinance as yf
import numpy as np
import pandas as pd
import statsmodels.api as sm

# 한국/미국 주식 + 시장 지수
tickers = {
    '삼성전자': '005930.KS',
    'SK하이닉스': '000660.KS',
    'KOSPI': '^KS11',
    'AAPL': 'AAPL',
    'TSLA': 'TSLA',
    'JNJ': 'JNJ',
    'SPY': 'SPY',
}

# 5년 월간 데이터
prices = yf.download(
    list(tickers.values()), period='5y', interval='1mo'
)['Close']
prices.columns = list(tickers.keys())
returns = prices.pct_change().dropna()

# 무위험이자율 (월간 약 0.35%)
rf = 0.0035

# 한국 주식 베타 (KOSPI 기준)
print("=== 한국 주식 베타 (vs KOSPI) ===")
for stock in ['삼성전자', 'SK하이닉스']:
    excess_s = returns[stock] - rf
    excess_m = returns['KOSPI'] - rf
    X = sm.add_constant(excess_m)
    model = sm.OLS(excess_s, X).fit()
    print(f"  {stock:10}: β={model.params[1]:.3f}, α={model.params[0]:.4f}, R²={model.rsquared:.3f}")

# 미국 주식 베타 (SPY 기준)
print("\n=== 미국 주식 베타 (vs SPY) ===")
for stock in ['AAPL', 'TSLA', 'JNJ']:
    excess_s = returns[stock] - rf
    excess_m = returns['SPY'] - rf
    X = sm.add_constant(excess_m)
    model = sm.OLS(excess_s, X).fit()
    print(f"  {stock:6}: β={model.params[1]:.3f}, α={model.params[0]:.4f}, R²={model.rsquared:.3f}")
Output (예시): === 한국 주식 베타 (vs KOSPI) === 삼성전자 : β=1.052, α=-0.0031, R²=0.512 SK하이닉스 : β=1.438, α=0.0085, R²=0.421 === 미국 주식 베타 (vs SPY) === AAPL : β=1.198, α=0.0042, R²=0.623 TSLA : β=1.854, α=0.0128, R²=0.312 JNJ : β=0.621, α=0.0011, R²=0.385

6.2 롤링 베타 (Rolling Beta)

베타는 시간에 따라 변한다. 24개월 롤링 윈도우로 베타의 시간 변화를 추적할 수 있다.

import pandas as pd
import numpy as np

def rolling_beta(stock_ret, market_ret, window=24):
    """롤링 윈도우 베타 계산"""
    betas = []
    dates = []
    for i in range(window, len(stock_ret)):
        s = stock_ret.iloc[i-window:i]
        m = market_ret.iloc[i-window:i]
        cov = np.cov(s, m)[0, 1]
        var = np.var(m, ddof=1)
        betas.append(cov / var)
        dates.append(stock_ret.index[i])
    return pd.Series(betas, index=dates)

# AAPL과 TSLA의 롤링 베타
beta_aapl = rolling_beta(returns['AAPL']-rf, returns['SPY']-rf)
beta_tsla = rolling_beta(returns['TSLA']-rf, returns['SPY']-rf)

print("AAPL 롤링 베타 (최근 5개월):")
print(beta_aapl.tail().round(3))
print(f"\nTSLA 롤링 베타 범위: [{beta_tsla.min():.2f}, {beta_tsla.max():.2f}]")

실전 인사이트: TSLA의 롤링 베타는 1.1~2.5 범위에서 크게 변동한다. 이는 테슬라의 시장 민감도가 시기에 따라 극적으로 달라짐을 의미한다. 반면 JNJ(존슨앤존슨)는 0.3~0.9 범위에서 안정적이다. 포트폴리오 구성 시 베타의 시간 변동성을 고려해야 한다.
