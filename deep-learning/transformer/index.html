1. Transformer의 탄생 배경

1.1 TPU가 RNN을 지원하지 않았다

2017년 Transformer가 등장한 배경에는 흥미로운 이야기가 있다. Google은 2016년 TPU(Tensor Processing Unit)를 개발했는데, TPU는 RNN 아키텍처를 효율적으로 지원하지 못했다. 당시 NLP 연구자들은 RNN/LSTM에 의존하고 있었기 때문에, TPU를 활용하려면 새로운 아키텍처가 필요했다.

Transformer의 진짜 탄생 이유:

"Google wanted to change the usage of GPU because GPU is too expensive. So they built the TPU in 2016. But the limitation of TPU is that TPU cannot support RNN architecture. So NLP researchers in Google decided to make their own architecture that can be supported by TPU. And that is the Transformer."

— 자원의 제약이 기술 혁신을 이끈 사례다.

1.2 RNN의 한계

RNN은 순차적으로 처리해야 하므로 병렬화가 불가능하다. 또한 긴 시퀀스에서 vanishing gradient 문제가 발생하고, long-range dependency를 학습하기 어렵다. LSTM이 이를 완화했지만 완전히 해결하지는 못했다.

Transformer의 핵심 아이디어는 간단하다: 출력 토큰을 모든 입력 토큰에 직접 연결하자. RNN처럼 순차적으로 정보를 전달하는 대신, 모든 위치를 한 번에 참조할 수 있게 하자.

2. Self-Attention — 핵심 메커니즘

2.1 Query, Key, Value

Self-Attention의 핵심 개념은 Query(Q), Key(K), Value(V)다. 도서관 비유로 이해하면:

  • Query: 내가 찾고 싶은 것 (검색어)
  • Key: 각 책의 제목/태그 (식별자)
  • Value: 책의 실제 내용 (정보)

입력 임베딩 \(X\)에서 Q, K, V를 생성한다:

\[ Q = XW_Q, \quad K = XW_K, \quad V = XW_V \]

2.2 Scaled Dot-Product Attention

Definition — Scaled Dot-Product Attention

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V \]

단계별 분해:

  1. \(QK^T\): Query와 Key의 dot product로 유사도 계산. \((i,j)\) 원소는 Query \(i\)와 Key \(j\)의 관련성.
  2. \(\div \sqrt{d_k}\): 스케일링. \(d_k\)가 크면 dot product의 분산이 커져서 softmax가 극단적이 됨.
  3. Softmax: 각 Query에 대해 모든 Key의 중요도를 확률 분포로 변환.
  4. \(\times V\): Attention weight로 Value를 가중 평균.

2.3 왜 \(\sqrt{d_k}\)로 나누는가?

수학적 이유:

\(q, k\)가 평균 0, 분산 1인 독립 성분이면:

\[ \text{Var}(q \cdot k) = \text{Var}\left(\sum_{i=1}^{d_k} q_i k_i\right) = d_k \]

분산이 \(d_k\)에 비례하므로, \(\sqrt{d_k}\)로 나눠서 분산을 1로 정규화한다. 이렇게 하지 않으면 softmax 입력이 너무 커져서 gradient가 매우 작아진다 (softmax saturation).

import numpy as np

def scaled_dot_product_attention(Q, K, V):
    """Scaled Dot-Product Attention 구현"""
    d_k = Q.shape[-1]
    
    # Step 1: QK^T (유사도)
    scores = Q @ K.T
    
    # Step 2: Scale by sqrt(d_k)
    scores = scores / np.sqrt(d_k)
    
    # Step 3: Softmax
    attention_weights = np.exp(scores) / np.sum(np.exp(scores), axis=-1, keepdims=True)
    
    # Step 4: Weighted sum of values
    output = attention_weights @ V
    
    return output, attention_weights

3. Multi-Head Attention

여러 개의 attention을 병렬로 수행하고 결과를 concat한다. 각 head가 다른 관점에서 정보를 추출한다.

\[ \text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h) W^O \]

왜 Multi-Head인가?

  • CNN의 multiple filters와 유사한 아이디어
  • 각 head가 다른 sub-task를 학습 (문법, 의미, 위치 관계 등)
  • 실험적으로 각 head가 다른 패턴을 학습한다는 것이 관찰됨

4. Positional Encoding

Self-Attention은 순서 정보를 모른다. "I love you"와 "You love I"가 같은 결과를 낸다. 위치 정보를 주입해야 한다.

Definition — Sinusoidal Positional Encoding

\[ PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]
\[ PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]

핵심 포인트: "Without positional encoding, Transformer cannot handle sequential data. Only with positional encoding, Transformer can know the position information."

5. Encoder-Decoder 구조

5.1 Encoder Block

Encoder는 Multi-Head Self-Attention → Add & Norm → Feed-Forward → Add & Norm 구조다. 입력 시퀀스의 모든 토큰을 양방향으로 참조할 수 있다.

5.2 Decoder Block

Decoder에는 두 개의 attention이 있다:

  1. Masked Self-Attention (Causal): 미래 토큰을 볼 수 없도록 마스킹
  2. Cross-Attention: Q는 Decoder에서, K/V는 Encoder 출력에서

Self-Attention같은 시퀀스같은 시퀀스Encoder
Causal/Masked같은 시퀀스같은 시퀀스 (미래 마스킹)Decoder 1st

5.3 Layer Normalization

Transformer는 Batch Normalization 대신 Layer Normalization을 사용한다.

왜 Layer Norm인가?

"The only reason to use Layer Normalization in Transformer is that the original paper was tuning the network based on Layer Normalization. If you want to use Batch Normalization, you need to tune hyperparameters again."

6.2 Label Smoothing

One-hot label 대신 soft label 사용. 모델의 과신(overconfidence)을 방지한다.

\[ y_{\text{smooth}} = (1 - \epsilon) \cdot y_{\text{one-hot}} + \frac{\epsilon}{K} \]

Label Smoothing 효과:

  • ε=0.1이면 정답 클래스는 0.9, 나머지는 약 0.02
  • 모델이 100% 확신하지 않도록 유도 → 일반화 성능 향상
  • BLEU score 향상 (실험적으로 검증됨)
