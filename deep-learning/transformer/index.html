자연어 처리(NLP)는 텍스트 데이터를 다루는 머신러닝 분야다. 주요 태스크는 다음과 같다:

1.2 평가 지표

Definition 1.1 — BLEU Score (Bilingual Evaluation Understudy)

기계 번역 품질 평가. Precision 기반 — 생성된 텍스트가 참조 텍스트와 얼마나 일치하는가.

\[ \text{BLEU} = \text{BP} \cdot \exp\left(\sum_{n=1}^{N} w_n \log p_n\right) \]

여기서 \(p_n\)은 n-gram precision, BP는 brevity penalty (짧은 번역에 페널티).

모델이 다음 단어에 높은 확률을 부여할수록 PPL이 낮아진다. 낮을수록 좋다.

강의 포인트: BLEU, ROUGE는 ground truth가 필요해서 비용이 많이 든다. Perplexity는 ground truth 없이 계산 가능하므로 실용적이다.

2. Tokenization

2.1 토큰화란?

토큰화(Tokenization)는 텍스트를 모델이 처리할 수 있는 단위(토큰)로 분할하는 과정이다. 예를 들어 "A cute teddy bear is reading"을 ["A", "cute", "teddy", "bear", "is", "reading"]으로 나눈다.

2.2 토큰화 방법 비교

Table 3. 토큰화 방법 비교
방법설명장점단점
Word-level공백/구두점 기준 분할구현 간단, 직관적OOV 문제, "bears"≠"bear"
Subword-levelBPE, WordPiece 등OOV 감소, 접두사/접미사 공유여전히 OOV 가능
Character-level글자 단위 분할OOV 없음, 오타에 강함시퀀스 길어짐, 해석 어려움

Definition 2.1 — OOV (Out-of-Vocabulary) 문제

학습 시 보지 못한 단어가 추론 시 등장하는 문제. Word-level 토큰화에서 심각하다.

예: 학습 데이터에 "reading"만 있고 "reads"가 없으면, "reads"는 [UNK] 토큰으로 처리된다.

강의 포인트:

  • Word-level: "bears"와 "bear"가 완전히 다른 토큰 — 의미적 유사성 무시
  • Subword-level: "reading" = "read" + "ing" — 어근 공유로 효율적
  • Character-level: "World Health Organization" = 24개 토큰 — 계산 비용 증가
# 토큰화 예시
sentence = "A cute teddy bear is reading"

# Word-level
word_tokens = sentence.split()
print(f"Word-level: {word_tokens}")
# ['A', 'cute', 'teddy', 'bear', 'is', 'reading']

# Character-level
char_tokens = list(sentence.replace(" ", ""))
print(f"Char-level: {char_tokens}")
# ['A', 'c', 'u', 't', 'e', 't', 'e', 'd', 'd', 'y', ...]

# Subword (BPE 스타일 예시)
subword_tokens = ["A", "cute", "ted", "##dy", "bear", "is", "read", "##ing"]
print(f"Subword: {subword_tokens}")

실무 선택: 현대 LLM은 대부분 Subword 토큰화(BPE, SentencePiece)를 사용한다. OOV 문제를 줄이면서도 시퀀스 길이를 적절히 유지한다.

3. Word2Vec — 단어의 분산 표현

3.1 왜 Word Embedding이 필요한가?

One-hot encoding의 문제: 모든 단어가 서로 직교(orthogonal)한다. "teddy bear"와 "soft"의 dot product = 0. 의미적 유사성을 전혀 반영하지 못한다.

Definition 3.1 — Word Embedding

단어를 저차원 밀집 벡터(dense vector)로 표현. 의미적으로 유사한 단어는 벡터 공간에서 가깝다.

\[ \text{similarity}(\text{teddy bear}, \text{soft}) = \cos(\vec{v}_{\text{teddy bear}}, \vec{v}_{\text{soft}}) \approx 1 \]

3.2 Word2Vec 학습 방법

Word2Vec(2013, Mikolov et al.)은 두 가지 proxy task로 학습한다:

Table 4. Word2Vec 학습 방법
방법입력출력설명
CBOW주변 단어들중심 단어"A", "cute", "is", "reading" → "teddy bear"
Skip-gram중심 단어주변 단어들"teddy bear" → "A", "cute", "is", "reading"

핵심 아이디어: Ground truth 없이 학습 가능! 텍스트 자체가 레이블이다. "다음 단어 예측" 또는 "주변 단어 예측"이라는 proxy task를 통해 의미 있는 표현을 학습한다.

3.3 Word2Vec 구조

# Word2Vec 구조 (Skip-gram)
# Input: one-hot vector (V차원, V=vocabulary size)
# Hidden: embedding vector (d차원, d=embedding dimension)
# Output: probability over vocabulary (V차원)

import numpy as np

V = 10000  # vocabulary size
d = 300    # embedding dimension

# 학습되는 파라미터
W_embed = np.random.randn(V, d)  # input → hidden
W_out = np.random.randn(d, V)    # hidden → output

# Forward pass (Skip-gram)
def forward(word_idx):
    # 1. Embedding lookup (one-hot @ W_embed = W_embed[word_idx])
    h = W_embed[word_idx]  # shape: (d,)
    
    # 2. Output projection
    logits = h @ W_out     # shape: (V,)
    
    # 3. Softmax
    probs = np.exp(logits) / np.sum(np.exp(logits))
    return probs

# Word2Vec의 결과물 = W_embed (embedding matrix)
# 학습 후 W_embed[word_idx]가 해당 단어의 embedding vector

Word2Vec의 마법: 학습 후 embedding space에서 의미적 관계가 보존된다.

  • \(\vec{v}_{\text{king}} - \vec{v}_{\text{man}} + \vec{v}_{\text{woman}} \approx \vec{v}_{\text{queen}}\)
  • 유사한 단어는 가까운 벡터: \(\cos(\vec{v}_{\text{cat}}, \vec{v}_{\text{dog}}) \approx 0.8\)

4. RNN & LSTM — 순차 데이터 처리

4.1 RNN의 필요성

Word2Vec의 한계: 단어의 위치/순서 정보를 반영하지 못한다. "I love you"와 "You love I"가 같은 표현이 된다. 순차 정보(sequential information)를 처리하려면 RNN이 필요하다.

Definition 4.1 — RNN (Recurrent Neural Network)

\[ h_t = \tanh(W_{hh} h_{t-1} + W_{xh} x_t + b_h) \]
\[ y_t = W_{hy} h_t + b_y \]

현재 hidden state \(h_t\)는 이전 hidden state \(h_{t-1}\)와 현재 입력 \(x_t\)의 함수. 가중치 \(W_{hh}, W_{xh}, W_{hy}\)는 모든 time step에서 공유된다.

4.2 RNN의 문제점: Vanishing/Exploding Gradient

긴 시퀀스에서 gradient가 소실되거나 폭발한다. 출력에서 입력까지 역전파할 때 hidden state를 여러 번 거치면서 gradient가 기하급수적으로 작아지거나 커진다.

Long-range dependency 문제: "The cat, which was sitting on the mat and looking at the bird through the window, was hungry." — "cat"과 "was hungry"의 관계를 학습하기 어렵다.

4.3 LSTM — Vanishing Gradient 해결

Definition 4.2 — LSTM (Long Short-Term Memory)

1997년 Hochreiter & Schmidhuber가 제안. 핵심: Cell state \(C_t\)라는 "고속도로(highway)"를 추가하여 gradient가 직접 흐를 수 있게 한다.

Table 5. LSTM의 4개 게이트
게이트역할수식
Forget gate \(f_t\)이전 정보 중 버릴 것 결정\(\sigma(W_f \cdot [h_{t-1}, x_t] + b_f)\)
Input gate \(i_t\)새 정보 중 저장할 것 결정\(\sigma(W_i \cdot [h_{t-1}, x_t] + b_i)\)
Cell update \(\tilde{C}_t\)새로운 후보 정보\(\tanh(W_C \cdot [h_{t-1}, x_t] + b_C)\)
Output gate \(o_t\)출력할 정보 결정\(\sigma(W_o \cdot [h_{t-1}, x_t] + b_o)\)
\[ C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t \]
\[ h_t = o_t \odot \tanh(C_t) \]

LSTM의 핵심: Cell state \(C_t\)는 \(C_{t-1}\)에서 직접 연결된다 (hidden state를 거치지 않음). 이 "고속도로" 덕분에 gradient가 멀리까지 전파될 수 있다.

4.4 Seq2Seq (Encoder-Decoder)

2014년 Sutskever et al. 기계 번역을 위한 구조:

  • Encoder: 입력 시퀀스를 하나의 context vector로 압축
  • Decoder: context vector에서 출력 시퀀스 생성

Seq2Seq의 한계 (Transformer의 동기):

  • 모든 입력 정보가 하나의 context vector에 압축 — 정보 손실
  • 긴 시퀀스에서 long-range dependency 문제 여전
  • 순차 처리 — 병렬화 불가능 (GPU 활용 어려움)

5. Self-Attention — Transformer의 핵심 ⭐

5.1 Attention의 목표

RNN의 long-range dependency 문제를 해결하고 싶다. 핵심 아이디어: 출력 토큰을 모든 입력 토큰에 직접 연결하자!

강의 인용: "The main goal of the attention is that we want to connect the long distance tokens... We want to connect the output token to all input tokens directly."

5.2 Query, Key, Value (Q, K, V)

Definition 5.1 — Query, Key, Value

  • Query (Q): "내가 찾고 싶은 것" — 현재 토큰의 질문
  • Key (K): "나를 찾을 때 사용할 키" — 각 토큰의 식별자
  • Value (V): "실제 정보" — 각 토큰이 가진 내용

비유: 도서관에서 책 찾기. Query = 검색어, Key = 책 제목/태그, Value = 책 내용

입력 임베딩 \(X\)에서 Q, K, V를 생성:

\[ Q = X W_Q, \quad K = X W_K, \quad V = X W_V \]

여기서 \(W_Q, W_K, W_V \in \mathbb{R}^{d_{model} \times d_k}\)는 학습되는 파라미터.

5.3 Scaled Dot-Product Attention ⭐⭐⭐

Definition 5.2 — Scaled Dot-Product Attention (가장 중요한 수식!)

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V \]

단계별 분해:

Step 1: \(QK^T\) — 유사도 계산
\[ QK^T \in \mathbb{R}^{n \times n} \]

\((i, j)\) 원소 = Query \(i\)와 Key \(j\)의 dot product = 유사도. Dot product가 클수록 두 토큰이 관련 있다.

Step 2: \(\frac{1}{\sqrt{d_k}}\) — 스케일링
\[ \frac{QK^T}{\sqrt{d_k}} \]

왜 \(\sqrt{d_k}\)로 나누는가? (퀴즈 단골 문제!)

\(d_k\)가 크면 dot product의 분산이 커진다. \(q, k\)가 평균 0, 분산 1인 독립 성분이면:

\[ \text{Var}(q \cdot k) = \text{Var}\left(\sum_{i=1}^{d_k} q_i k_i\right) = d_k \]

분산이 \(d_k\)에 비례하므로, \(\sqrt{d_k}\)로 나눠서 분산을 1로 정규화한다. 이렇게 하지 않으면 softmax 입력이 너무 커져서 gradient가 매우 작아진다 (softmax saturation).

Step 3: Softmax — 확률 분포로 변환
\[ \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)_{ij} = \frac{\exp(s_{ij})}{\sum_k \exp(s_{ik})} \]

각 Query에 대해 모든 Key의 중요도를 확률로 변환. 합이 1이 되는 attention weight.

Step 4: \(\times V\) — 가중 평균
\[ \text{Output} = \text{Attention Weights} \times V \]

Attention weight로 Value를 가중 평균. 중요한 토큰의 정보를 더 많이 반영.

import numpy as np

def scaled_dot_product_attention(Q, K, V):
    """
    Q: (n, d_k) - n개 query
    K: (n, d_k) - n개 key  
    V: (n, d_v) - n개 value
    """
    d_k = Q.shape[-1]
    
    # Step 1: QK^T (유사도)
    scores = Q @ K.T  # (n, n)
    
    # Step 2: Scale by sqrt(d_k)
    scores = scores / np.sqrt(d_k)
    
    # Step 3: Softmax (각 행이 확률 분포)
    attention_weights = np.exp(scores) / np.sum(np.exp(scores), axis=-1, keepdims=True)
    
    # Step 4: Weighted sum of values
    output = attention_weights @ V  # (n, d_v)
    
    return output, attention_weights

# 예시: "A cute teddy bear" (4 tokens)
n, d_k, d_v = 4, 64, 64
Q = np.random.randn(n, d_k)
K = np.random.randn(n, d_k)
V = np.random.randn(n, d_v)

output, weights = scaled_dot_product_attention(Q, K, V)
print(f"Output shape: {output.shape}")  # (4, 64)
print(f"Attention weights:\n{weights.round(3)}")

5.4 Multi-Head Attention

Definition 5.3 — Multi-Head Attention

여러 개의 attention을 병렬로 수행하고 결과를 concat. 각 head가 다른 관점에서 정보를 추출한다.

\[ \text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h) W^O \]
\[ \text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V) \]

왜 Multi-Head인가?

  • CNN의 multiple filters와 유사한 아이디어
  • 각 head가 다른 sub-task를 학습 (문법, 의미, 위치 관계 등)
  • 실험적으로 각 head가 다른 패턴을 학습한다는 것이 관찰됨
  • head 수를 늘리면 모델이 더 강력해짐
Table 6. Original Transformer 하이퍼파라미터
파라미터값설명
\(N\)6Encoder/Decoder layer 수
\(h\)8Attention head 수
\(d_{model}\)512Embedding dimension
\(d_k = d_v\)64\(d_{model} / h = 512/8\)
\(d_{ff}\)2048Feed-forward hidden dimension
import numpy as np

class MultiHeadAttention:
    def __init__(self, d_model=512, h=8):
        self.d_model = d_model
        self.h = h
        self.d_k = d_model // h  # 512/8 = 64
        
        # 각 head별 projection matrices
        self.W_Q = [np.random.randn(d_model, self.d_k) for _ in range(h)]
        self.W_K = [np.random.randn(d_model, self.d_k) for _ in range(h)]
        self.W_V = [np.random.randn(d_model, self.d_k) for _ in range(h)]
        self.W_O = np.random.randn(h * self.d_k, d_model)
    
    def forward(self, X):
        """X: (n, d_model)"""
        heads = []
        for i in range(self.h):
            Q = X @ self.W_Q[i]  # (n, d_k)
            K = X @ self.W_K[i]
            V = X @ self.W_V[i]
            head_i, _ = scaled_dot_product_attention(Q, K, V)
            heads.append(head_i)
        
        # Concat all heads: (n, h*d_k) = (n, 512)
        concat = np.concatenate(heads, axis=-1)
        
        # Final projection
        output = concat @ self.W_O  # (n, d_model)
        return output

5.5 Self-Attention vs Cross-Attention

Table 7. Attention 유형 비교
유형Q 출처K, V 출처사용 위치
Self-Attention같은 시퀀스같은 시퀀스Encoder, Decoder (첫 번째)
Cross-AttentionDecoderEncoder 출력Decoder (두 번째)
Causal/Masked같은 시퀀스같은 시퀀스 (미래 마스킹)Decoder (첫 번째)

강의 인용: "In the decoder block, there are two attention blocks. The first is masked multi-head attention (causal attention). The second is cross-attention — Q is from the decoder, K and V are from the encoder."

6. Transformer 아키텍처

6.1 Transformer의 탄생 배경

강의 인용 (Transformer의 진짜 탄생 이유!):

"Google wanted to change the usage of GPU because GPU is too expensive. So they built the TPU in 2016. But the limitation of TPU is that TPU cannot support RNN architecture. So NLP researchers in Google decided to make their own architecture that can be supported by TPU. And that is the Transformer."

→ TPU가 RNN을 지원하지 않아서 Transformer가 탄생했다! 자원의 제약이 기술 혁신을 이끈 사례.

6.2 Positional Encoding

Self-Attention은 순서 정보를 모른다. "I love you"와 "You love I"가 같은 결과를 낸다. 위치 정보를 주입해야 한다.

Definition 6.1 — Sinusoidal Positional Encoding (Original Transformer)

\[ PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]
\[ PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]

\(pos\)는 토큰 위치, \(i\)는 차원 인덱스. 짝수 차원은 sin, 홀수 차원은 cos.

왜 sin/cos인가?

  • 상대적 위치를 선형 변환으로 표현 가능: \(PE_{pos+k}\)를 \(PE_{pos}\)의 선형 함수로 표현
  • 학습 없이 사용 가능 (hard-coded)
  • 임의의 시퀀스 길이에 일반화 가능

대안: Learned positional encoding — 위치별 embedding을 학습. BERT 등에서 사용.

강의 포인트: "Without positional encoding, Transformer cannot handle sequential data. Only with positional encoding, Transformer can know the position information."

Vision Transformer처럼 순서가 덜 중요한 도메인에서는 positional encoding을 다르게 설계하거나 생략할 수 있다.

6.3 Encoder Block

# Encoder Block 구조
def encoder_block(x):
    # 1. Multi-Head Self-Attention
    attn_out = multi_head_attention(x, x, x)  # Q=K=V=x
    
    # 2. Residual Connection + Layer Norm
    x = layer_norm(x + attn_out)
    
    # 3. Feed-Forward Network
    ff_out = feed_forward(x)  # Linear → ReLU → Linear
    
    # 4. Residual Connection + Layer Norm
    x = layer_norm(x + ff_out)
    
    return x

# Feed-Forward Network
def feed_forward(x, d_model=512, d_ff=2048):
    # 512 → 2048 → 512
    h = relu(x @ W1 + b1)  # (n, 2048)
    out = h @ W2 + b2       # (n, 512)
    return out

Definition 6.2 — Layer Normalization

\[ \text{LayerNorm}(x) = \gamma \cdot \frac{x - \mu}{\sigma} + \beta \]

각 샘플 내에서 정규화 (feature dimension 기준). Batch size에 독립적.

6.5 Decoder Block

# Decoder Block 구조
def decoder_block(x, encoder_output):
    # 1. Masked Multi-Head Self-Attention (Causal)
    # 미래 토큰을 볼 수 없도록 마스킹
    masked_attn = masked_multi_head_attention(x, x, x)
    x = layer_norm(x + masked_attn)
    
    # 2. Cross-Attention
    # Q: decoder, K/V: encoder output
    cross_attn = multi_head_attention(
        Q=x, 
        K=encoder_output, 
        V=encoder_output
    )
    x = layer_norm(x + cross_attn)
    
    # 3. Feed-Forward Network
    ff_out = feed_forward(x)
    x = layer_norm(x + ff_out)
    
    return x

Decoder의 핵심:

  • 첫 번째 attention: Masked Self-Attention (Causal) — 미래 토큰 마스킹
  • 두 번째 attention: Cross-Attention — Encoder 출력 참조
  • Inference 시 autoregressive: 이전 출력이 다음 입력

7. Training Tricks

7.1 Teacher Forcing vs Autoregressive Inference

Definition 7.1 — Teacher Forcing

학습 시 ground truth를 다음 스텝의 입력으로 사용. 병렬 학습 가능!

Table 9. Teacher Forcing vs Autoregressive
특성Teacher Forcing (학습)Autoregressive (추론)
입력Ground truth 시퀀스이전 스텝의 예측
병렬화가능 (모든 위치 동시 계산)불가능 (순차 생성)
속도빠름느림
에러 전파없음있음 (에러 누적)

강의 인용: "During training, we already know the ground truth. So we use the ground truth sentence and train all positions simultaneously. We do not need to iterate. But during inference, we have to iterate because we need the previous output for the current input."

# Teacher Forcing (학습)
# 입력: [BOS, "un", "ours", "en", "peluche"]
# 타겟: ["un", "ours", "en", "peluche", EOS]
# → 모든 위치를 동시에 학습!

# Autoregressive Inference (추론)
output = [BOS]
while output[-1] != EOS:
    next_token = model.predict(output)  # 이전 출력이 입력
    output.append(next_token)
# → 한 토큰씩 순차 생성

7.2 Causal Masking

Teacher Forcing 시 미래 토큰을 보면 안 된다. Attention score에 마스크 적용:

\[ \text{Mask}_{ij} = \begin{cases} 0 & \text{if } j \leq i \\ -\infty & \text{if } j > i \end{cases} \]
# Causal Mask 예시 (4 tokens)
# 위치 i는 위치 j ≤ i만 볼 수 있음
mask = np.array([
    [0, -np.inf, -np.inf, -np.inf],  # 위치 0: 자기만
    [0, 0, -np.inf, -np.inf],        # 위치 1: 0, 1
    [0, 0, 0, -np.inf],              # 위치 2: 0, 1, 2
    [0, 0, 0, 0]                     # 위치 3: 모두
])

# Attention 계산 시
scores = (Q @ K.T) / np.sqrt(d_k) + mask  # -inf는 softmax 후 0이 됨

7.3 Label Smoothing

Definition 7.2 — Label Smoothing

One-hot label 대신 soft label 사용. 모델의 과신(overconfidence)을 방지.

\[ y_{\text{smooth}} = (1 - \epsilon) \cdot y_{\text{one-hot}} + \frac{\epsilon}{K} \]

여기서 \(\epsilon\)은 smoothing parameter (보통 0.1), \(K\)는 클래스 수.

# Label Smoothing 예시
# 원래 one-hot: [0, 0, 1, 0, 0] (정답 = 클래스 2)
# Smoothed (ε=0.1): [0.02, 0.02, 0.92, 0.02, 0.02]

def label_smoothing(one_hot, epsilon=0.1):
    K = len(one_hot)
    return (1 - epsilon) * one_hot + epsilon / K

one_hot = np.array([0, 0, 1, 0, 0])
smoothed = label_smoothing(one_hot, epsilon=0.1)
print(smoothed)  # [0.02, 0.02, 0.92, 0.02, 0.02]

Label Smoothing의 효과:

  • 모델이 100% 확신하지 않도록 유도 → 일반화 성능 향상
  • Overfitting 방지
  • BLEU score 향상 (실험적으로 검증됨)
  • GoogLeNet(2015)에서 처음 소개, Transformer에서 널리 사용

7.4 Residual Connection

ResNet에서 가져온 아이디어. Gradient flow를 개선하고 깊은 네트워크 학습을 가능하게 한다.

\[ \text{Output} = \text{LayerNorm}(x + \text{SubLayer}(x)) \]

강의 인용: "Many design choices are influenced by computer vision architectures. Residual connections from ResNet. Multi-head attention is similar to multiple filters in CNN."

Q1. Why do we divide by √d_k in the Self-Attention formula?

Self-Attention 수식에서 √d_k로 나누는 이유는?

A. To speed up computation
B. When d_k is large, the variance of dot product increases, causing softmax saturation
C. To reduce memory usage
D. To prevent gradient vanishing
Answer: B
If q, k have mean 0 and variance 1, then Var(q·k) = d_k. Large variance → extreme softmax inputs → very small gradients (softmax saturation). Dividing by √d_k normalizes variance to 1.
분산이 d_k에 비례 → softmax 입력이 극단적 → gradient 소실. √d_k로 나눠서 분산 정규화.

Self-Attention은 O(n²) 복잡도. 긴 시퀀스에서 병목.

Q30. What is the difference between Skip-gram and CBOW?

Skip-gram과 CBOW의 차이는?

A. Skip-gram is faster
B. Skip-gram: center→surrounding, CBOW: surrounding→center
C. CBOW is more accurate
D. No difference
Answer: B
Skip-gram: Predict surrounding words from center word. CBOW: Predict center word from surrounding words. Skip-gram tends to be better for rare words.
Skip-gram: 중심→주변. CBOW: 주변→중심. Skip-gram이 희귀 단어에 더 좋은 경향.

Q31. Why use sin/cos in Positional Encoding?

Positional Encoding에서 sin/cos를 사용하는 이유는?

A. Faster computation
B. Relative positions can be expressed as linear transformations
C. Save memory
D. Improve gradient flow
Answer: B
PE_{pos+k} can be expressed as a linear function of PE_{pos}. Makes it easier for model to learn relative position relationships. Also requires no training.
상대적 위치를 선형 변환으로 표현 가능. 학습 없이 사용 가능.

Q32. How many attention mechanisms are in a Decoder block?

Decoder block에 있는 attention 개수는?

A. 1
B. 2 (Masked Self-Attention + Cross-Attention)
C. 3
D. 0
Answer: B
Decoder: (1) Masked Multi-Head Self-Attention (Causal), (2) Cross-Attention (Q: decoder, K/V: encoder).
Decoder: (1) Masked Self-Attention, (2) Cross-Attention.

Q33. With Label Smoothing ε=0.1, what is the target value for the correct class?

Label Smoothing에서 ε=0.1일 때, 정답 클래스의 target 값은?

A. 1.0
B. 0.9
C. 0.1
D. 0.5
Answer: B
(1-ε) × 1 + ε/K ≈ 0.9 (when K is large). Correct class gets 0.9, others get ε/K ≈ 0.02.
정답 클래스는 0.9, 나머지는 ε/K ≈ 0.02.

Q34. What are the limitations of Seq2Seq?

Seq2Seq의 한계는?

A. All input compressed into one context vector, information loss
B. Cannot be parallelized
C. Long-range dependency problem
D. All of the above
Answer: D
Seq2Seq (RNN-based): Information bottleneck, sequential processing, long-range problems. Attention was introduced to address these, eventually leading to Transformer.
정보 병목, 순차 처리, long-range 문제. 이를 해결하기 위해 Attention → Transformer 등장.

Q35. What is "context-aware embedding" in Transformer?

Transformer에서 "context-aware embedding"이란?

A. Embedding with positional information
B. Embedding that incorporates surrounding token information after Self-Attention
C. Word2Vec embedding
D. One-hot encoding
Answer: B
Input embedding is position-aware. After Self-Attention, information from other tokens is incorporated, making it context-aware.
입력 embedding은 position-aware. Self-Attention 후에는 다른 토큰 정보가 반영되어 context-aware.

Q36. What is the disadvantage of Character-level tokenization?

Character-level 토큰화의 단점은?

A. OOV problem
B. Longer sequences increase computation cost, harder to interpret meaning
C. Difficult to implement
D. Cannot be trained
Answer: B
"World Health Organization" = 24 tokens. Longer sequence → O(n²) attention cost increases. Also, single character "D" is hard to interpret semantically.
시퀀스 길어짐 → O(n²) 비용 증가. "D" 하나로는 의미 해석 어려움.

Q37. What does each head learn in Multi-Head Attention?

Multi-Head Attention에서 각 head가 학습하는 것은?

A. Same patterns
B. Different sub-tasks (syntax, semantics, positional relations, etc.)
C. Random patterns
D. Noise
Answer: B
Experimentally observed that each head learns different patterns. Unintentional but natural division of labor.
실험적으로 각 head가 다른 패턴을 학습하는 것이 관찰됨. 자연스럽게 역할 분담.

Q38. Why is Transformer also used in Vision?

Transformer가 Vision에서도 사용되는 이유는?

A. Always better than CNN
B. Can be applied to various domains by changing Positional Encoding
C. Faster computation
D. Uses less memory
Answer: B
Lecture quote: "If your data is not sequential, you can replace positional encoding with other components. That is the strength of Transformer."
Positional Encoding을 바꾸면 다양한 도메인에 적용 가능. Transformer의 강점.

Q39. What is the sum of attention weights?

Attention weight의 합은?

A. 0
B. 1 (for each Query)
C. d_k
D. n (sequence length)
Answer: B
After softmax, each row (Query)'s attention weights sum to 1. It's a probability distribution.
Softmax 적용 후 각 행(Query)의 attention weight 합 = 1. 확률 분포.

Q40. What is the key trend in NLP since 2017?

2017년 이후 NLP의 핵심 트렌드는?

A. Developing new architectures
B. Scaling Transformer-based models (bigger = better)
C. Return to RNN
D. CNN-based NLP
Answer: B
BERT, GPT, LLaMA are all Transformer-based. "No new architecture innovation, just scaling for better performance" — Scaling Laws.
BERT, GPT, LLaMA 모두 Transformer 기반. 새로운 아키텍처 혁신 없이 스케일링만으로 성능 향상.
