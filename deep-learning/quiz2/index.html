NLP 기초 & Embedding (Q1–5)

NLP 기초 & Embedding (Q1–5)

Question 1 Embedding

Which property best explains why one-hot vectors fail to capture semantic similarity?

  1. High dimensionality
  2. Orthogonality between vectors
  3. Sparsity
  4. Fixed vocabulary
정답: ② Orthogonality between vectors
One-hot 벡터는 모든 단어 쌍의 dot product = 0 (직교). 따라서 cos similarity도 0이 되어 의미적 유사성을 전혀 반영 못 함. High dimensionality나 sparsity도 문제지만, semantic similarity 실패의 직접적 원인은 orthogonality.
Question 2 Word2Vec

Which limitation of Word2Vec is most critical for sentence-level understanding?

  1. High memory usage
  2. Lack of contextual awareness
  3. Slow training
  4. Poor generalization
정답: ② Lack of contextual awareness
Word2Vec은 단어마다 하나의 고정 벡터를 부여. "bank"가 은행인지 강둑인지 문맥에 따라 달라져야 하는데, 항상 같은 벡터. 이것이 BERT/GPT 같은 contextual embedding의 등장 동기.
Question 3 Word2Vec

Which statement is TRUE about Word2Vec?

  1. CBOW uses future context only
  2. Skip-gram predicts context from center word
  3. CBOW captures word order
  4. Skip-gram requires labeled data
정답: ② Skip-gram predicts context from center word
Skip-gram: 중심 단어 → 주변 단어 예측. CBOW: 주변 단어 → 중심 단어 예측. 둘 다 self-supervised (labeled data 불필요). CBOW는 bag-of-words라 word order 무시.
Question 4 Tokenization

Why does subword tokenization outperform word-level tokenization?

  1. Removes need for embeddings
  2. Captures morphology and reduces OOV
  3. Always produces shorter sequences
  4. Requires smaller vocabulary
정답: ② Captures morphology and reduces OOV
Subword (BPE, WordPiece): "reading" = "read" + "ing". 어근을 공유하므로 OOV 감소 + 형태소 정보 활용. 시퀀스가 항상 짧아지는 건 아님 (character보다는 짧지만 word보다는 길 수 있음).
Question 5 Tokenization

Which trade-off is MOST accurate for character-level tokenization?

  1. High interpretability / low robustness
  2. Low OOV / higher computational cost
  3. Low dimensionality / high sparsity
  4. High speed / low flexibility
정답: ② Low OOV / higher computational cost
Character-level: 알파벳 26자 + 특수문자만 있으면 되니 OOV 거의 없음. 하지만 "World Health Organization" = 24+ 토큰 → 시퀀스 길어져서 계산 비용 증가.

RNN & LSTM (Q6–8)

Question 6 RNN

Why do RNNs struggle with long sequences?

  1. Attention bottleneck
  2. Gradient instability (vanishing/exploding)
  3. Tokenization error
  4. Memory overflow
정답: ② Gradient instability (vanishing/exploding)
RNN은 역전파 시 hidden state를 여러 번 거치면서 gradient가 기하급수적으로 작아지거나(vanishing) 커짐(exploding). 이것이 long-range dependency 학습을 어렵게 만드는 핵심 원인.
Question 7 LSTM

What is the key innovation of LSTM over vanilla RNN?

  1. Parallel computation
  2. Gated control of information flow
  3. Larger hidden dimension
  4. Removal of recurrence
정답: ② Gated control of information flow
LSTM의 핵심 = forget gate, input gate, output gate로 정보 흐름을 제어. Cell state라는 "고속도로"를 통해 gradient가 직접 흐를 수 있게 함. 병렬 처리는 여전히 불가 (그건 Transformer의 장점).
Question 8 RNN

Which scenario best demonstrates RNN limitation?

  1. Short sentence classification
  2. Predicting next word in long paragraph
  3. Tokenization
  4. Word embedding
정답: ② Predicting next word in long paragraph
긴 문단에서 다음 단어 예측 = long-range dependency가 필요한 전형적 태스크. RNN은 vanishing gradient 때문에 먼 과거 정보를 잃어버림. 짧은 문장 분류는 RNN도 잘 함.

Attention Mechanism (Q9–14)

Question 9 Attention

Why was attention introduced in seq2seq models?

  1. Reduce training cost
  2. Eliminate embeddings
  3. Overcome fixed-length context bottleneck
  4. Improve tokenization
정답: ③ Overcome fixed-length context bottleneck
Seq2Seq의 문제: 전체 입력을 하나의 고정 길이 context vector로 압축 → 정보 손실. Attention은 디코더가 인코더의 모든 hidden state에 직접 접근하게 해서 이 bottleneck을 해결.
Question 10 Attention

What does attention compute fundamentally?

  1. Convolution
  2. Weighted average of values
  3. Matrix factorization
  4. Token classification
정답: ② Weighted average of values
\(\text{Attention}(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V\). softmax가 weight를 만들고, 그 weight로 V의 가중 평균을 구함. 핵심은 "어떤 value에 얼마나 주목할지"를 결정하는 것.
Question 11 Q/K/V

Which is TRUE about Query-Key interaction?

  1. Key selects Query
  2. Query compares against Keys
  3. Values determine similarity
  4. Query stores output
정답: ② Query compares against Keys
Q·K^T = Query가 모든 Key와 비교하여 유사도(similarity score)를 계산. Query = "내가 찾고 싶은 것", Key = "나를 찾을 때 쓸 식별자". Value는 유사도 계산에 관여하지 않고, 최종 출력에만 사용됨.
Question 12 Attention

Why is dot product used in attention?

  1. Minimizes memory
  2. Efficient similarity computation
  3. Ensures normalization
  4. Reduces sequence length
정답: ② Efficient similarity computation
Dot product = 두 벡터의 유사도를 O(d)로 빠르게 계산. 행렬 곱으로 배치 처리 가능 → GPU 친화적. Normalization은 softmax + scaling이 담당.
Question 13 Attention

What happens if attention weights are uniform?

  1. Model fails
  2. Output becomes average of all values
  3. Gradients explode
  4. No embeddings
정답: ② Output becomes average of all values
Uniform weights = 모든 토큰에 동일한 가중치 (1/n). 그러면 output = V의 단순 평균. 특정 토큰에 "주목"하는 능력을 잃어버림. 모델이 완전히 실패하진 않지만 성능이 크게 떨어짐.
Question 14 Self-Attention

Which is TRUE about self-attention?

  1. Only attends to previous tokens
  2. Requires recurrence
  3. Each token attends to all tokens
  4. Uses convolution
정답: ③ Each token attends to all tokens
Self-attention에서 Q, K, V 모두 같은 시퀀스에서 옴. 각 토큰이 시퀀스의 모든 토큰과 attention을 계산. ①은 causal/masked attention (decoder)의 특성. Self-attention 자체는 양방향.

Transformer 구조 (Q15–20)

Question 15 Transformer

Which component is NOT in Transformer encoder?

  1. Self-attention
  2. FFN
  3. Positional encoding
  4. Recurrence
정답: ④ Recurrence
Transformer의 핵심 = "Attention Is All You Need" → RNN의 recurrence를 완전히 제거. Encoder는 Self-Attention + FFN + Residual + LayerNorm으로 구성. Recurrence 없이 병렬 처리 가능.
Question 16 Transformer

Why is Transformer more parallelizable than RNN?

  1. No embeddings
  2. No sequential dependency in computation
  3. No gradients
  4. Smaller model
정답: ② No sequential dependency in computation
RNN: h_t는 h_{t-1}에 의존 → 순차 처리 필수. Transformer: self-attention은 모든 토큰을 동시에 처리 (행렬 곱). 이것이 GPU/TPU 활용의 핵심 이점.
Question 17 FFN

What is the main role of FFN layer?

  1. Token interaction
  2. Position encoding
  3. Non-linear transformation per token
  4. Reduce sequence length
정답: ③ Non-linear transformation per token
FFN = position-wise feed-forward network. 각 토큰에 독립적으로 적용 (토큰 간 상호작용 없음 — 그건 attention의 역할). FFN(x) = ReLU(xW₁+b₁)W₂+b₂. 비선형 변환으로 표현력 증가.
Question 18 Multi-Head

Why is multi-head attention useful?

  1. Reduces parameters
  2. Captures diverse relationships
  3. Removes positional encoding
  4. Reduces vocabulary
정답: ② Captures diverse relationships
각 head가 다른 subspace에서 attention을 계산 → 문법적 관계, 의미적 관계, 위치 관계 등 다양한 패턴을 동시에 포착. CNN의 multiple filters와 유사한 아이디어.
Question 19 Transformer

If we remove attention but keep FFN, what happens?

  1. Model still captures dependencies
  2. Model cannot model interactions between tokens
  3. Model becomes RNN
  4. No effect
정답: ② Model cannot model interactions between tokens
FFN은 각 토큰에 독립적으로 적용 (position-wise). 토큰 간 상호작용은 오직 attention이 담당. Attention 없으면 각 토큰이 다른 토큰의 정보를 전혀 참조 못 함.
Question 20 Output

Which is TRUE about Transformer output layer?

  1. Regression task
  2. Predicts token probabilities over vocabulary
  3. Removes embeddings
  4. Uses convolution
정답: ② Predicts token probabilities over vocabulary
최종 linear layer + softmax → vocabulary 크기의 확률 분포 출력. 각 위치에서 다음 토큰(또는 마스킹된 토큰)의 확률을 예측.

Position Encoding (Q21–27)

Question 21 PE

Without positional encoding, Transformer becomes:

  1. Sequential model
  2. Order-aware
  3. Permutation-equivariant
  4. Recurrent
정답: ③ Permutation-equivariant
PE 없으면 입력 토큰 순서를 바꿔도 출력이 같은 순서로 바뀜 (permutation-equivariant). "I love you" = "You love I". Self-attention 자체는 순서 정보가 없기 때문.
Question 22 Learned PE

Why do learned positional embeddings fail on longer sequences?

  1. Overfitting
  2. Fixed embedding table size
  3. Gradient vanishing
  4. Token mismatch
정답: ② Fixed embedding table size
Learned PE = 위치별로 학습된 embedding 벡터 테이블. 학습 시 최대 길이(예: 512)까지만 테이블이 있으므로, 그보다 긴 시퀀스에는 대응하는 embedding이 없음. 외삽(extrapolation) 불가.
Question 23 Sinusoidal

Key idea of sinusoidal encoding:

  1. Random noise
  2. Encode positions using periodic functions
  3. Learn embeddings
  4. Reduce parameters
정답: ② Encode positions using periodic functions
\(PE_{(pos,2i)} = \sin(pos/10000^{2i/d})\), \(PE_{(pos,2i+1)} = \cos(pos/10000^{2i/d})\). 주기 함수로 위치를 인코딩 → 학습 파라미터 없이 사용 가능 (hard-coded).
Question 24 Sinusoidal

Why does sinusoidal encoding generalize?

  1. Uses learned weights
  2. Mathematical structure allows extrapolation
  3. Smaller embedding
  4. Uses attention
정답: ② Mathematical structure allows extrapolation
Sin/cos의 수학적 성질: PE_{pos+k}를 PE_{pos}의 선형 변환으로 표현 가능. 이 구조 덕분에 학습 시 보지 못한 더 긴 위치에도 자연스럽게 확장(extrapolation) 가능.
Question 25 Relative PE

Why is relative position more important than absolute?

  1. Reduces memory
  2. Attention depends on relative distances
  3. Easier to train
  4. Removes embeddings
정답: ② Attention depends on relative distances
"The cat sat" — "cat"과 "sat"의 관계는 절대 위치(2번, 3번)보다 상대 거리(1칸 차이)가 중요. Attention score는 본질적으로 토큰 간 상대적 거리에 의존. ALiBi, RoPE 모두 상대 위치 인코딩.
Question 26 ALiBi

What does ALiBi do?

  1. Adds positional embeddings
  2. Adds bias to attention scores
  3. Rotates embeddings
  4. Removes position info
정답: ② Adds bias to attention scores
ALiBi = Attention with Linear Biases. Attention score에 거리 비례 페널티를 더함 (additive bias): score -= m·|i-j|. 학습 파라미터 없음. 멀리 있는 토큰일수록 attention score가 낮아짐. 키워드: bias, additive, penalty.
Question 27 RoPE

What is RoPE doing mathematically?

  1. Scaling embeddings
  2. Rotating Q/K vectors
  3. Masking tokens
  4. Adding bias
정답: ② Rotating Q/K vectors
RoPE = Rotary Position Embedding. Q, K 벡터를 위치에 따라 회전 (multiplicative). 회전 행렬 R(θ)을 곱함. 내적 q·k가 자동으로 상대 위치에 의존하게 됨. 키워드: rotate, multiply, Q/K 변환.

ALiBi vs RoPE 구분:
• ALiBi = 덧셈 (attention score에 bias 더함)
• RoPE = 곱셈 (Q/K에 회전 행렬 곱함)

Normalization & 종합 (Q28–30)

Question 28 LayerNorm

Why is Pre-Norm more stable than Post-Norm?

  1. Reduces parameters
  2. Improves gradient flow in deep networks
  3. Removes residuals
  4. Eliminates attention
정답: ② Improves gradient flow in deep networks
Pre-Norm: LayerNorm을 sublayer 앞에 배치 → residual path가 깨끗하게 유지됨 → gradient가 직접 흐를 수 있음. Post-Norm은 sublayer 뒤에 norm → 깊은 네트워크에서 gradient 흐름이 불안정. GPT 계열은 대부분 Pre-Norm 사용.
Question 29 종합 ⭐ 고난도

Consider a Transformer without positional encoding trained on sequence data. Which of the following tasks would be least affected by removing positional encoding?

  1. Machine translation
  2. Sentiment classification of a sentence
  3. Next-word prediction
  4. Part-of-speech tagging
정답: ② Sentiment classification of a sentence
Sentiment classification = 문장 전체의 감정 판단. 단어 순서보다 어떤 단어가 있는지(bag-of-words적 특성)가 더 중요. "I love this movie" vs "this movie I love" → 둘 다 긍정. 반면 번역, 다음 단어 예측, POS 태깅은 순서가 핵심.
Question 30 종합 ⭐ 고난도

Suppose we modify self-attention by replacing the dot product QK^T with a constant matrix (i.e., all similarities are equal). What is the most likely consequence?

  1. Model becomes equivalent to an RNN
  2. Attention layer reduces to averaging value vectors
  3. Model learns positional encoding automatically
  4. Model cannot be trained
정답: ② Attention layer reduces to averaging value vectors
QK^T가 상수 → softmax 후 모든 weight가 동일 (1/n) → output = V의 단순 평균. Q13과 연결되는 문제. Attention의 "선택적 주목" 능력이 사라지고, 모든 토큰 정보를 균등하게 섞는 것과 같아짐.
