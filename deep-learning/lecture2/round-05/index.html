1. BERT — Bidirectional Encoder Representations from Transformers

1.1 BERT가 해결한 문제

2018년 이전의 언어 모델은 대부분 단방향이었다. GPT-1은 왼쪽에서 오른쪽으로만 문맥을 읽었고, ELMo는 양방향이었지만 두 개의 단방향 LSTM을 단순히 결합한 것이었다. Devlin et al. (2018)은 "진정한 양방향" 사전학습이 가능한 BERT를 제안했다.

Definition 1.1 — BERT (Devlin et al., 2018)

BERT는 Transformer의 Encoder만 사용하는 모델로, Masked Language Modeling (MLM)과 Next Sentence Prediction (NSP)을 통해 양방향 문맥 표현을 학습한다.

\[ \text{BERT} = \text{Transformer Encoder} + \text{MLM} + \text{NSP} \]

1.2 BERT의 핵심 아이디어

왜 양방향이 중요한가?

예문: "The bank of the river was steep."

  • 단방향 (GPT): "The bank of the" → "bank"의 의미를 왼쪽 문맥만으로 판단. 금융? 강둑?
  • 양방향 (BERT): "The __ of the river was steep" → 왼쪽("The")과 오른쪽("river", "steep") 모두 참조 → "bank" = 강둑

양방향 문맥이 단어의 의미를 정확히 파악하는 데 결정적이다.

2. BERT 아키텍처 상세

2.1 입력 표현 (Input Representation)

BERT의 입력은 세 가지 임베딩의 합으로 구성된다. 이것이 BERT를 다른 모델과 구분짓는 핵심 설계 중 하나다.

BERT Input Representation Input: [CLS] I love deep learning [SEP] It is fun [SEP] │ │ │ │ │ │ │ │ │ │ Token E_CLS E_I E_love ... E_learning E_SEP E_It ... E_SEP Embed + + + + + + + + + + Segment E_A E_A E_A E_A E_A E_A E_B E_B E_B E_B Embed + + + + + + + + + + Position E_0 E_1 E_2 E_3 E_4 E_5 E_6 E_7 E_8 E_9 Embed ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓ ┌──────────────────────────────────────────────────┐ │ Sum of 3 Embeddings │ └──────────────────────────────────────────────────┘

Definition 2.1 — BERT 입력 임베딩

\[ \mathbf{x}_i = \mathbf{E}_{\text{token}}(w_i) + \mathbf{E}_{\text{segment}}(s_i) + \mathbf{E}_{\text{position}}(i) \]
  • Token Embedding: WordPiece 토큰을 벡터로 변환. 30,522개 어휘
  • Segment Embedding: 문장 A/B 구분. \( E_A \) 또는 \( E_B \). NSP를 위해 필요
  • Position Embedding: Learned PE. 0~511까지 512개 위치 벡터 학습

2.2 특수 토큰

[CLS]와 [SEP]의 역할

  • [CLS] (Classification): 항상 입력의 첫 번째 토큰. 이 토큰의 최종 hidden state가 전체 시퀀스의 "요약" 표현으로 사용됨. 분류 태스크에서 이 벡터에 classifier를 붙임
  • [SEP] (Separator): 두 문장을 구분하는 토큰. 문장 A와 B 사이, 그리고 입력의 끝에 위치
  • [MASK]: MLM에서 마스킹된 토큰을 대체. Pre-training에서만 사용

2.3 Encoder 구조

BERT는 Transformer Encoder를 L개 쌓은 구조다. 각 레이어는 Multi-Head Self-Attention + Feed-Forward Network로 구성되며, Post-Norm (Add & Norm) 방식을 사용한다.

3. Pre-training: MLM + NSP

3.1 Masked Language Modeling (MLM)

MLM은 BERT의 핵심 학습 목표다. 입력 토큰의 15%를 무작위로 선택하여 마스킹하고, 모델이 양방향 문맥을 사용하여 원래 토큰을 예측하도록 학습한다.

Definition 3.1 — Masked Language Modeling

입력 토큰의 15%를 선택한 후, 다음 규칙으로 변환:

  • 80%: [MASK] 토큰으로 대체 → "I love [MASK] learning"
  • 10%: 랜덤 토큰으로 대체 → "I love pizza learning"
  • 10%: 원래 토큰 유지 → "I love deep learning"
\[ \mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(x_i | \mathbf{x}_{\backslash \mathcal{M}}) \]

여기서 \( \mathcal{M} \)은 마스킹된 위치의 집합, \( \mathbf{x}_{\backslash \mathcal{M}} \)은 마스킹되지 않은 토큰들.

왜 80/10/10인가? (Pre-training vs Fine-tuning Mismatch 해결)

  • 문제: Pre-training에서는 [MASK] 토큰이 있지만, fine-tuning에서는 [MASK]가 없다. 이 불일치가 성능을 저하시킬 수 있다
  • 80% [MASK]: 모델이 마스킹된 토큰을 예측하는 주요 학습 신호
  • 10% Random: 모델이 모든 위치에서 올바른 토큰인지 판단하도록 강제. "이 토큰이 맞는지" 항상 의심하게 만듦
  • 10% Unchanged: 실제 입력(fine-tuning 시)과의 분포 차이를 줄임. 마스킹되지 않은 토큰도 예측 대상이 될 수 있음을 학습
# MLM 예시 (PyTorch-style pseudocode)
import torch

# 입력: "The cat sat on the mat"
tokens = ["The", "cat", "sat", "on", "the", "mat"]

# 15% 선택 → "cat" 선택됨
mask_prob = 0.15
selected_idx = 1  # "cat"

# 80/10/10 규칙 적용
r = torch.rand(1).item()
if r < 0.8:
    tokens[1] = "[MASK]"     # 80%: "The [MASK] sat on the mat"
elif r < 0.9:
    tokens[1] = "dog"        # 10%: "The dog sat on the mat"
else:
    pass                      # 10%: "The cat sat on the mat" (유지)

# 모델은 위치 1에서 "cat"을 예측해야 함
# loss = CrossEntropy(model_output[1], vocab_id("cat"))

3.2 Next Sentence Prediction (NSP)

NSP는 두 문장 간의 관계를 이해하기 위한 학습 목표다. 모델에 문장 A와 B를 주고, B가 A의 실제 다음 문장인지(IsNext) 아닌지(NotNext)를 예측한다.

Definition 3.2 — Next Sentence Prediction

  • 50% IsNext: 실제 연속된 문장 쌍. "I went to the store. [SEP] I bought some milk."
  • 50% NotNext: 랜덤으로 선택된 문장. "I went to the store. [SEP] The sun is a star."
\[ \mathcal{L}_{\text{NSP}} = -\log P(\text{IsNext} | \mathbf{h}_{\text{[CLS]}}) \]

[CLS] 토큰의 최종 hidden state에 binary classifier를 붙여 예측.

NSP의 한계 (RoBERTa에서 제거됨):

  • NSP 태스크가 너무 쉬움: 다른 문서에서 가져온 문장은 주제 자체가 달라서 topic 차이만으로 구분 가능
  • 실제로 문장 간 "논리적 연결"을 학습하는 것이 아니라 "주제 유사성"만 학습
  • RoBERTa (2019)는 NSP를 제거하고도 더 좋은 성능을 달성 → NSP는 불필요했다

3.3 전체 Pre-training Loss

\[ \mathcal{L}_{\text{BERT}} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}} \]

Pre-training 세부 사항:

  • 데이터: BooksCorpus (800M words) + English Wikipedia (2,500M words)
  • 배치 크기: 256 sequences × 512 tokens = 131,072 tokens/batch
  • 학습 스텝: 1,000,000 steps (~40 epochs)
  • Optimizer: Adam (lr=1e-4, warmup 10,000 steps)
  • 학습 시간: 4일 (TPU v3 × 16 pods) — BERT-large 기준

4. Fine-tuning: BERT를 실전에 투입하기

4.1 Fine-tuning의 핵심 아이디어

BERT의 혁신은 "pre-train once, fine-tune for many tasks"에 있다. 거대한 코퍼스에서 언어를 이해하는 능력을 학습한 후, 소량의 라벨 데이터로 특정 태스크에 적응시킨다.

BERT Fine-tuning Pipeline Pre-trained BERT (110M params) │ │ 모든 파라미터를 가져옴 ▼ ┌─────────────────────────────────────┐ │ BERT Encoder (frozen or fine-tuned)│ │ + Task-specific Head (새로 추가) │ └─────────────────────────────────────┘ │ ┌────┼────┬────────┐ ▼ ▼ ▼ ▼ 분류 NER QA 유사도 학습: 소량 라벨 데이터 (수천~수만 개) 시간: 수 시간 (GPU 1개) 핵심: Pre-trained 가중치가 좋은 초기값 역할

4.2 태스크별 Fine-tuning 방법

4.2.1 문장 분류 (Sentence Classification)

감정 분석, 주제 분류 등

입력: [CLS] This movie is great [SEP]

방법: [CLS] 토큰의 hidden state \( \mathbf{h}_{\text{[CLS]}} \in \mathbb{R}^H \)에 linear classifier 추가

\[ P(y | \mathbf{x}) = \text{softmax}(\mathbf{W} \cdot \mathbf{h}_{\text{[CLS]}} + \mathbf{b}) \]

여기서 \( \mathbf{W} \in \mathbb{R}^{K \times H} \), K = 클래스 수

4.2.2 토큰 분류 (Token Classification / NER)

Named Entity Recognition, POS Tagging 등

입력: [CLS] John lives in New York [SEP]

방법: 각 토큰의 hidden state에 classifier 추가

\[ P(y_i | \mathbf{x}) = \text{softmax}(\mathbf{W} \cdot \mathbf{h}_i + \mathbf{b}), \quad \forall i \]

출력: John→B-PER, lives→O, in→O, New→B-LOC, York→I-LOC

4.2.3 문장 쌍 분류 (Sentence Pair Classification)

자연어 추론 (NLI), 의미 유사도 등

입력: [CLS] A man is eating [SEP] Someone is having lunch [SEP]

방법: [CLS] 토큰으로 두 문장의 관계 분류 (entailment / contradiction / neutral)

4.2.4 질의응답 (Question Answering)

SQuAD 스타일 Extractive QA

입력: [CLS] Where is Paris? [SEP] Paris is the capital of France. [SEP]

방법: 답의 시작 위치(start)와 끝 위치(end)를 예측

\[ P_{\text{start}}(i) = \text{softmax}(\mathbf{w}_s^T \cdot \mathbf{h}_i), \quad P_{\text{end}}(i) = \text{softmax}(\mathbf{w}_e^T \cdot \mathbf{h}_i) \]

답: "the capital of France" (start=5, end=8)

# BERT Fine-tuning for Classification (HuggingFace)
from transformers import BertForSequenceClassification, BertTokenizer, Trainer

# 1. Pre-trained BERT 로드
model = BertForSequenceClassification.from_pretrained(
    "bert-base-uncased",
    num_labels=2  # positive / negative
)
tokenizer = BertTokenizer.from_pretrained("bert-base-uncased")

# 2. 입력 토큰화
inputs = tokenizer("This movie is great!", return_tensors="pt")
# {'input_ids': [101, 2023, 3185, 2003, 2307, 999, 102],
#  'token_type_ids': [0, 0, 0, 0, 0, 0, 0],  ← Segment A
#  'attention_mask': [1, 1, 1, 1, 1, 1, 1]}

# 3. Fine-tuning (Trainer API)
trainer = Trainer(
    model=model,
    train_dataset=train_data,
    eval_dataset=val_data,
    # lr=2e-5, epochs=3, batch_size=32 (BERT 논문 권장)
)
trainer.train()

5. BERT 변형: DistilBERT & RoBERTa

5.1 DistilBERT — 작지만 강한 BERT

DistilBERT (Sanh et al., 2019)는 Knowledge Distillation을 사용하여 BERT의 크기를 40% 줄이면서 97%의 성능을 유지한 모델이다.

Definition 5.1 — Knowledge Distillation

큰 모델(Teacher)의 "지식"을 작은 모델(Student)에게 전달하는 기법. Teacher의 soft probability distribution을 Student가 모방하도록 학습한다.

\[ \mathcal{L}_{\text{distil}} = \alpha \cdot \mathcal{L}_{\text{CE}}(y, p_S) + \beta \cdot \mathcal{L}_{\text{KD}}(p_T^\tau, p_S^\tau) + \gamma \cdot \mathcal{L}_{\text{cos}}(h_T, h_S) \]
  • \( \mathcal{L}_{\text{CE}} \): Student의 예측과 실제 라벨 간 Cross-Entropy
  • \( \mathcal{L}_{\text{KD}} \): Teacher와 Student의 soft label 간 KL Divergence (temperature \( \tau \))
  • \( \mathcal{L}_{\text{cos}} \): Teacher와 Student의 hidden state 간 Cosine Similarity
Knowledge Distillation Teacher (BERT-base: 12 layers, 110M) │ │ Soft labels (probability distribution) │ p_T = softmax(logits / τ) ← temperature τ로 "부드럽게" │ ▼ Student (DistilBERT: 6 layers, 66M) │ │ Student도 같은 temperature로 예측 │ p_S = softmax(logits / τ) │ ▼ Loss = α·CE(y, p_S) + β·KL(p_T, p_S) + γ·cos(h_T, h_S) 핵심: Teacher의 "dark knowledge" (틀린 답에 대한 확률 분포)를 학습 예: "cat" 예측 시 Teacher가 P(cat)=0.8, P(dog)=0.15, P(car)=0.01 → "cat과 dog이 비슷하다"는 정보를 Student가 학습

DistilBERT의 핵심 설계 결정:

  • 레이어 수를 12→6으로 줄임 (2개 중 1개 제거). Hidden size는 유지 (768)
  • Token type embedding (Segment) 제거 → NSP도 제거
  • Teacher의 가중치로 Student 초기화 (매 2번째 레이어)
  • Triple loss: CE + KD + Cosine embedding loss

5.2 RoBERTa — Robustly Optimized BERT

RoBERTa (Liu et al., 2019)는 BERT의 학습 방법을 최적화하여 같은 아키텍처에서 훨씬 더 좋은 성능을 달성했다. "BERT was significantly undertrained"라는 것을 증명한 연구다.

Definition 5.2 — RoBERTa의 4가지 개선

  1. NSP 제거: NSP 태스크가 오히려 성능을 저하시킴을 발견. 제거 후 성능 향상
  2. Dynamic Masking: BERT는 데이터 전처리 시 마스킹을 고정 (static). RoBERTa는 매 epoch마다 다른 마스킹 패턴 적용
  3. 더 많은 데이터: 16GB → 160GB (10배). CC-News, OpenWebText, Stories 추가
  4. 더 큰 배치 & 더 긴 학습: 배치 크기 8K, 학습 스텝 500K. BERT보다 훨씬 오래 학습

RoBERTa의 교훈:

아키텍처를 바꾸지 않고도 학습 방법만 최적화하면 성능이 크게 향상될 수 있다. 이는 "BERT가 충분히 학습되지 않았다"는 것을 의미하며, 학습 레시피의 중요성을 보여준다.

📝 Quiz — BERT Deep Dive

Q1. BERT의 입력 표현은 어떤 임베딩들의 합인가?

A. Token + Position
B. Token + Segment + Position
C. Token + Segment + Rotary
D. Word2Vec + Position
Answer: B
BERT 입력 = Token Embedding (WordPiece) + Segment Embedding (문장 A/B 구분) + Position Embedding (Learned, 0~511). 세 가지의 element-wise sum.

Q2. MLM에서 선택된 토큰의 80/10/10 규칙 중 "10% random replacement"의 목적은?

A. 데이터 증강
B. 모델이 모든 위치에서 올바른 토큰인지 판단하도록 강제
C. 학습 속도 향상
D. 과적합 방지
Answer: B
10% random replacement는 모델이 "[MASK]가 아닌 토큰도 틀릴 수 있다"고 인식하게 만든다. 모든 위치에서 올바른 표현을 유지하도록 강제하여 pre-train/fine-tune 불일치를 줄인다.

Q3. [CLS] 토큰의 역할은?

A. 문장의 시작을 표시
B. 전체 시퀀스의 요약 표현으로 사용되며, 분류 태스크에서 classifier 입력
C. 마스킹된 토큰을 대체
D. 두 문장을 구분
Answer: B
[CLS]의 최종 hidden state는 전체 시퀀스의 aggregate representation. NSP에서는 IsNext/NotNext 분류에, fine-tuning에서는 문장 분류의 입력으로 사용된다.

Q4. RoBERTa가 BERT에서 제거한 것은?

A. MLM
B. Position Embedding
C. Next Sentence Prediction (NSP)
D. [CLS] 토큰
Answer: C
RoBERTa는 NSP가 오히려 성능을 저하시킨다는 것을 발견하고 제거. NSP 태스크가 너무 쉬워서 topic 차이만 학습하고 실제 문장 관계는 학습하지 못했기 때문.

Q5. DistilBERT의 Knowledge Distillation에서 "dark knowledge"란?

A. 학습되지 않은 파라미터
B. Teacher 모델의 soft probability distribution에 담긴 클래스 간 유사성 정보
C. 숨겨진 레이어의 가중치
D. 학습 데이터의 노이즈
Answer: B
Hard label "cat"은 정보가 적지만, Teacher의 soft label P(cat)=0.8, P(dog)=0.15, P(car)=0.01은 "cat과 dog이 비슷하다"는 풍부한 정보를 담고 있다. 이것이 dark knowledge.

Q6. BERT의 QA fine-tuning에서 답을 추출하는 방법은?

A. [CLS] 토큰으로 답을 생성
B. 각 토큰에 대해 start/end 확률을 계산하여 답 span을 추출
C. Decoder로 답을 autoregressive하게 생성
D. 가장 높은 attention score를 가진 토큰을 선택
Answer: B
Extractive QA: 각 토큰 위치에 대해 P_start(i)와 P_end(i)를 계산. argmax(P_start) ~ argmax(P_end) 범위의 토큰이 답. 생성이 아닌 "추출".

Q7. RoBERTa의 "Dynamic Masking"이란?

A. 마스킹 비율을 동적으로 변경
B. 매 epoch마다 다른 마스킹 패턴을 적용 (BERT는 고정)
C. 학습 중 마스킹 비율을 점진적으로 증가
D. 중요한 토큰만 선택적으로 마스킹
Answer: B
BERT: 전처리 시 마스킹 고정 (static) → 같은 입력은 항상 같은 마스킹. RoBERTa: 매 epoch마다 새로운 마스킹 패턴 → 더 다양한 학습 신호.

Q8. BERT가 GPT와 달리 양방향 학습이 가능한 이유는?

A. 더 많은 파라미터를 사용하기 때문
B. MLM이 마스킹된 토큰만 예측하므로 정보 누출 없이 양방향 문맥 사용 가능
C. Decoder를 사용하기 때문
D. Cross-attention을 사용하기 때문
Answer: B
GPT는 next token prediction → 미래 토큰을 보면 "답을 미리 보는 것" → causal mask 필수. BERT는 [MASK] 토큰을 예측 → 마스킹된 위치의 정보가 이미 숨겨져 있으므로 양방향 참조 가능.

Q9. DistilBERT에서 temperature (τ)의 역할은?

A. 학습률을 조절
B. softmax 출력을 "부드럽게" 만들어 클래스 간 관계 정보를 더 많이 전달
C. 모델 크기를 조절
D. 배치 크기를 조절
Answer: B
τ>1이면 softmax가 더 uniform해짐 → 작은 확률의 클래스도 의미 있는 값을 가짐 → Teacher의 "dark knowledge"가 더 잘 전달됨. τ=1이면 일반 softmax.

Q10. BERT의 Position Embedding 방식과 그 한계는?

A. Sinusoidal PE — 학습이 불안정
B. RoPE — 계산 비용이 높음
C. Learned PE — 최대 512 토큰까지만 처리 가능
D. ALiBi — 가까운 토큰만 참조 가능
Answer: C
BERT는 Learned PE 사용. 512개 위치 벡터만 학습하므로 L_max=512. 513번째 토큰부터는 처리 불가. 이것이 BERT의 고정 길이 한계. (Round 1에서 학습한 내용!)

📋 Cheat Sheet — BERT Deep Dive 한눈에

🔑 핵심 수식

1. 입력 표현:

\[ \mathbf{x}_i = \mathbf{E}_{\text{token}}(w_i) + \mathbf{E}_{\text{segment}}(s_i) + \mathbf{E}_{\text{position}}(i) \]

2. MLM Loss:

\[ \mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(x_i | \mathbf{x}_{\backslash \mathcal{M}}) \]

3. Fine-tuning (분류):

\[ P(y | \mathbf{x}) = \text{softmax}(\mathbf{W} \cdot \mathbf{h}_{\text{[CLS]}} + \mathbf{b}) \]

4. Distillation:

\[ \mathcal{L} = \alpha \cdot \text{CE} + \beta \cdot \text{KL}(p_T^\tau \| p_S^\tau) + \gamma \cdot \text{cos}(h_T, h_S) \]

🎯 시험 대비 핵심 포인트

  • BERT = Encoder-only, 양방향, MLM + NSP, Learned PE (L_max=512)
  • 입력 = Token + Segment + Position Embedding의 합
  • [CLS] = 시퀀스 요약 표현, [SEP] = 문장 구분, [MASK] = MLM용
  • MLM: 15% 마스킹 → 80% [MASK] + 10% random + 10% keep
  • 80/10/10 이유: pre-train/fine-tune 불일치 해결
  • NSP: 50% IsNext + 50% NotNext → RoBERTa에서 제거 (너무 쉬움)
  • Fine-tuning: [CLS]→분류, 각 토큰→NER, start/end→QA
  • DistilBERT: 6 layers, 66M params, 97% 성능, Knowledge Distillation
  • RoBERTa: NSP 제거 + Dynamic Masking + 10× 데이터 + 큰 배치 → GLUE 88.5
  • Dark knowledge: Teacher의 soft label에 담긴 클래스 간 유사성 정보
