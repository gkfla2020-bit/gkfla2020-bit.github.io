Layer Normalization — 학습 안정성과 수렴의 핵심

Batch Norm → Layer Norm → Pre-Norm → RMSNorm: 정규화 전략의 진화
Based on: Stanford CME 295 Lecture 2 (Afshine & Shervine Amidi)
Learning Objectives

이 라운드를 마치면 다음을 할 수 있다:

  • Normalization이 딥러닝 학습에서 왜 필수적인지 설명할 수 있다
  • Layer Normalization의 수식을 쓰고 각 항의 역할을 설명할 수 있다
  • Batch Norm과 Layer Norm의 정규화 축 차이를 시각적으로 이해한다
  • Post-Norm과 Pre-Norm의 구조적 차이와 각각의 장단점을 비교할 수 있다
  • RMSNorm이 Layer Norm을 어떻게 단순화했는지 설명할 수 있다
  • 현대 LLM이 Pre-Norm + RMSNorm 조합을 사용하는 이유를 이해한다

1. 왜 정규화(Normalization)가 필요한가

1.1 Internal Covariate Shift 문제

딥러닝에서 각 레이어의 입력 분포는 학습이 진행되면서 계속 변한다. 이전 레이어의 파라미터가 업데이트되면, 다음 레이어가 받는 입력의 분포도 바뀐다. 이것을 Internal Covariate Shift라고 한다.

문제: 입력 분포가 계속 변하면...

  • 학습이 불안정해진다 (loss가 진동)
  • 학습률을 작게 설정해야 한다 → 학습이 느려진다
  • 깊은 네트워크일수록 문제가 심각해진다
  • Gradient vanishing/exploding이 발생하기 쉽다

1.2 정규화의 핵심 아이디어

각 레이어의 입력을 평균 0, 분산 1로 정규화하면 분포가 안정된다. 이것이 모든 normalization 기법의 공통 아이디어다.

\[ \hat{x} = \frac{x - \mu}{\sigma} \]

하지만 단순히 정규화만 하면 표현력이 줄어든다. 그래서 학습 가능한 파라미터 \(\gamma\) (scale)와 \(\beta\) (shift)를 추가한다:

\[ y = \gamma \cdot \hat{x} + \beta \]

모델이 필요하면 정규화를 "되돌릴" 수 있도록 하는 것이다. \(\gamma = \sigma, \beta = \mu\)로 학습하면 원래 분포로 돌아간다.

핵심 질문: "어떤 축(dimension)을 기준으로 평균과 분산을 계산할 것인가?" — 이 질문에 대한 답이 Batch Norm과 Layer Norm을 구분한다.

2. Layer Normalization — Transformer의 선택

2.1 수식

Definition 2.1 — Layer Normalization (Ba et al., 2016)

각 샘플 내에서 feature dimension을 기준으로 정규화:

\[ \text{LayerNorm}(x) = \gamma \odot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta \]

여기서:

\[ \mu = \frac{1}{d} \sum_{i=1}^{d} x_i, \quad \sigma^2 = \frac{1}{d} \sum_{i=1}^{d} (x_i - \mu)^2 \]
  • \(\mu, \sigma^2\): 해당 샘플의 feature dimension에 대한 평균과 분산
  • \(\gamma \in \mathbb{R}^d\): 학습 가능한 scale 파라미터 (초기값 1)
  • \(\beta \in \mathbb{R}^d\): 학습 가능한 shift 파라미터 (초기값 0)
  • \(\epsilon\): 수치 안정성을 위한 작은 값 (보통 \(10^{-5}\))
  • \(\odot\): element-wise 곱셈

2.2 단계별 계산 예시

import numpy as np

def layer_norm(x, gamma, beta, eps=1e-5):
    """
    Layer Normalization 구현
    x: (batch_size, seq_len, d_model) 또는 (seq_len, d_model)
    gamma: (d_model,) - scale
    beta: (d_model,) - shift
    """
    # 마지막 축(feature dimension)에 대해 평균/분산 계산
    mu = x.mean(axis=-1, keepdims=True)       # 평균
    sigma2 = x.var(axis=-1, keepdims=True)     # 분산
    
    # 정규화
    x_hat = (x - mu) / np.sqrt(sigma2 + eps)
    
    # Scale & Shift
    out = gamma * x_hat + beta
    return out

# 예시: 3개 토큰, d_model=4
x = np.array([
    [1.0, 2.0, 3.0, 4.0],   # 토큰 0
    [2.0, 4.0, 6.0, 8.0],   # 토큰 1
    [1.0, 1.0, 1.0, 1.0],   # 토큰 2
])

gamma = np.ones(4)   # 초기값 1
beta = np.zeros(4)   # 초기값 0

out = layer_norm(x, gamma, beta)
print("Input:")
print(x)
print("\nAfter LayerNorm:")
print(out.round(4))
# 토큰 0: [-1.3416, -0.4472, 0.4472, 1.3416]  (평균≈0, 분산≈1)
# 토큰 1: [-1.3416, -0.4472, 0.4472, 1.3416]  (같은 패턴!)
# 토큰 2: [ 0.0,     0.0,    0.0,    0.0   ]  (모두 같으면 0)

# 핵심: 각 토큰(행)별로 독립적으로 정규화!
# 다른 토큰이나 다른 배치 샘플에 영향받지 않음

강의 포인트: Layer Norm은 각 토큰(샘플)을 독립적으로 정규화한다. 배치 내 다른 샘플에 의존하지 않으므로:

  • 배치 크기에 독립적 (batch size = 1이어도 동작)
  • 가변 길이 시퀀스를 자연스럽게 처리
  • 추론 시 별도의 running statistics 불필요

3. Batch Normalization vs Layer Normalization

3.1 정규화 축의 차이

Batch Norm과 Layer Norm의 핵심 차이는 "어떤 축을 기준으로 평균/분산을 계산하는가"이다.

┌─────────────────────────────────────────────────┐ │ 입력 텐서: (Batch, Sequence, Feature) │ │ │ │ ┌─────────────────────────────────┐ │ │ │ Batch Norm │ │ │ │ → Batch 축 기준으로 정규화 │ │ │ │ → 같은 feature의 모든 샘플 사용 │ │ │ │ → 축: dim=0 (batch) │ │ │ │ │ │ │ │ B₁ ■ ■ ■ ■ ← feature j의 │ │ │ │ B₂ ■ ■ ■ ■ 모든 배치를 │ │ │ │ B₃ ■ ■ ■ ■ 함께 정규화 │ │ │ │ ↕ ↕ ↕ ↕ │ │ │ │ μ σ (feature별) │ │ │ └─────────────────────────────────┘ │ │ │ │ ┌─────────────────────────────────┐ │ │ │ Layer Norm │ │ │ │ → Feature 축 기준으로 정규화 │ │ │ │ → 한 샘플 내의 모든 feature 사용│ │ │ │ → 축: dim=-1 (feature) │ │ │ │ │ │ │ │ B₁ ■ ■ ■ ■ → μ,σ (샘플별) │ │ │ │ B₂ ■ ■ ■ ■ → μ,σ (샘플별) │ │ │ │ B₃ ■ ■ ■ ■ → μ,σ (샘플별) │ │ │ │ ←─────────→ │ │ │ │ 각 행을 독립적으로 정규화 │ │ │ └─────────────────────────────────┘ │ └─────────────────────────────────────────────────┘

3.2 상세 비교

특성Batch NormalizationLayer Normalization
정규화 축Batch dimension (같은 feature, 모든 샘플)Feature dimension (같은 샘플, 모든 feature)
통계량 계산배치 내 모든 샘플의 같은 feature한 샘플 내 모든 feature
배치 크기 의존Yes (작은 배치에서 불안정)No (batch size=1도 OK)

가변 길이처리 어려움 (패딩 문제)자연스럽게 처리

import numpy as np

# 입력: (batch=3, seq_len=4, d_model=5)
x = np.random.randn(3, 4, 5)

# ── Batch Norm ──
# 같은 feature의 모든 배치/시퀀스를 함께 정규화
# 축: (batch, seq_len) → feature별 통계
bn_mean = x.mean(axis=(0, 1), keepdims=True)  # shape: (1, 1, 5)
bn_var = x.var(axis=(0, 1), keepdims=True)
bn_out = (x - bn_mean) / np.sqrt(bn_var + 1e-5)

# ── Layer Norm ──
# 각 토큰의 모든 feature를 독립적으로 정규화
# 축: (feature) → 토큰별 통계
ln_mean = x.mean(axis=-1, keepdims=True)  # shape: (3, 4, 1)
ln_var = x.var(axis=-1, keepdims=True)
ln_out = (x - ln_mean) / np.sqrt(ln_var + 1e-5)

print(f"BN mean shape: {bn_mean.shape}")  # (1, 1, 5) - feature별
print(f"LN mean shape: {ln_mean.shape}")  # (3, 4, 1) - 토큰별

3.3 왜 Transformer는 Layer Norm을 선택했는가?

강의 인용 (Lecture 1): "The only reason to use Layer Normalization in Transformer is that the original paper was tuning the network based on Layer Normalization. If you want to use Batch Normalization, you need to tune hyperparameters again."

실용적 이유들:

  • 가변 길이: NLP에서 시퀀스 길이가 다양하다. Batch Norm은 패딩된 위치도 통계에 포함되어 문제가 된다
  • 배치 크기: LLM 학습은 메모리 제약으로 배치 크기가 작을 수 있다. Batch Norm은 작은 배치에서 불안정
  • 추론 편의: Layer Norm은 running statistics가 불필요. 추론 시 입력만으로 계산 가능
  • 역사적 이유: 원 논문이 Layer Norm으로 튜닝됨. 바꾸려면 재튜닝 필요

강의 포인트: "Helps with training stability and convergence." — Layer Norm의 핵심 기여는 학습 안정성과 수렴 속도 향상이다. 이것 없이는 깊은 Transformer를 학습하기 매우 어렵다.

4. Post-Norm vs Pre-Norm — 어디에 놓을 것인가

4.1 두 가지 배치 방식

Layer Norm을 Transformer 블록의 어디에 배치하느냐에 따라 학습 동작이 크게 달라진다. Xiong et al. (2020), "On Layer Normalization in the Transformer Architecture"에서 체계적으로 분석했다.

Definition 4.1 — Post-Norm (Original Transformer)

Residual connection 이후에 Layer Norm을 적용:

\[ x' = \text{LayerNorm}(x + \text{SubLayer}(x)) \]

순서: Input → SubLayer → Add (Residual) → LayerNorm → Output

Definition 4.2 — Pre-Norm (현대 방식)

SubLayer 이전에 Layer Norm을 적용:

\[ x' = x + \text{SubLayer}(\text{LayerNorm}(x)) \]

순서: Input → LayerNorm → SubLayer → Add (Residual) → Output

4.2 구조 비교 다이어그램

┌──────────────────────┐ ┌──────────────────────┐ │ Post-Norm │ │ Pre-Norm │ │ (Original, 2017) │ │ (Modern, 현재 표준) │ │ │ │ │ │ Input x │ │ Input x │ │ │ │ │ │ │ │ ├──────┐ │ │ ├──────┐ │ │ │ │ │ │ │ │ │ │ ▼ │ │ │ LayerNorm │ │ │ SubLayer │ │ │ │ │ │ │ (Attn/FFN) │ │ │ ▼ │ │ │ │ │ │ │ SubLayer │ │ │ ▼ │ │ │ (Attn/FFN) │ │ │ Add ◄────┘ │ │ │ │ │ │ │ │ │ ▼ │ │ │ LayerNorm │ │ Add ◄────┘ │ │ │ │ │ │ │ │ Output │ │ Output │ └──────────────────────┘ └──────────────────────┘

4.3 왜 Pre-Norm이 더 좋은가?

Table 2. Post-Norm vs Pre-Norm 비교
특성Post-NormPre-Norm
학습 안정성불안정 (warm-up 필수)안정적 (warm-up 없이도 가능)
Gradient flowLayerNorm이 residual path를 차단Residual path가 깨끗하게 유지
깊은 모델학습 어려움더 깊은 모델 학습 가능
최종 성능약간 더 좋을 수 있음 (잘 학습되면)약간 낮을 수 있음
사용 모델Original Transformer, BERTGPT-2+, LLaMA, Mistral

핵심 직관 — Gradient Flow:

Post-Norm: Residual connection 후에 LayerNorm이 오므로, gradient가 역전파될 때 LayerNorm을 반드시 통과해야 한다. 깊은 네트워크에서 이것이 gradient flow를 방해한다.

Pre-Norm: Residual connection이 LayerNorm 바깥에 있으므로, gradient가 residual path를 통해 직접 흐를 수 있다. ResNet의 skip connection과 같은 원리.

# Post-Norm vs Pre-Norm 구현 비교

def post_norm_block(x, sublayer, ln):
    """Post-Norm: LN(x + SubLayer(x))"""
    # 1. SubLayer 적용
    sub_out = sublayer(x)
    # 2. Residual + LayerNorm
    out = ln(x + sub_out)  # LN이 residual 뒤에!
    return out

def pre_norm_block(x, sublayer, ln):
    """Pre-Norm: x + SubLayer(LN(x))"""
    # 1. LayerNorm 먼저
    normed = ln(x)
    # 2. SubLayer 적용
    sub_out = sublayer(normed)
    # 3. Residual (깨끗한 skip connection)
    out = x + sub_out  # Residual path가 깨끗!
    return out

# Pre-Norm에서 gradient flow:
# ∂L/∂x = ∂L/∂out × (1 + ∂SubLayer(LN(x))/∂x)
#                     ↑ 이 "1"이 gradient를 직접 전달!
# → ResNet의 skip connection과 동일한 원리

강의 포인트: "Nowadays: Pre-Norm." — 현대 LLM은 거의 모두 Pre-Norm을 사용한다. 학습 안정성이 훨씬 좋고, learning rate warm-up에 덜 민감하다.

5. RMSNorm — Root Mean Square Layer Normalization ⭐

5.1 Layer Norm의 비효율성

Layer Norm은 평균(\(\mu\))과 분산(\(\sigma^2\))을 모두 계산한다. Zhang et al. (2019)은 "평균을 빼는 것(re-centering)이 정말 필요한가?"라는 질문을 던졌다.

관찰: Layer Norm의 성능 향상은 주로 re-scaling (\(\frac{1}{\sigma}\))에서 온다. Re-centering (\(-\mu\))의 기여는 미미하다. 그렇다면 평균 계산을 생략하면 더 효율적이지 않을까?

5.2 RMSNorm 수식

Definition 5.1 — RMSNorm (Zhang et al., 2019)

평균을 빼지 않고, Root Mean Square로만 정규화:

\[ \text{RMSNorm}(x) = \gamma \odot \frac{x}{\text{RMS}(x)}, \quad \text{RMS}(x) = \sqrt{\frac{1}{d} \sum_{i=1}^{d} x_i^2} \]

Layer Norm과의 차이:

  • 평균 \(\mu\)를 계산하지 않음 (re-centering 없음)
  • Shift 파라미터 \(\beta\)도 없음
  • RMS = Root Mean Square = 제곱 평균의 제곱근

5.3 Layer Norm vs RMSNorm 비교

Table 3. Layer Norm vs RMSNorm
특성Layer NormRMSNorm
수식\(\gamma \cdot \frac{x - \mu}{\sigma} + \beta\)\(\gamma \cdot \frac{x}{\text{RMS}(x)}\)
Re-centeringYes (\(-\mu\))No
Re-scalingYes (\(\frac{1}{\sigma}\))Yes (\(\frac{1}{\text{RMS}}\))
학습 파라미터\(\gamma, \beta\) (2d개)\(\gamma\)만 (d개)
계산량\(\mu\)와 \(\sigma^2\) 모두 계산RMS만 계산 (더 빠름)
성능좋음동등하거나 약간 좋음
import numpy as np

def rms_norm(x, gamma, eps=1e-5):
    """
    RMSNorm 구현
    x: (..., d_model)
    gamma: (d_model,) - scale만 (shift 없음!)
    """
    # RMS 계산 (평균을 빼지 않음!)
    rms = np.sqrt(np.mean(x ** 2, axis=-1, keepdims=True) + eps)
    
    # 정규화 + scale
    out = gamma * (x / rms)
    return out

def layer_norm_compare(x, gamma, beta, eps=1e-5):
    """Layer Norm (비교용)"""
    mu = x.mean(axis=-1, keepdims=True)
    sigma = np.sqrt(x.var(axis=-1, keepdims=True) + eps)
    return gamma * (x - mu) / sigma + beta

# 비교
x = np.array([[1.0, 2.0, 3.0, 4.0]])
gamma = np.ones(4)
beta = np.zeros(4)

ln_out = layer_norm_compare(x, gamma, beta)
rms_out = rms_norm(x, gamma)

print(f"Input:     {x[0]}")
print(f"LayerNorm: {ln_out[0].round(4)}")
print(f"RMSNorm:   {rms_out[0].round(4)}")
# LayerNorm: [-1.3416, -0.4472, 0.4472, 1.3416]  (평균=0)
# RMSNorm:   [ 0.3651,  0.7303, 1.0954, 1.4606]  (평균≠0)

# 파라미터 수 비교 (d_model=4096 기준)
d = 4096
print(f"\nLayerNorm params: {2*d:,} (γ + β)")  # 8,192
print(f"RMSNorm params:   {d:,} (γ only)")    # 4,096

5.4 현대 LLM의 선택: Pre-Norm + RMSNorm

강의 결론: "Nowadays: Pre-Norm + RMSNorm"

현대 LLM (LLaMA, LLaMA 2, Mistral, Gemma 등)은 거의 모두 이 조합을 사용한다:

  • Pre-Norm: 학습 안정성, 깨끗한 gradient flow
  • RMSNorm: Layer Norm과 동등한 성능, 더 빠른 계산, 더 적은 파라미터
# 현대 LLM의 Transformer Block (Pre-Norm + RMSNorm)

def modern_transformer_block(x, attn, ffn, rms1, rms2):
    """LLaMA-style Transformer Block"""
    # 1. Pre-Norm + Self-Attention
    h = x + attn(rms1(x))      # RMSNorm → Attention → Residual
    
    # 2. Pre-Norm + Feed-Forward
    out = h + ffn(rms2(h))      # RMSNorm → FFN → Residual
    
    return out

# Original Transformer (Post-Norm + LayerNorm) 비교
def original_transformer_block(x, attn, ffn, ln1, ln2):
    """Original Transformer Block"""
    # 1. Self-Attention + Post-Norm
    h = ln1(x + attn(x))       # Attention → Residual → LayerNorm
    
    # 2. Feed-Forward + Post-Norm
    out = ln2(h + ffn(h))       # FFN → Residual → LayerNorm
    
    return out

Q1. Layer Normalization이 정규화하는 축은?

A. Batch dimension (같은 feature, 모든 샘플)
B. Feature dimension (같은 샘플, 모든 feature)
C. Sequence dimension (같은 위치, 모든 시퀀스)
D. 모든 축을 동시에
Answer: B
Layer Norm은 각 샘플(토큰) 내에서 feature dimension을 기준으로 평균/분산을 계산한다. 다른 샘플이나 배치에 독립적.
Feature 축 정규화. 각 토큰을 독립적으로 처리.

Q2. Batch Norm이 NLP에서 잘 안 쓰이는 이유는?

A. 계산이 느려서
B. 가변 길이 시퀀스 처리가 어렵고, 작은 배치에서 불안정
C. 파라미터가 너무 많아서
D. GPU에서 지원하지 않아서
Answer: B
NLP에서 시퀀스 길이가 다양하고 패딩이 필요. Batch Norm은 패딩된 위치도 통계에 포함. 또한 LLM 학습은 메모리 제약으로 배치가 작을 수 있어 통계가 불안정.
가변 길이 + 작은 배치 = Batch Norm 부적합.

Q3. Layer Norm 수식에서 γ와 β의 역할은?

A. 정규화를 수행한다
B. 학습 가능한 scale과 shift로, 모델이 필요하면 정규화를 되돌릴 수 있게 한다
C. Gradient를 안정화한다
D. 위치 정보를 인코딩한다
Answer: B
γ(scale)와 β(shift)는 학습 가능한 파라미터. γ=σ, β=μ로 학습하면 원래 분포로 돌아갈 수 있다. 정규화의 표현력 손실을 보상.
학습 가능한 scale/shift. 필요하면 정규화를 되돌릴 수 있음.

Q4. Post-Norm과 Pre-Norm의 핵심 차이는?

A. 사용하는 정규화 방법이 다르다
B. Post-Norm은 residual 후 LN, Pre-Norm은 SubLayer 전 LN
C. 파라미터 수가 다르다
D. 학습 데이터가 다르다
Answer: B
Post-Norm: LN(x + SubLayer(x)). Pre-Norm: x + SubLayer(LN(x)). Pre-Norm에서는 residual path가 깨끗하게 유지되어 gradient flow가 좋다.
Post: LN이 residual 뒤. Pre: LN이 SubLayer 앞. Pre가 gradient flow 우수.

Q5. Pre-Norm이 Post-Norm보다 학습이 안정적인 이유는?

A. 파라미터가 적어서
B. Residual path가 LayerNorm을 거치지 않아 gradient가 직접 흐를 수 있어서
C. 학습률이 자동으로 조절되어서
D. 배치 크기에 독립적이어서
Answer: B
Pre-Norm: ∂L/∂x = ∂L/∂out × (1 + ...). "1"이 gradient를 직접 전달. ResNet의 skip connection과 동일 원리. Post-Norm은 LN이 residual path를 차단.
Residual path가 깨끗 → gradient 직접 전달 → 깊은 모델 학습 가능.

Q6. RMSNorm이 Layer Norm에서 생략한 것은?

A. Scale 파라미터 γ
B. 분산 계산
C. 평균 계산 (re-centering)과 shift 파라미터 β
D. 정규화 자체
Answer: C
RMSNorm은 평균 μ를 계산하지 않고(re-centering 없음), shift 파라미터 β도 없다. RMS(Root Mean Square)로만 정규화. 성능은 동등하면서 더 빠르다.
평균 빼기(re-centering) 생략 + β 제거. 더 빠르고 동등한 성능.

Q7. 현대 LLM (LLaMA, Mistral)이 사용하는 정규화 조합은?

A. Post-Norm + Batch Norm
B. Post-Norm + Layer Norm
C. Pre-Norm + Layer Norm
D. Pre-Norm + RMSNorm
Answer: D
"Nowadays: Pre-Norm + RMSNorm." Pre-Norm으로 학습 안정성 확보, RMSNorm으로 계산 효율성 향상. LLaMA, Mistral, Gemma 등이 사용.
Pre-Norm(안정성) + RMSNorm(효율성) = 현대 LLM의 표준.

Q8. Layer Norm이 Batch Norm보다 추론 시 편리한 이유는?

A. 계산이 더 빠르다
B. Running mean/var 같은 별도 통계가 불필요하고, 입력만으로 계산 가능
C. GPU 메모리를 덜 사용한다
D. 정확도가 더 높다
Answer: B
Batch Norm은 학습 시 축적한 running mean/var를 추론 시 사용해야 한다. Layer Norm은 현재 입력에서 직접 통계를 계산하므로 별도 저장이 불필요.
Layer Norm: 입력만으로 계산. Batch Norm: running statistics 필요.

Q9. RMSNorm의 수식에서 RMS(x)는?

A. \(\sqrt{\frac{1}{d}\sum x_i}\)
B. \(\sqrt{\frac{1}{d}\sum x_i^2}\)
C. \(\frac{1}{d}\sum |x_i|\)
D. \(\sqrt{\frac{1}{d}\sum (x_i - \mu)^2}\)
Answer: B
RMS = Root Mean Square = √(평균(x²)). 평균을 빼지 않고 제곱의 평균의 제곱근. D는 표준편차(σ)로 Layer Norm에서 사용.
RMS = √(1/d × Σx²). 평균을 빼지 않는 것이 핵심 차이.

Q10. Normalization의 학습 가능한 파라미터 γ를 1, β를 0으로 초기화하는 이유는?

A. 랜덤 초기화보다 빠르기 때문
B. 초기에는 identity mapping (입력 그대로 통과)에 가깝게 시작하기 위해
C. 메모리를 절약하기 위해
D. Gradient를 0으로 만들기 위해
Answer: B
γ=1, β=0이면 정규화된 값이 그대로 출력된다 (y = 1·x̂ + 0 = x̂). 학습 초기에 안정적인 시작점을 제공. 학습이 진행되면서 모델이 최적의 γ, β를 찾아간다.
초기에 identity에 가깝게 → 안정적 시작 → 점진적 학습.

📋 Cheat Sheet — Layer Normalization 한눈에

🔑 핵심 수식 2개

1. Layer Norm:

\[ \text{LN}(x) = \gamma \cdot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta, \quad \mu = \frac{1}{d}\sum x_i, \quad \sigma^2 = \frac{1}{d}\sum(x_i-\mu)^2 \]

2. RMSNorm:

\[ \text{RMSNorm}(x) = \gamma \cdot \frac{x}{\sqrt{\frac{1}{d}\sum x_i^2 + \epsilon}} \]

🎯 시험 대비 핵심 포인트

  • Batch Norm: batch 축 정규화, CNN용. Layer Norm: feature 축 정규화, Transformer용
  • Layer Norm이 NLP에 적합한 이유: 배치 독립, 가변 길이, running stats 불필요
  • Post-Norm: LN(x + Sub(x)) — Original. Pre-Norm: x + Sub(LN(x)) — 현대
  • Pre-Norm이 좋은 이유: residual path가 깨끗 → gradient 직접 전달
  • RMSNorm: 평균 빼기 생략, β 제거. 더 빠르고 동등한 성능
  • 현대 표준: Pre-Norm + RMSNorm (LLaMA, Mistral)
