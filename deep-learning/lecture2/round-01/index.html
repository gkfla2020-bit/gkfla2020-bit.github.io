Position Embeddings — 위치 정보를 Transformer에 주입하는 방법

Sinusoidal → Learned → ALiBi → RoPE: 절대 위치에서 상대 위치로의 진화
Based on: Stanford CME 295 Lecture 2 (Afshine & Shervine Amidi)
Learning Objectives

이 라운드를 마치면 다음을 할 수 있다:

  • Self-Attention이 왜 위치 정보를 잃는지 수학적으로 설명할 수 있다
  • Sinusoidal Positional Encoding의 수식과 장단점을 설명할 수 있다
  • Learned PE의 한계와 Sinusoidal PE와의 차이를 비교할 수 있다
  • ALiBi가 attention layer를 어떻게 변형하는지 이해한다
  • RoPE의 핵심 아이디어(회전 행렬)와 왜 현재 표준인지 설명할 수 있다
  • 절대 위치 vs 상대 위치 인코딩의 차이를 구분할 수 있다

1. 위치 정보의 필요성 — Why Position Matters

1.1 Self-Attention은 순서를 모른다

Lecture 1에서 배운 Self-Attention을 떠올려보자. \(\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\) 이 수식에서 토큰의 순서 정보는 어디에도 없다. Q, K, V는 입력 임베딩 \(X\)에 projection matrix를 곱해서 만드는데, 이 과정에서 "몇 번째 토큰인지"는 반영되지 않는다.

핵심 문제: Self-Attention은 set operation이다. 입력 토큰의 순서를 바꿔도 (permutation) 출력이 동일하다.

  • "I love you" → Attention 결과 A
  • "You love I" → Attention 결과 A (동일!)

이것은 NLP에서 치명적이다. 어순이 의미를 결정하기 때문이다.

강의 인용: "Direct links 'lose' position info. We need to add position-specific information to the token vector."

→ 해결책: 토큰 임베딩에 위치 정보를 더해주자!

1.2 위치 인코딩의 기본 아이디어

토큰 임베딩 벡터에 위치 벡터를 더한다:

\[ \text{Input} = \text{Token Embedding} + \text{Position Embedding} \]

이렇게 하면 같은 단어라도 위치에 따라 다른 벡터가 된다. "love"가 2번째 위치에 있을 때와 5번째 위치에 있을 때 다른 표현을 갖게 된다.

Position Embedding의 두 가지 접근법:

  • 절대 위치 (Absolute): 각 위치에 고유한 벡터를 할당. "나는 3번째 위치야"
  • 상대 위치 (Relative): 토큰 간의 거리를 인코딩. "나는 너보다 2칸 앞이야"

역사적으로 절대 위치 → 상대 위치로 발전해왔다.

# 위치 인코딩의 기본 개념
import numpy as np

d_model = 512
seq_len = 6  # "A cute teddy bear is reading"

# Token embedding (Word2Vec 등으로 학습된 것)
token_embed = np.random.randn(seq_len, d_model)

# Position embedding (방법에 따라 다름)
pos_embed = np.zeros((seq_len, d_model))  # 이걸 어떻게 채울 것인가?

# 최종 입력 = 토큰 + 위치
input_embed = token_embed + pos_embed
print(f"Input shape: {input_embed.shape}")  # (6, 512)

2. Sinusoidal Positional Encoding — Original Transformer (2017)

2.1 수식

Vaswani et al. (2017)의 "Attention Is All You Need"에서 제안한 방법. 학습 없이 수학적 함수로 위치를 인코딩한다.

Definition 2.1 — Sinusoidal Positional Encoding

\[ PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]
\[ PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]

여기서:

  • \(pos\): 토큰의 위치 (0, 1, 2, ...)
  • \(i\): 차원 인덱스 (0, 1, 2, ..., d_model/2 - 1)
  • 짝수 차원(2i)은 sin, 홀수 차원(2i+1)은 cos
  • 10000은 base frequency (하이퍼파라미터)

2.2 직관적 이해

이 수식이 하는 일을 직관적으로 이해해보자. 각 차원마다 다른 주파수의 sin/cos 파동을 사용한다:

Table 1. 차원별 주파수
차원 \(i\)주파수 \(\frac{1}{10000^{2i/d}}\)파장의미
0 (낮은 차원)높은 주파수짧은 파장가까운 위치 구분
128중간 주파수중간 파장중간 거리 구분
255 (높은 차원)낮은 주파수긴 파장먼 위치 구분

비유: 시계의 초침, 분침, 시침과 같다!

  • 초침 (낮은 차원) = 빠르게 회전 → 가까운 시간(위치) 구분
  • 분침 (중간 차원) = 중간 속도 → 중간 거리 구분
  • 시침 (높은 차원) = 느리게 회전 → 먼 시간(위치) 구분

이 조합으로 모든 위치를 고유하게 표현할 수 있다.

2.3 핵심 성질: 상대 위치의 선형 표현

Sinusoidal PE의 가장 중요한 수학적 성질: 위치 \(pos+k\)의 인코딩을 위치 \(pos\)의 인코딩으로부터 선형 변환으로 얻을 수 있다.

Theorem 2.1 — 상대 위치의 선형 표현

삼각함수의 덧셈 정리에 의해:

\[ \begin{pmatrix} \sin(\omega \cdot (pos+k)) \\ \cos(\omega \cdot (pos+k)) \end{pmatrix} = \begin{pmatrix} \cos(\omega k) & \sin(\omega k) \\ -\sin(\omega k) & \cos(\omega k) \end{pmatrix} \begin{pmatrix} \sin(\omega \cdot pos) \\ \cos(\omega \cdot pos) \end{pmatrix} \]

즉, \(PE_{pos+k}\)는 \(PE_{pos}\)에 \(k\)에만 의존하는 회전 행렬을 곱한 것이다. 이 덕분에 모델이 상대적 위치를 학습할 수 있다.

2.4 위치 간 유사도

Sinusoidal PE의 또 다른 중요한 성질: 가까운 위치일수록 PE 벡터의 내적(유사도)이 크다.

\[ \text{similarity}(PE_{pos}, PE_{pos+k}) \propto \sum_i \cos(\omega_i \cdot k) \]

\(k\)가 작을수록 (가까운 위치) 유사도가 높고, \(k\)가 클수록 (먼 위치) 유사도가 낮아진다. 이것은 자연어에서 가까운 단어가 더 관련 있다는 직관과 일치한다.

import numpy as np

def sinusoidal_pe(max_len, d_model):
    """Sinusoidal Positional Encoding 구현"""
    pe = np.zeros((max_len, d_model))
    position = np.arange(max_len)[:, np.newaxis]  # (max_len, 1)
    
    # 각 차원의 주파수 계산
    div_term = np.exp(
        np.arange(0, d_model, 2) * (-np.log(10000.0) / d_model)
    )  # (d_model/2,)
    
    # 짝수 차원: sin, 홀수 차원: cos
    pe[:, 0::2] = np.sin(position * div_term)
    pe[:, 1::2] = np.cos(position * div_term)
    
    return pe

# 생성 및 확인
pe = sinusoidal_pe(max_len=100, d_model=512)
print(f"PE shape: {pe.shape}")  # (100, 512)

# 위치 간 유사도 확인
sim_1 = np.dot(pe[0], pe[1])   # 가까운 위치 → 높은 유사도
sim_50 = np.dot(pe[0], pe[50])  # 먼 위치 → 낮은 유사도
print(f"Similarity(pos 0, pos 1): {sim_1:.2f}")
print(f"Similarity(pos 0, pos 50): {sim_50:.2f}")

2.5 장단점

장점:

  • 학습 파라미터 없음 (hard-coded) → 추가 학습 비용 없음
  • 임의의 시퀀스 길이에 일반화 가능 (학습 시 보지 못한 길이도 처리)
  • 상대 위치를 선형 변환으로 표현 가능

한계:

  • 절대 위치 인코딩 — 실제로 attention에서 중요한 것은 상대 위치
  • 위치 정보가 토큰 임베딩에 더해지므로 (additive), 정보가 섞일 수 있음
  • 긴 시퀀스에서 위치 구분력이 떨어질 수 있음

3. Learned Positional Embedding

3.1 아이디어

Sinusoidal PE가 수학적 함수로 위치를 인코딩했다면, Learned PE는 각 위치의 임베딩을 학습 가능한 파라미터로 만든다. BERT, GPT-2 등에서 사용.

Definition 3.1 — Learned Positional Embedding

위치 임베딩 행렬 \(P \in \mathbb{R}^{L_{max} \times d_{model}}\)을 학습 가능한 파라미터로 정의. \(L_{max}\)는 최대 시퀀스 길이.

\[ \text{Input}_i = \text{TokenEmbed}(x_i) + P_i \]

여기서 \(P_i\)는 위치 \(i\)에 해당하는 학습된 벡터.

import numpy as np

# Learned PE: 학습 가능한 파라미터
L_max = 512    # 최대 시퀀스 길이
d_model = 768  # BERT-Base의 d_model

# 이 행렬이 학습됨 (random init → backprop으로 업데이트)
P = np.random.randn(L_max, d_model) * 0.02  # 작은 값으로 초기화

# 위치 i의 임베딩
pos_3 = P[3]  # 3번째 위치의 학습된 벡터
print(f"Position 3 embedding shape: {pos_3.shape}")  # (768,)

# 총 추가 파라미터 수
print(f"추가 파라미터: {L_max * d_model:,}")  # 393,216개

3.2 Sinusoidal vs Learned 비교

Table 2. Sinusoidal PE vs Learned PE
특성Sinusoidal PELearned PE
파라미터없음 (hard-coded)\(L_{max} \times d_{model}\)개
길이 일반화가능 (임의 길이)불가능 (\(L_{max}\) 초과 시 재학습 필요)
성능좋음비슷하거나 약간 좋음

강의 포인트: Original Transformer 논문에서 Sinusoidal과 Learned PE의 성능 차이가 거의 없었다고 보고했다. 하지만 Learned PE는 최대 길이를 넘는 시퀀스를 처리할 수 없다는 치명적 한계가 있다. "Need to retrain for longer sequences."

Learned PE의 치명적 한계:

BERT는 \(L_{max} = 512\)로 학습되었다. 513번째 토큰이 오면? 해당 위치의 임베딩이 존재하지 않는다! 더 긴 시퀀스를 처리하려면 모델을 재학습해야 한다.

→ 이 한계가 "절대 위치에서 상대 위치로"의 전환을 촉발했다.

4. ALiBi — Attention with Linear Biases

4.1 절대 위치에서 상대 위치로의 전환

Sinusoidal PE와 Learned PE는 모두 절대 위치를 인코딩한다. 하지만 실제로 attention에서 중요한 것은 상대 위치다. "이 단어가 3번째에 있다"보다 "이 단어가 저 단어보다 2칸 앞에 있다"가 더 유용하다.

강의 인용: "Actually, we really care about relative position with attention layer. Let's change the attention layer instead."

→ 핵심 발상의 전환: 입력에 위치를 더하는 대신, attention 메커니즘 자체를 변형하자!

4.2 ALiBi의 아이디어

Press et al. (2021), "Train Short, Test Long"에서 제안. Attention score에 위치 기반 bias를 더한다.

Definition 4.1 — ALiBi (Attention with Linear Biases)

기존 attention score에 선형 bias를 더한다:

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + m \cdot B\right) V \]

여기서:

  • \(B_{ij} = -|i - j|\): 위치 \(i\)와 \(j\)의 거리에 비례하는 페널티
  • \(m\): head별 기울기 (slope). 각 head마다 다른 값
  • 학습 파라미터 없음! \(m\)은 기하급수적으로 설정 (deterministic)

4.3 Bias 행렬의 구조

# ALiBi bias 행렬 예시 (5 tokens)
# B[i][j] = -|i - j|
#
#     pos0  pos1  pos2  pos3  pos4
# pos0 [  0,  -1,  -2,  -3,  -4 ]
# pos1 [ -1,   0,  -1,  -2,  -3 ]
# pos2 [ -2,  -1,   0,  -1,  -2 ]
# pos3 [ -3,  -2,  -1,   0,  -1 ]
# pos4 [ -4,  -3,  -2,  -1,   0 ]

import numpy as np

def alibi_bias(seq_len):
    """ALiBi bias 행렬 생성"""
    positions = np.arange(seq_len)
    # B[i][j] = -|i - j|
    bias = -np.abs(positions[:, None] - positions[None, :])
    return bias

B = alibi_bias(5)
print("ALiBi Bias Matrix:")
print(B)
# [[ 0 -1 -2 -3 -4]
#  [-1  0 -1 -2 -3]
#  [-2 -1  0 -1 -2]
#  [-3 -2 -1  0 -1]
#  [-4 -3 -2 -1  0]]

# Head별 slope (기하급수적)
# 8 heads: m = 1/2, 1/4, 1/8, 1/16, 1/32, 1/64, 1/128, 1/256
n_heads = 8
slopes = [1 / (2 ** (i + 1)) for i in range(n_heads)]
print(f"Slopes: {slopes}")
# [0.5, 0.25, 0.125, 0.0625, 0.03125, ...]

# Head 0 (steep slope): 가까운 토큰에 강하게 집중
# Head 7 (gentle slope): 먼 토큰도 어느 정도 참조

ALiBi의 핵심 장점: "Train Short, Test Long"

짧은 시퀀스로 학습하고 긴 시퀀스에서 추론할 수 있다! Bias가 선형이고 결정적이므로, 학습 시 보지 못한 긴 거리에도 자연스럽게 확장된다. 이것이 Learned PE의 한계를 극복한 핵심.

5. RoPE — Rotary Position Embeddings ⭐ (현재 표준)

5.1 핵심 아이디어

Su et al. (2021), "RoFormer"에서 제안. 현재 LLaMA, Mistral, GPT-NeoX 등 대부분의 최신 LLM이 사용하는 표준 방법.

Definition 5.1 — RoPE (Rotary Position Embeddings)

핵심 아이디어: Query와 Key 벡터를 위치에 따라 회전시킨다.

\[ \tilde{q}_m = R_m \cdot q_m, \quad \tilde{k}_n = R_n \cdot k_n \]

여기서 \(R_m\)은 위치 \(m\)에 대한 회전 행렬:

\[ R_m = \begin{pmatrix} \cos(m\theta) & -\sin(m\theta) \\ \sin(m\theta) & \cos(m\theta) \end{pmatrix} \]

\(\theta\)는 차원별 주파수 (Sinusoidal PE와 유사한 개념).

5.2 왜 회전인가? — 상대 위치의 자연스러운 인코딩

RoPE의 마법: 회전된 Q와 K의 내적이 자동으로 상대 위치를 반영한다!

Theorem 5.1 — RoPE의 핵심 성질

\[ \tilde{q}_m^T \tilde{k}_n = (R_m q_m)^T (R_n k_n) = q_m^T R_m^T R_n k_n = q_m^T R_{n-m} k_n \]

회전 행렬의 성질 \(R_m^T R_n = R_{n-m}\)에 의해, attention score가 자동으로 상대 위치 \(n-m\)에만 의존한다!

→ 절대 위치를 인코딩하지만, attention 계산에서는 상대 위치만 반영된다. 두 마리 토끼를 잡은 것.

5.3 고차원으로의 확장

실제 \(d_k\)는 2보다 훨씬 크다 (보통 64 또는 128). 2차원 회전을 블록 단위로 적용한다:

\[ R_m^{(d)} = \begin{pmatrix} R_m^{(\theta_0)} & & \\ & R_m^{(\theta_1)} & \\ & & \ddots \\ & & & R_m^{(\theta_{d/2-1})} \end{pmatrix} \]

각 2차원 블록마다 다른 주파수 \(\theta_i\)로 회전. 총 \(d/2\)개의 독립적인 회전.

import numpy as np

def rope_embedding(x, position, d_model, base=10000):
    """
    RoPE: 벡터 x를 position에 따라 회전
    x: (d_model,) - query 또는 key 벡터
    position: int - 토큰 위치
    """
    output = np.zeros_like(x)
    
    for i in range(0, d_model, 2):
        # 각 2차원 블록의 주파수
        theta = 1.0 / (base ** (i / d_model))
        angle = position * theta
        
        # 2D 회전 적용
        cos_a = np.cos(angle)
        sin_a = np.sin(angle)
        
        output[i]   = x[i] * cos_a - x[i+1] * sin_a
        output[i+1] = x[i] * sin_a + x[i+1] * cos_a
    
    return output

# 예시
d_model = 8  # 간단한 예시
q = np.random.randn(d_model)
k = np.random.randn(d_model)

# 위치 3의 query, 위치 5의 key
q_rot = rope_embedding(q, position=3, d_model=d_model)
k_rot = rope_embedding(k, position=5, d_model=d_model)

# 내적 = 상대 위치 (5-3=2)에만 의존!
score = np.dot(q_rot, k_rot)
print(f"Attention score: {score:.4f}")
print("이 score는 상대 위치 2에만 의존한다")

5.4 Attention Weight의 Long-term Decay

RoPE의 또 다른 중요한 성질: attention weight가 거리에 따라 자연스럽게 감소한다.

Property 5.1 — Long-term Decay

RoPE를 적용한 attention score의 상한:

\[ |\tilde{q}_m^T \tilde{k}_n| \leq f(|m-n|) \]

여기서 \(f\)는 거리 \(|m-n|\)에 대해 감소하는 함수. 즉, 먼 토큰일수록 attention score의 상한이 작아진다.

→ 가까운 토큰에 자연스럽게 더 많은 attention을 부여. 이것은 자연어의 locality 특성과 일치한다.

5.5 왜 RoPE가 현재 표준인가?

RoPE의 장점 종합:

  • 상대 위치를 자연스럽게 인코딩 (회전 행렬의 수학적 성질)
  • Long-term decay: 가까운 토큰에 자연스럽게 더 많은 attention
  • 길이 일반화 가능 (학습 시보다 긴 시퀀스 처리 가능)
  • 구현이 효율적 (element-wise 연산)
  • 추가 파라미터 없음

→ LLaMA, LLaMA 2, Mistral, GPT-NeoX, PaLM 등 현대 LLM의 사실상 표준.

Q1. Self-Attention이 위치 정보를 잃는 이유는?

A. Softmax가 위치를 무시하기 때문
B. Self-Attention은 set operation이라 입력 순서를 바꿔도 결과가 동일
C. Q, K, V 행렬이 위치를 삭제하기 때문
D. Residual connection이 위치를 제거하기 때문
Answer: B
Self-Attention의 QK^T 계산은 토큰 간 내적이므로 순서에 무관하다. 입력을 permutation해도 출력이 동일한 set operation이다.
Self-Attention은 순서 불변 연산. "I love you"와 "You love I"가 같은 결과.

Q2. Sinusoidal PE에서 낮은 차원과 높은 차원의 차이는?

A. 낮은 차원은 느린 주파수, 높은 차원은 빠른 주파수
B. 낮은 차원은 빠른 주파수(가까운 위치 구분), 높은 차원은 느린 주파수(먼 위치 구분)
C. 모든 차원이 같은 주파수
D. 차원에 따른 차이 없음
Answer: B
시계 비유: 초침(낮은 차원)=빠른 주파수→가까운 위치 구분, 시침(높은 차원)=느린 주파수→먼 위치 구분. 이 조합으로 모든 위치를 고유하게 표현.
낮은 차원=빠른 진동=가까운 위치, 높은 차원=느린 진동=먼 위치.

Q3. Learned PE의 치명적 한계는?

A. 학습이 느리다
B. 메모리를 많이 사용한다
C. 최대 시퀀스 길이(L_max)를 넘는 입력을 처리할 수 없다
D. 상대 위치를 인코딩할 수 없다
Answer: C
Learned PE는 L_max개의 위치 벡터만 학습한다. L_max+1번째 위치의 벡터가 존재하지 않으므로 처리 불가. "Need to retrain for longer sequences."
BERT의 L_max=512. 513번째 토큰이 오면 처리 불가 → 재학습 필요.

Q4. ALiBi의 핵심 아이디어는?

A. 입력 임베딩에 위치 벡터를 더한다
B. Attention score에 거리 기반 선형 bias를 더한다
C. Q, K 벡터를 회전시킨다
D. 위치별 학습 가능한 벡터를 사용한다
Answer: B
ALiBi: Attention score에 -|i-j| × m (선형 bias)을 더한다. 먼 토큰일수록 큰 페널티. 학습 파라미터 없이 "Train Short, Test Long" 가능.
Attention score에 거리 비례 페널티. 파라미터 없음, 길이 일반화 우수.

Q5. RoPE가 상대 위치를 인코딩하는 원리는?

A. 입력에 상대 위치 벡터를 더한다
B. Attention score에 bias를 더한다
C. Q, K를 위치에 따라 회전시켜, 내적이 상대 위치에만 의존하게 한다
D. 위치별 mask를 적용한다
Answer: C
RoPE: R_m^T × R_n = R_{n-m}. 회전된 Q, K의 내적이 자동으로 상대 위치 (n-m)에만 의존. 절대 위치를 인코딩하지만 attention에서는 상대 위치만 반영.
회전 행렬의 성질로 내적이 상대 위치에만 의존. 두 마리 토끼.

Q6. 현재 대부분의 최신 LLM (LLaMA, Mistral 등)이 사용하는 Position Embedding은?

A. Sinusoidal PE
B. Learned PE
C. ALiBi
D. RoPE (Rotary Position Embeddings)
Answer: D
RoPE는 상대 위치 인코딩, 파라미터 없음, long-term decay, 길이 일반화 등의 장점으로 현재 사실상 표준. LLaMA, Mistral, GPT-NeoX, PaLM 등이 사용.
RoPE = 현재 표준. 상대 위치 + 파라미터 없음 + long-term decay.

Q7. Sinusoidal PE에서 PE_{pos+k}를 PE_{pos}로부터 얻을 수 있는 이유는?

A. 선형 보간이 가능하기 때문
B. 삼각함수의 덧셈 정리에 의해 선형 변환(회전 행렬)으로 표현 가능
C. 학습된 변환 행렬이 있기 때문
D. 모든 PE가 같은 값이기 때문
Answer: B
sin(ω(pos+k)) = cos(ωk)sin(ωpos) + sin(ωk)cos(ωpos). 삼각함수 덧셈 정리에 의해 PE_{pos+k}는 PE_{pos}에 k에만 의존하는 회전 행렬을 곱한 것.
삼각함수 덧셈 정리 → 상대 위치 k에 대한 회전 행렬로 표현.

Q8. ALiBi에서 head별 slope의 역할은?

A. 모든 head가 같은 범위를 본다
B. 큰 slope의 head는 가까운 토큰에 집중, 작은 slope의 head는 먼 토큰도 참조
C. Slope는 학습되는 파라미터다
D. Slope는 attention weight를 정규화한다
Answer: B
Head 0 (m=1/2): steep slope → 가까운 토큰에 강하게 집중. Head 7 (m=1/256): gentle slope → 먼 토큰도 어느 정도 참조. Multi-Head의 다양성과 유사한 효과.
큰 slope=local attention, 작은 slope=global attention. 다양한 범위 커버.

Q9. RoPE의 "long-term decay" 성질이란?

A. 학습이 진행될수록 attention이 감소
B. 먼 토큰일수록 attention score의 상한이 자연스럽게 감소
C. 시퀀스가 길수록 모델 성능이 감소
D. 학습률이 시간에 따라 감소
Answer: B
RoPE에서 |q̃_m^T k̃_n| ≤ f(|m-n|), f는 감소 함수. 먼 토큰일수록 attention score 상한이 작아져 가까운 토큰에 자연스럽게 더 많은 attention 부여.
거리가 멀수록 attention 상한 감소 → 자연어의 locality와 일치.

Q10. Position Embedding의 진화 순서로 올바른 것은?

A. Learned → Sinusoidal → ALiBi → RoPE
B. Sinusoidal → Learned → T5 Bias → ALiBi → RoPE
C. RoPE → ALiBi → Sinusoidal → Learned
D. ALiBi → RoPE → Sinusoidal → Learned
Answer: B
Sinusoidal(2017) → Learned(2018, BERT) → T5 Bias(2020) → ALiBi(2021) → RoPE(2021). 절대 위치에서 상대 위치로, 입력 수정에서 attention 수정으로 진화.
절대→상대, 입력 수정→attention 수정으로의 진화.

📋 Cheat Sheet — Position Embeddings 한눈에

🔑 핵심 수식 3개

1. Sinusoidal PE:

\[ PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right) \]

2. ALiBi:

\[ \text{score}_{ij} = \frac{q_i^T k_j}{\sqrt{d_k}} + m \cdot (-|i-j|) \]

3. RoPE:

\[ \tilde{q}_m^T \tilde{k}_n = q_m^T R_{n-m} k_n \quad \text{(상대 위치에만 의존)} \]

🎯 시험 대비 핵심 포인트

  • Self-Attention은 set operation → 위치 정보 필요
  • Sinusoidal: sin/cos, 학습 없음, 길이 일반화 가능, 절대 위치
  • Learned: 학습 파라미터, L_max 제한, BERT/GPT-2
  • ALiBi: attention score에 선형 bias, "Train Short Test Long"
  • RoPE: Q/K 회전, 내적이 상대 위치에만 의존, long-term decay, 현재 표준
  • 진화: 절대→상대, 입력 수정→attention 수정
