Attention Approximation — 효율적 Attention의 세계

Longformer, Sliding Window, GQA, MQA: O(n²) 병목을 극복하는 전략들
Based on: Stanford CME 295 Lecture 2 (Afshine & Shervine Amidi)
Learning Objectives

이 라운드를 마치면 다음을 할 수 있다:

  • Self-Attention의 O(n²) 복잡도가 왜 문제인지 구체적으로 설명할 수 있다
  • Sparse Attention (Longformer)의 local + global 전략을 이해한다
  • Sliding Window Attention의 원리와 "receptive field" 개념을 설명할 수 있다
  • MHA, GQA, MQA의 차이를 그림으로 설명할 수 있다
  • GQA가 MHA와 MQA 사이의 trade-off인 이유를 이해한다
  • 각 방법이 어떤 모델에서 사용되는지 알고 있다

1. Self-Attention의 O(n²) 병목

1.1 문제의 본질

Lecture 1에서 배운 Self-Attention의 핵심 연산은 \(QK^T\)이다. Q와 K가 각각 \((n, d_k)\) 행렬일 때, \(QK^T\)는 \((n, n)\) 행렬이 된다. 이 행렬의 각 원소는 모든 토큰 쌍의 유사도를 나타낸다.

복잡도 분석:

  • 시간 복잡도: \(O(n^2 \cdot d_k)\) — 모든 토큰 쌍에 대해 내적 계산
  • 공간 복잡도: \(O(n^2)\) — attention weight 행렬 저장

시퀀스 길이 \(n\)이 2배가 되면 계산량은 4배, 메모리는 4배가 된다!

핵심 문제: 현대 LLM은 수만~수백만 토큰의 context window를 지원해야 한다. Full attention의 O(n²)은 이를 불가능하게 만든다. 해결책이 필요하다.

import numpy as np

# Full Attention의 메모리 계산
def attention_memory(n, n_heads=32, dtype_bytes=2):
    """Attention 행렬의 메모리 사용량 (bytes)"""
    # 각 head마다 (n, n) 행렬
    total = n_heads * n * n * dtype_bytes
    return total

# 시퀀스 길이별 메모리
for n in [512, 2048, 8192, 32768, 128000]:
    mem = attention_memory(n)
    print(f"n={n:>7,}: {mem / 1e9:.2f} GB")
# n=    512: 0.03 GB
# n=  2,048: 0.54 GB
# n=  8,192: 8.59 GB
# n= 32,768: 137.4 GB  ← GPU 메모리 초과!
# n=128,000: 2,097 GB  ← 불가능

1.2 해결 전략 개요

O(n²) 병목을 극복하는 두 가지 주요 전략이 있다:

전략 1: Sparse Attention — 어디를 볼 것인가

모든 토큰 쌍을 보지 않고, 중요한 쌍만 선택적으로 본다.

  • Longformer: Local window + Global tokens
  • Sliding Window Attention (SWA): 고정 크기 윈도우

전략 2: Head Sharing — K/V를 공유

Attention head 간에 Key/Value를 공유하여 메모리와 계산을 절약한다.

  • GQA (Grouped Query Attention): 그룹별 K/V 공유
  • MQA (Multi-Query Attention): 모든 head가 하나의 K/V 공유

2. Longformer — Sparse Attention의 시작

2.1 핵심 아이디어

Beltagy et al. (2020), "Longformer: The Long-Document Transformer". 모든 토큰이 모든 토큰을 볼 필요는 없다. 대부분의 attention은 가까운 토큰에 집중된다는 관찰에 기반한다.

Definition 2.1 — Longformer의 Attention 패턴

두 가지 attention을 결합한다:

  • Local Attention (Sliding Window): 각 토큰이 주변 \(w\)개 토큰만 본다. 복잡도: \(O(n \cdot w)\)
  • Global Attention: 특정 토큰(예: [CLS])이 모든 토큰을 본다. 복잡도: \(O(n \cdot g)\)

총 복잡도: \(O(n \cdot (w + g))\) — \(w, g \ll n\)이면 \(O(n)\)에 가까움!

2.2 Attention 패턴 시각화

import numpy as np

def longformer_mask(n, window_size, global_tokens):
    """
    Longformer attention mask 생성
    n: 시퀀스 길이
    window_size: local attention 윈도우 크기 (한쪽)
    global_tokens: global attention을 받는 토큰 인덱스 리스트
    """
    mask = np.zeros((n, n), dtype=np.bool_)
    
    # 1. Local attention (sliding window)
    for i in range(n):
        start = max(0, i - window_size)
        end = min(n, i + window_size + 1)
        mask[i, start:end] = True
    
    # 2. Global attention (특정 토큰이 모든 토큰을 봄)
    for g in global_tokens:
        mask[g, :] = True   # global 토큰 → 모든 토큰
        mask[:, g] = True   # 모든 토큰 → global 토큰
    
    return mask

# 예시: 10 토큰, window=2, [CLS]=global
mask = longformer_mask(n=10, window_size=2, global_tokens=[0])
print("Longformer Mask:")
print(mask.astype(int))

# Full attention: 10×10 = 100 connections
# Longformer:    mask.sum() connections
print(f"\nFull: {10*10} connections")
print(f"Longformer: {mask.sum()} connections")
print(f"절약: {(1 - mask.sum()/(10*10))*100:.0f}%")

왜 Global Token이 필요한가?

Local attention만으로는 먼 토큰 간의 정보 교환이 불가능하다. [CLS] 같은 global token이 "허브" 역할을 하여 전체 시퀀스의 정보를 집약한다. 분류 태스크에서 [CLS]의 임베딩을 사용하는 것과 같은 원리.

3. Sliding Window Attention (SWA) — Mistral의 선택

3.1 핵심 아이디어

Longformer의 local attention 부분을 더 체계적으로 발전시킨 것. Mistral 7B (2023)에서 핵심 기법으로 사용되었다.

Definition 3.1 — Sliding Window Attention

각 토큰이 고정 크기 윈도우 \(W\) 내의 토큰만 attend한다:

\[ \text{Attend}(i) = \{j : |i - j| \leq W/2\} \]

복잡도: \(O(n \cdot W)\) — \(W\)가 고정이면 \(n\)에 대해 선형!

3.2 Receptive Field — CNN과의 연결

SWA의 가장 중요한 통찰: 레이어를 쌓으면 receptive field가 확장된다. 이것은 CNN의 receptive field 개념과 정확히 동일하다.

Definition 3.2 — Receptive Field in SWA

윈도우 크기 \(W\), 레이어 수 \(L\)일 때:

\[ \text{Effective Receptive Field} = L \times W \]

각 레이어에서 \(W\)개 토큰을 보고, \(L\)개 레이어를 쌓으면 총 \(L \times W\)개 토큰의 정보가 전파된다.

Layer 1 (W=3): 각 토큰이 좌우 1개씩 봄 tok0 tok1 tok2 tok3 tok4 tok5 tok6 ├──┤ ├──┤ ├──┤ ├──┤ ├──┤ ├──┤ ├──┤ Layer 2 (W=3): Layer 1의 출력에서 다시 좌우 1개씩 → tok2는 tok0~tok4의 정보를 가짐! tok0 tok1 tok2 tok3 tok4 tok5 tok6 ├────────┤ ├────────┤ ├────────┤ Layer 3 (W=3): → tok3은 tok0~tok6 전체 정보! tok0 tok1 tok2 tok3 tok4 tok5 tok6 ├──────────────┤ ├──────────────┤ Receptive Field = L × W = 3 × 3 = 9 토큰
# Mistral 7B의 SWA 설정
W = 4096   # 윈도우 크기
L = 32     # 레이어 수

effective_rf = L * W
print(f"Window size: {W:,}")
print(f"Layers: {L}")
print(f"Effective receptive field: {effective_rf:,} tokens")
# Effective receptive field: 131,072 tokens
# → 128K 토큰의 정보를 O(n·W) 복잡도로 처리!

# 메모리 비교
n = 32768
full_mem = n * n  # O(n²)
swa_mem = n * W   # O(n·W)
print(f"\nFull attention: {full_mem:,} elements")
print(f"SWA (W={W}):    {swa_mem:,} elements")
print(f"절약: {(1 - swa_mem/full_mem)*100:.1f}%")
# 절약: 87.5%

강의 인용: "Similar to 'receptive field' in computer vision." — SWA에서 레이어를 쌓아 receptive field를 확장하는 것은 CNN에서 작은 커널을 여러 층 쌓아 넓은 영역을 보는 것과 동일한 원리다.

SWA의 변형: Interleaving

일부 모델은 local attention 레이어와 global attention 레이어를 번갈아 배치한다. 예를 들어 홀수 레이어는 SWA, 짝수 레이어는 full attention. 이렇게 하면 효율성과 전역 정보 접근을 모두 확보할 수 있다.

4. Attention Head 공유: MHA → GQA → MQA ⭐

4.1 문제: KV Cache의 메모리 부담

Autoregressive 추론에서 이전 토큰의 Key와 Value를 캐싱한다 (KV Cache). Multi-Head Attention에서는 각 head마다 별도의 K, V가 있으므로 캐시 크기가 head 수에 비례한다.

KV Cache 메모리:

\[ \text{KV Cache} = 2 \times L \times h \times n \times d_k \times \text{bytes} \]

여기서 \(L\)=레이어 수, \(h\)=head 수, \(n\)=시퀀스 길이, \(d_k\)=head 차원. 2는 K와 V 두 개.

예: LLaMA 7B (L=32, h=32, d_k=128, n=4096, FP16) → KV Cache = 2 GB!

4.2 Multi-Head Attention (MHA) — 기본

Definition 4.1 — MHA (Multi-Head Attention)

각 head가 독립적인 Q, K, V projection을 가진다:

  • Q heads: \(h\)개 (각각 독립적인 \(W_Q^i\))
  • K heads: \(h\)개 (각각 독립적인 \(W_K^i\))
  • V heads: \(h\)개 (각각 독립적인 \(W_V^i\))

총 KV 파라미터: \(2 \times h \times d_{model} \times d_k\)

4.3 Multi-Query Attention (MQA) — 극단적 공유

Definition 4.2 — MQA (Multi-Query Attention)

모든 Query head가 하나의 K, V를 공유한다:

  • Q heads: \(h\)개 (각각 독립적)
  • K heads: 1개 (모든 Q head가 공유)
  • V heads: 1개 (모든 Q head가 공유)

KV Cache 절약: \(h\)배 감소!

4.4 Grouped Query Attention (GQA) — 균형점

Definition 4.3 — GQA (Grouped Query Attention)

Query head를 \(G\)개 그룹으로 나누고, 각 그룹이 하나의 K, V를 공유:

  • Q heads: \(h\)개 (각각 독립적)
  • K heads: \(G\)개 (그룹당 1개)
  • V heads: \(G\)개 (그룹당 1개)
  • 각 그룹에 \(h/G\)개의 Q head가 속함

MHA (\(G=h\))와 MQA (\(G=1\))의 일반화!

import numpy as np

class GroupedQueryAttention:
    def __init__(self, d_model=512, n_heads=8, n_kv_heads=2):
        """
        GQA 구현
        n_heads: Query head 수
        n_kv_heads: KV head 수 (그룹 수)
        """
        self.n_heads = n_heads
        self.n_kv_heads = n_kv_heads
        self.heads_per_group = n_heads // n_kv_heads
        self.d_k = d_model // n_heads
        
        # Q: n_heads개의 독립적인 projection
        self.W_Q = [np.random.randn(d_model, self.d_k) 
                    for _ in range(n_heads)]
        
        # K, V: n_kv_heads개만! (그룹별 공유)
        self.W_K = [np.random.randn(d_model, self.d_k) 
                    for _ in range(n_kv_heads)]
        self.W_V = [np.random.randn(d_model, self.d_k) 
                    for _ in range(n_kv_heads)]
    
    def forward(self, X):
        heads = []
        for i in range(self.n_heads):
            # 각 Q head는 독립적
            Q = X @ self.W_Q[i]
            
            # K, V는 그룹 내 공유
            group_idx = i // self.heads_per_group
            K = X @ self.W_K[group_idx]  # 같은 그룹 = 같은 K
            V = X @ self.W_V[group_idx]  # 같은 그룹 = 같은 V
            
            # Scaled dot-product attention
            d_k = Q.shape[-1]
            scores = Q @ K.T / np.sqrt(d_k)
            weights = np.exp(scores) / np.exp(scores).sum(axis=-1, keepdims=True)
            head_out = weights @ V
            heads.append(head_out)
        
        return np.concatenate(heads, axis=-1)

# MHA: n_kv_heads = n_heads = 8
mha = GroupedQueryAttention(n_heads=8, n_kv_heads=8)
# GQA: n_kv_heads = 2 (4개 Q head가 1개 KV 공유)
gqa = GroupedQueryAttention(n_heads=8, n_kv_heads=2)
# MQA: n_kv_heads = 1 (모든 Q head가 1개 KV 공유)
mqa = GroupedQueryAttention(n_heads=8, n_kv_heads=1)

print(f"MHA KV params: {8 * 2} matrices")   # 16
print(f"GQA KV params: {2 * 2} matrices")   # 4 (75% 절약)
print(f"MQA KV params: {1 * 2} matrices")   # 2 (87.5% 절약)

GQA가 현재 가장 인기 있는 이유:

  • MHA 대비 KV Cache를 크게 절약 (보통 4~8배)
  • MQA보다 성능 하락이 적음 (다양성 유지)
  • 추론 속도 향상 (KV Cache 접근 감소)
  • LLaMA 2 (70B)에서 GQA 채택 후 사실상 표준이 됨

  • 이 조합으로 7B 파라미터로도 13B 모델에 필적하는 성능

Q1. Self-Attention의 시간 복잡도가 O(n²)인 이유는?

A. Softmax 계산이 O(n²)이기 때문
B. QK^T 계산에서 모든 토큰 쌍의 내적을 계산하기 때문
C. Value 행렬이 O(n²) 크기이기 때문
D. Positional encoding이 O(n²)이기 때문
Answer: B
QK^T는 (n, d_k) × (d_k, n) = (n, n) 행렬. 모든 토큰 쌍(n²개)에 대해 내적을 계산하므로 O(n² × d_k).
QK^T가 n×n 행렬 → 모든 토큰 쌍의 유사도 계산 → O(n²).

Q2. Longformer의 두 가지 attention 유형은?

A. Self-Attention + Cross-Attention
B. Local (Sliding Window) + Global Attention
C. Causal + Bidirectional Attention
D. Multi-Head + Single-Head Attention
Answer: B
Longformer: Local attention(주변 w개 토큰만 봄) + Global attention([CLS] 등 특정 토큰이 모든 토큰을 봄). 총 복잡도 O(n·(w+g)).
Local(주변만) + Global(특정 토큰이 전체) = Longformer.

Q3. Sliding Window Attention에서 receptive field를 확장하는 방법은?

A. 윈도우 크기를 키운다
B. 레이어를 쌓는다 (Effective RF = L × W)
C. Head 수를 늘린다
D. A와 B 모두
Answer: D
Effective Receptive Field = L × W. 윈도우 크기(W)를 키우거나 레이어(L)를 쌓으면 확장된다. CNN의 receptive field와 동일한 원리.
RF = L × W. 윈도우 키우기 또는 레이어 쌓기. CNN receptive field와 동일.

Q4. GQA에서 G=h이면 어떤 방법과 동일한가?

A. MQA
B. MHA (Multi-Head Attention)
C. Longformer
D. SWA
Answer: B
GQA에서 G=h이면 각 그룹에 Q head 1개 = 각 head가 독립적인 KV를 가짐 = MHA. G=1이면 MQA. GQA는 MHA와 MQA의 일반화.
G=h → MHA, G=1 → MQA. GQA는 둘의 일반화.

Q5. MQA의 장점과 단점은?

A. 장점: 성능 최고, 단점: 메모리 많이 사용
B. 장점: KV Cache 최소, 추론 가장 빠름. 단점: 성능 약간 하락
C. 장점: 학습 빠름, 단점: 추론 느림
D. 장점: 구현 간단, 단점: GPU 활용 어려움
Answer: B
MQA: 모든 Q head가 1개 KV 공유 → KV Cache h배 절약, 추론 가장 빠름. 하지만 KV의 다양성이 줄어 성능이 약간 하락할 수 있음.
KV Cache 최소(1/h) + 추론 최고속. 대신 성능 약간 하락.

Q6. SWA가 CNN의 어떤 개념과 유사한가?

A. Pooling
B. Receptive field
C. Batch normalization
D. Skip connection
Answer: B
"Similar to 'receptive field' in computer vision." SWA에서 레이어를 쌓아 정보 범위를 확장하는 것은 CNN에서 작은 커널을 여러 층 쌓아 넓은 영역을 보는 것과 동일.

Q7. KV Cache가 필요한 이유는?

A. 학습 속도를 높이기 위해
B. Autoregressive 추론에서 이전 토큰의 K, V를 재계산하지 않기 위해
C. 모델 파라미터를 줄이기 위해
D. Gradient를 저장하기 위해
Answer: B
Autoregressive 추론에서 새 토큰 생성 시 이전 토큰의 K, V를 매번 재계산하면 비효율적. 캐싱하면 새 토큰의 Q만 계산하면 됨. 하지만 캐시 크기가 문제.
이전 토큰 K, V 재계산 방지. 추론 효율화의 핵심.

Q8. Longformer에서 Global token의 역할은?

A. 학습 속도를 높인다
B. 먼 토큰 간 정보 교환의 "허브" 역할
C. 메모리를 절약한다
D. Gradient flow를 개선한다
Answer: B
Local attention만으로는 먼 토큰 간 정보 교환 불가. [CLS] 같은 global token이 모든 토큰을 보고, 모든 토큰이 global token을 봄 → 전체 정보 집약의 허브.
Global token = 정보 허브. 먼 토큰 간 간접적 정보 교환 가능.

Q9. Mistral 7B가 사용하는 효율적 Attention 조합은?

A. Full Attention + MHA
B. Longformer + MQA
C. Sliding Window Attention + GQA
D. Sparse Attention + Batch Norm
Answer: C
Mistral 7B: SWA(W=4096)로 긴 시퀀스 처리 + GQA로 KV Cache 절약. 이 조합으로 7B 파라미터로 13B 모델에 필적하는 성능.
SWA + GQA = Mistral의 핵심 조합. 효율성과 성능의 균형.

Q10. 시퀀스 길이가 2배가 되면 Full Attention의 메모리는?

A. 2배
B. 4배
C. 8배
D. 변화 없음
Answer: B
Attention 행렬은 (n, n). n이 2배 → n² = 4배. 시간 복잡도도 4배. 이것이 O(n²)의 의미이며, 긴 시퀀스에서 치명적인 이유.
O(n²): n이 2배 → 메모리/계산 4배. 이것이 효율적 attention이 필요한 이유.

📋 Cheat Sheet — Attention Approximation 한눈에

🔑 핵심 수식 & 복잡도

1. Full Attention:

\[ \text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V, \quad O(n^2 \cdot d_k) \]

2. Longformer:

\[ O(n \cdot (w + g)) \approx O(n) \quad \text{where } w = \text{window}, \; g = \text{global tokens} \]

3. SWA Receptive Field:

\[ \text{Effective RF} = L \times W \quad \text{(layers × window size)} \]

4. KV Cache (MHA):

\[ \text{KV Cache} = 2 \times L \times h \times n \times d_k \times \text{bytes} \]

5. GQA KV Cache 절약:

\[ \text{GQA ratio} = \frac{g}{h}, \quad \text{MQA} = \frac{1}{h} \quad \text{(g = KV groups)} \]

🎯 시험 대비 핵심 포인트

  • Full Attention: O(n²) — n이 2배 → 메모리/계산 4배
  • Longformer: Local window + Global tokens → O(n). [CLS]가 정보 허브 역할
  • SWA: 고정 윈도우 W, 레이어 쌓으면 RF 확장 (CNN과 동일 원리). Mistral 7B: W=4096
  • MHA: 각 head 독립 K/V → KV Cache 큼
  • GQA: h개 Q head를 g개 그룹으로 묶어 K/V 공유 → KV Cache g/h로 축소. LLaMA 2 70B, Mistral
  • MQA: 모든 head가 1개 K/V 공유 → KV Cache 1/h. 빠르지만 품질 저하 가능
  • Mistral 7B = SWA + GQA 조합 → 효율성과 성능의 균형
  • Sparse Attention은 "어디를 볼 것인가", Head Sharing은 "K/V를 어떻게 공유할 것인가"
