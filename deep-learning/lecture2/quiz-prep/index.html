📚 LECTURE 1 — NLP 기초 · Tokenization · Word2Vec · RNN/LSTM (Q1-10)

Q1. Perplexity의 특징으로 올바른 것은? Lec1

A. Ground truth가 반드시 필요하다
B. Ground truth 없이 계산 가능하며, 낮을수록 좋다
C. 높을수록 좋은 모델이다
D. 번역 품질만 측정한다
정답: B
PPL = exp(-1/N Σ log P(w_i|w_{<i})). 모델이 다음 단어에 높은 확률을 부여할수록 PPL이 낮아진다. Ground truth 없이 계산 가능해서 실용적.
BLEU/ROUGE는 ground truth 필요. Perplexity는 모델 자체의 확률로 계산.

Q2. Subword 토큰화(BPE)의 장점은? Lec1

A. OOV 문제를 완전히 해결한다
B. 어근/접사 공유로 효율적이며 OOV를 크게 줄인다
C. 시퀀스 길이가 가장 짧다
D. 구현이 가장 간단하다
정답: B
"reading" = "read" + "ing" → 어근 공유. OOV를 완전히 해결하지는 않지만 크게 줄인다. 현대 LLM의 표준 토큰화 방식.
Word-level은 OOV 심각, Character-level은 시퀀스 너무 길어짐. Subword가 균형점.

Q3. Word2Vec에서 Skip-gram과 CBOW의 차이는? Lec1

A. Skip-gram이 항상 더 빠르다
B. Skip-gram: 중심→주변 예측, CBOW: 주변→중심 예측
C. CBOW가 항상 더 정확하다
D. 차이가 없다
정답: B
Skip-gram: 중심 단어로 주변 단어 예측. CBOW: 주변 단어로 중심 단어 예측. Skip-gram이 희귀 단어에 더 좋은 경향.
둘 다 proxy task로 학습. 결과물은 embedding matrix.

Q4. One-hot encoding의 근본적 문제는? Lec1

A. 차원이 너무 작다
B. 학습이 불가능하다
C. 모든 단어가 직교하여 의미적 유사성을 표현할 수 없다
D. 계산이 너무 느리다
정답: C
One-hot에서 모든 단어 쌍의 dot product = 0. "cat"과 "dog"의 유사성을 표현할 수 없다. → Word Embedding이 필요한 이유.
Word2Vec이 이 문제를 해결: 의미적으로 유사한 단어 → 가까운 벡터.

Q5. RNN의 주요 문제점으로 올바른 것은? Lec1

A. 병렬화가 불가능하다
B. Vanishing/Exploding gradient 문제
C. Long-range dependency 학습이 어렵다
D. 위의 모두
정답: D
RNN: (1) 순차 처리라 병렬화 불가, (2) 긴 시퀀스에서 gradient 소실/폭발, (3) 먼 토큰 간 관계 학습 어려움. LSTM이 gradient 문제를 완화하지만 완전히 해결하지는 못함.
이 세 가지 문제가 Transformer 탄생의 동기.

Q6. LSTM이 vanilla RNN보다 나은 핵심 이유는? Lec1

A. 계산이 더 빠르다
B. Cell state가 "고속도로" 역할을 하여 gradient가 직접 흐를 수 있다
C. 파라미터가 더 적다
D. 병렬화가 가능하다
정답: B
LSTM의 Cell state C_t는 C_{t-1}에서 직접 연결 (hidden state를 거치지 않음). 이 "고속도로" 덕분에 gradient가 멀리까지 전파 가능.
C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t. Forget gate가 gradient flow를 제어.

Q7. BLEU score의 특징은? Lec1

A. Recall 기반 지표이다
B. Ground truth 없이 계산 가능하다
C. Precision 기반이며, 생성 텍스트의 n-gram이 참조 텍스트에 얼마나 있는지 측정
D. 분류 태스크에 사용된다
정답: C
BLEU = Precision 기반 (생성→참조). ROUGE = Recall 기반 (참조→생성). 둘 다 ground truth 필요.
BLEU는 번역 품질 평가의 표준. Brevity penalty로 짧은 번역에 페널티.

Q8. Character-level 토큰화의 단점은? Lec1

A. OOV 문제가 심각하다
B. 시퀀스가 길어져 O(n²) 비용 증가, 의미 해석 어려움
C. 구현이 불가능하다
D. 학습이 불가능하다
정답: B
"World Health Organization" = 24개 토큰. 시퀀스 길어짐 → O(n²) attention 비용 증가. 글자 "D" 하나로는 의미 해석 어려움.
장점: OOV 없음, 오타에 강함. 단점: 길이, 의미 해석.

Q9. Seq2Seq의 한계로 Transformer가 등장한 이유는? Lec1

A. 모든 입력이 하나의 context vector에 압축되어 정보 손실
B. 순차 처리라 병렬화 불가
C. Long-range dependency 문제
D. 위의 모두
정답: D
Seq2Seq (RNN 기반): 정보 병목, 순차 처리, long-range 문제. Attention → Transformer로 이어지는 발전의 동기.
Transformer: 병렬화 가능 + 직접 연결로 long-range 해결 + 정보 병목 없음.

Q10. Transformer가 탄생한 실제 배경은? Lec1

A. GPU가 너무 비싸서
B. Google의 TPU가 RNN 아키텍처를 지원하지 않아서
C. RNN이 너무 느려서
D. CNN이 NLP에 적합하지 않아서
정답: B
강의 인용: "Google built TPU in 2016, but TPU cannot support RNN. So NLP researchers decided to make their own architecture that can be supported by TPU. That is Transformer."
자원의 제약이 기술 혁신을 이끈 사례.
⭐ LECTURE 1 — Self-Attention · Multi-Head Attention (Q11-20)

Q11. Self-Attention에서 √d_k로 나누는 이유는? Lec1

A. 계산 속도를 높이기 위해
B. d_k가 크면 dot product의 분산이 d_k에 비례하여 커지고, softmax가 극단적이 되어 gradient가 소실되므로
C. 메모리를 절약하기 위해
D. Gradient vanishing을 방지하기 위해
정답: B
q, k가 평균 0, 분산 1이면 Var(q·k) = d_k. 분산이 커지면 softmax 입력이 극단적 → gradient 매우 작아짐 (softmax saturation). √d_k로 나눠서 분산을 1로 정규화.
D도 관련 있지만, 핵심 원인은 "분산이 d_k에 비례"하는 것.

Q12. Multi-Head Attention의 목적은? Lec1

A. 계산 속도 향상
B. 메모리 절약
C. 각 head가 다른 sub-task를 학습 (CNN의 multiple filters와 유사)
D. Overfitting 방지
정답: C
CNN의 multiple filters와 유사한 아이디어. 각 head가 문법, 의미, 위치 관계 등 다른 패턴을 학습. 실험적으로 관찰됨.
h개 head의 출력을 concat → W_O로 projection. d_k = d_model / h.

Q13. Attention(Q,K,V)에서 V의 역할은? Lec1

A. 유사도를 계산한다
B. 가중치를 계산한다
C. 실제 정보를 담고 있으며, attention weight로 가중 평균된다
D. 위치 정보를 제공한다
정답: C
Q와 K로 attention weight(중요도) 계산. 이 weight로 V를 가중 평균. V가 실제 전달되는 정보.
비유: Q=검색어, K=책 제목, V=책 내용. 검색어와 제목으로 관련도 계산, 내용을 가중 평균.

Q14. Cross-Attention에서 Q, K, V의 출처는? Lec1

A. Q: Encoder, K/V: Decoder
B. Q: Decoder, K/V: Encoder 출력
C. Q/K/V 모두 Encoder에서
D. Q/K/V 모두 Decoder에서
정답: B
Cross-Attention: Query는 Decoder에서, Key와 Value는 Encoder 출력에서. Decoder가 "질문"하고 Encoder가 "답변"하는 구조.
Decoder block에는 2개 attention: (1) Masked Self-Attn, (2) Cross-Attn.

Q15. Self-Attention의 시간 복잡도는? Lec1

A. O(n)
B. O(n log n)
C. O(n² · d_k)
D. O(n³)
정답: C
QK^T 계산: (n, d_k) × (d_k, n) = O(n² · d_k). 모든 토큰 쌍의 attention 계산. 시퀀스 길이의 제곱에 비례.
n이 2배 → 계산량 4배. 긴 시퀀스에서 병목. Lecture 2 Round 3에서 해결책 다룸.

Q16. d_k = d_model / h 인 이유는? Lec1

A. 계산 효율성을 위해
B. h개 head를 concat하면 h × d_k = d_model이 되어 원래 차원을 유지하기 위해
C. Gradient flow 개선을 위해
D. 메모리 절약을 위해
정답: B
MultiHead = Concat(head_1, ..., head_h) × W_O. 각 head 출력이 d_k 차원이므로 concat하면 h × d_k. 이것이 d_model과 같아야 W_O로 projection 가능.
Original Transformer: d_model=512, h=8, d_k=64.

Q17. Original Transformer의 하이퍼파라미터 중 틀린 것은? Lec1

A. N (layers) = 6
B. h (heads) = 8
C. d_model = 512
D. d_ff = 1024
정답: D
d_ff = 2048이 맞다. N=6, h=8, d_model=512, d_k=d_v=64, d_ff=2048.
FFN: 512 → 2048 → 512. d_ff는 d_model의 4배.

Q18. Softmax의 역할은? Lec1

A. 비선형성 추가
B. 확률 분포로 변환 (합이 1)
C. 정규화
D. B와 C 모두
정답: D
softmax(x_i) = exp(x_i) / Σexp(x_j). 모든 값을 양수로 만들고 합이 1이 되게 정규화 → 확률 분포.
Attention에서: 각 Query에 대해 모든 Key의 중요도를 확률로 변환.

Q19. Self-Attention이 RNN보다 나은 핵심 이유는? Lec1

A. 모든 토큰 쌍을 직접 연결하여 long-range dependency를 해결하고 병렬화 가능
B. 파라미터가 더 적다
C. 메모리를 덜 사용한다
D. 구현이 더 간단하다
정답: A
강의 인용: "We want to connect the output token to all input tokens directly." 모든 토큰 쌍을 직접 연결 → long-range dependency 해결. 또한 행렬 연산으로 병렬화 가능.
RNN: 순차 처리, 먼 토큰 간 간접 연결. Transformer: 병렬 처리, 직접 연결.

Q20. "Context-aware embedding"이란? Lec1

A. 위치 정보가 포함된 embedding
B. Self-Attention 후 주변 토큰 정보가 반영된 embedding
C. Word2Vec embedding
D. One-hot encoding
정답: B
입력 embedding은 position-aware (PE 추가). Self-Attention 후에는 다른 토큰의 정보가 반영되어 context-aware가 된다.
"bank"의 의미가 주변 문맥에 따라 달라지는 것을 표현 가능.
🏗️ LECTURE 1 — Transformer 구조 · Training Tricks (Q21-28)

Q21. Transformer에서 Positional Encoding이 필요한 이유는? Lec1

A. Self-Attention은 순서 정보를 모르므로 "I love you"와 "You love I"가 같은 결과
B. 계산 효율성을 위해
C. Gradient flow 개선을 위해
D. 메모리 절약을 위해
정답: A
Self-Attention은 set operation — 순서를 모른다. PE 없으면 "I love you"와 "You love I"가 동일한 결과. 위치 정보를 주입해야 한다.
Original Transformer: Sinusoidal PE. BERT: Learned PE. 현대: RoPE.

Q22. Feed-Forward Network의 구조는? Lec1

A. Linear → Linear
B. Linear → ReLU → Linear (512 → 2048 → 512)
C. Conv → ReLU → Conv
D. Attention → Linear
정답: B
FFN(x) = ReLU(xW_1 + b_1)W_2 + b_2. d_model(512) → d_ff(2048) → d_model(512). 중간에 ReLU (또는 GELU) 활성화.
각 토큰에 독립적으로 적용 (position-wise).

Q23. Teacher Forcing의 장점은? Lec1

A. 추론 속도가 빨라진다
B. 병렬 학습이 가능하다
C. 이전 예측의 에러가 전파되지 않는다
D. B와 C 모두
정답: D
학습 시 ground truth를 입력으로 사용 → (1) 모든 위치를 동시에 학습 가능 (병렬화), (2) 이전 예측의 에러가 다음 스텝에 전파되지 않음.
추론 시에는 autoregressive (순차 생성). 학습과 추론의 차이를 이해하는 것이 중요.

Q24. Causal Masking의 목적은? Lec1

A. 계산 속도 향상
B. Teacher Forcing 시 미래 토큰을 보지 못하게 하여 cheating 방지
C. 메모리 절약
D. Gradient flow 개선
정답: B
미래 토큰을 보면 cheating. Mask로 미래 위치의 attention score를 -∞로 설정 → softmax 후 0이 됨.
하삼각 행렬 형태. 위치 i는 위치 j ≤ i만 attend 가능.

Q25. Label Smoothing (ε=0.1)에서 정답 클래스의 target 값은? Lec1

A. 1.0
B. 약 0.9 (정확히는 (1-ε) + ε/K)
C. 0.1
D. 0.5
정답: B
y_smooth = (1-ε)·y_one-hot + ε/K. 정답 클래스: ≈0.9, 나머지: ≈0.02. 모델의 과신(overconfidence) 방지, 일반화 향상.
GoogLeNet(2015)에서 처음 소개. Transformer에서 BLEU score 향상에 기여.

Q26. Residual Connection의 효과는? Lec1

A. 계산 속도 향상
B. Gradient가 직접 흐를 수 있어 깊은 네트워크 학습 가능 (ResNet에서 차용)
C. 메모리 절약
D. Attention 성능 향상
정답: B
Output = LayerNorm(x + SubLayer(x)). x + SubLayer(x) 형태로 gradient가 직접 흐를 수 있다. ResNet에서 가져온 아이디어.
강의: "Many design choices are influenced by computer vision architectures."

Q27. Decoder block에 있는 attention 메커니즘의 개수는? Lec1

A. 1개
B. 2개 (Masked Self-Attention + Cross-Attention)
C. 3개
D. 0개
정답: B
Decoder: (1) Masked Multi-Head Self-Attention (Causal), (2) Cross-Attention (Q: decoder, K/V: encoder). 그 후 FFN.
Encoder는 Self-Attention 1개만. Decoder는 2개.

Q28. Autoregressive inference의 특징은? Lec1

A. 병렬 생성이 가능하다
B. 이전 출력이 다음 입력이 되며, 한 토큰씩 순차 생성
C. Ground truth를 사용한다
D. 에러가 전파되지 않는다
정답: B
이전 스텝의 예측이 다음 스텝의 입력. 한 토큰씩 순차 생성. EOS 토큰이 나올 때까지 반복. 에러가 누적될 수 있음.
Teacher Forcing(학습)과 Autoregressive(추론)의 차이를 이해하는 것이 핵심.
📐 LECTURE 2 — Position Embeddings: Sinusoidal, Learned, ALiBi, RoPE (Q29-34)

Q29. Sinusoidal PE에서 PE_{pos+k}를 PE_{pos}로부터 얻을 수 있는 이유는? Lec2

A. 선형 보간이 가능하기 때문
B. 삼각함수 덧셈 정리에 의해 회전 행렬(선형 변환)로 표현 가능
C. 학습된 변환 행렬이 있기 때문
D. 모든 PE가 같은 값이기 때문
정답: B
sin(ω(pos+k)) = cos(ωk)sin(ωpos) + sin(ωk)cos(ωpos). 삼각함수 덧셈 정리에 의해 PE_{pos+k}는 PE_{pos}에 k에만 의존하는 회전 행렬을 곱한 것.
이 성질 덕분에 모델이 상대적 위치를 학습할 수 있다.

Q30. Learned Positional Embedding의 한계는? Lec2

A. 학습이 불가능하다
B. 계산이 너무 느리다
C. 학습 시 본 최대 길이(L_max)를 초과하는 시퀀스에 일반화 불가
D. 상대적 위치를 표현할 수 없다
정답: C
Learned PE는 위치별 embedding을 학습. L_max까지만 학습하므로 그 이상의 위치에는 embedding이 없다. BERT(L_max=512), GPT-2(L_max=1024).
Sinusoidal PE는 임의 길이에 일반화 가능. 이것이 Learned PE의 핵심 한계.

Q31. ALiBi의 핵심 아이디어는? Lec2

A. 입력 embedding에 위치 벡터를 더한다
B. Attention score에 거리에 비례하는 선형 bias를 빼서 가까운 토큰을 선호하게 한다
C. Q와 K를 회전시킨다
D. 위치별 학습 파라미터를 사용한다
정답: B
ALiBi: score_ij = (q_i^T k_j)/√d_k + m·(-|i-j|). 거리가 멀수록 큰 음수 bias → 가까운 토큰에 더 높은 attention. "Train Short, Test Long" 가능.
입력을 수정하지 않고 attention score를 수정. 학습 파라미터 없음 (m은 고정).

Q32. ALiBi에서 head별 slope(m)의 역할은? Lec2

A. 모든 head가 같은 범위를 본다
B. 큰 slope → 가까운 토큰에 집중, 작은 slope → 먼 토큰도 참조
C. Slope는 학습되는 파라미터다
D. Slope는 attention weight를 정규화한다
정답: B
큰 slope(m=1/2): steep → 가까운 토큰에 강하게 집중 (local). 작은 slope(m=1/256): gentle → 먼 토큰도 참조 (global). Multi-Head의 다양성과 유사.
Slope는 고정값 (학습 안 함). 기하급수적으로 감소: 2^(-8/h), ..., 2^(-1).

Q33. RoPE의 핵심 성질은? Lec2

A. Attention score에 bias를 더한다
B. 위치별 embedding을 학습한다
C. Q와 K를 회전시켜 내적이 상대 위치에만 의존하게 한다
D. 입력 embedding에 sin/cos를 더한다
정답: C
RoPE: q̃_m = R(θ_m)·q_m, k̃_n = R(θ_n)·k_n. 내적 q̃_m^T k̃_n = q_m^T R(θ_{n-m}) k_n → 상대 위치 (n-m)에만 의존. 현재 LLM의 표준 (LLaMA, Mistral).
Long-term decay: 먼 토큰일수록 attention score 상한이 자연스럽게 감소.

Q34. Position Embedding의 진화 순서로 올바른 것은? Lec2

A. Learned → Sinusoidal → ALiBi → RoPE
B. Sinusoidal(2017) → Learned(2018) → T5 Bias(2020) → ALiBi(2021) → RoPE(2021)
C. RoPE → ALiBi → Sinusoidal → Learned
D. ALiBi → RoPE → Sinusoidal → Learned
정답: B
절대 위치(Sinusoidal, Learned) → 상대 위치(T5 Bias, ALiBi, RoPE). 입력 수정 → attention 수정으로 진화. RoPE가 현재 표준.
Sinusoidal: Transformer. Learned: BERT/GPT-2. ALiBi: BLOOM. RoPE: LLaMA/Mistral.
🔧 LECTURE 2 — Layer Normalization · Pre-Norm · RMSNorm (Q35-40)

Q35. Layer Norm과 Batch Norm의 핵심 차이는? Lec2

A. Layer Norm이 항상 더 좋다
B. Batch Norm이 더 빠르다
C. Layer Norm은 feature 축, Batch Norm은 batch 축 기준으로 정규화
D. 차이가 없다
정답: C
Batch Norm: 같은 feature의 모든 배치 샘플로 μ, σ 계산 (batch 축). Layer Norm: 한 샘플 내 모든 feature로 μ, σ 계산 (feature 축).
Layer Norm: batch size에 독립, 가변 길이 자연스럽게 처리, 추론 시 running stats 불필요.

Q36. Transformer에서 Layer Norm을 사용하는 이유는? Lec2

A. Layer Norm이 항상 Batch Norm보다 우수하다
B. 원 논문이 Layer Norm으로 튜닝되었고, 가변 길이/작은 배치에 적합
C. 계산이 더 빠르다
D. 메모리를 덜 사용한다
정답: B
강의 인용: "The only reason to use Layer Normalization is that the original paper was tuning based on Layer Norm." 추가로: 가변 길이 시퀀스 처리, 작은 배치에서도 안정적, 추론 시 running stats 불필요.
Batch Norm을 쓰려면 하이퍼파라미터를 다시 튜닝해야 한다.

Q37. Post-Norm과 Pre-Norm의 차이는? Lec2

A. 성능 차이가 없다
B. Post-Norm이 항상 더 좋다
C. Post-Norm: LN(x + SubLayer(x)), Pre-Norm: x + SubLayer(LN(x)). Pre-Norm이 gradient flow가 더 좋아 학습이 안정적
D. Pre-Norm은 더 많은 파라미터가 필요하다
정답: C
Post-Norm: residual 후 LN → LN이 gradient path를 차단. Pre-Norm: LN 후 SubLayer → residual path가 깨끗하게 유지 → gradient 직접 흐름.
현대 LLM은 거의 모두 Pre-Norm. GPT-2+, LLaMA, Mistral 등.

Q38. RMSNorm이 Layer Norm과 다른 점은? Lec2

A. 분산을 계산하지 않는다
B. 평균을 빼지 않고(re-centering 없음) RMS로만 정규화하며, β 파라미터도 없다
C. 학습 파라미터가 없다
D. Batch 축으로 정규화한다
정답: B
RMSNorm(x) = γ ⊙ x/RMS(x). Layer Norm에서 평균 빼기(re-centering)와 β(shift)를 제거. 성능은 동등하면서 계산이 더 빠름.
RMS(x) = √(1/d Σx_i²). 학습 파라미터는 γ만 (d개). Layer Norm은 γ+β (2d개).

Q39. 현대 LLM (LLaMA, Mistral)이 사용하는 정규화 조합은? Lec2

A. Post-Norm + Batch Norm
B. Post-Norm + Layer Norm
C. Pre-Norm + RMSNorm
D. Pre-Norm + Batch Norm
정답: C
현대 LLM의 표준: Pre-Norm (학습 안정성) + RMSNorm (효율성). LLaMA, Mistral, PaLM 등이 이 조합 사용.
진화: Post-Norm+LayerNorm(2017) → Pre-Norm+LayerNorm(GPT-2) → Pre-Norm+RMSNorm(LLaMA).

Q40. Layer Normalization의 학습 가능한 파라미터 γ와 β의 역할은? Lec2

A. Gradient를 안정화한다
B. 정규화 후 표현력을 복원한다 (γ=σ, β=μ로 학습하면 원래 분포로 되돌릴 수 있음)
C. 학습 속도를 높인다
D. 메모리를 절약한다
정답: B
단순 정규화만 하면 표현력이 줄어든다. γ(scale)와 β(shift)로 모델이 필요하면 정규화를 "되돌릴" 수 있다. γ=σ, β=μ로 학습하면 원래 분포 복원.
γ 초기값=1, β 초기값=0. 학습을 통해 최적의 scale/shift를 찾는다.

4. Label Smoothing:

🔑 Lecture 2 핵심 수식

5. Sinusoidal PE:

\[ PE_{(pos,2i)} = \sin\!\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos,2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d}}\right) \]

6. ALiBi:

\[ \text{score}_{ij} = \frac{q_i^Tk_j}{\sqrt{d_k}} + m\cdot(-|i-j|) \]

7. RoPE:

\[ \tilde{q}_m^T\tilde{k}_n = q_m^T R_{n-m} k_n \quad \text{(상대 위치에만 의존)} \]

8. Layer Norm:

\[ \text{LN}(x) = \gamma \odot \frac{x-\mu}{\sqrt{\sigma^2+\epsilon}} + \beta \]

9. RMSNorm:

\[ \text{RMSNorm}(x) = \gamma \odot \frac{x}{\text{RMS}(x)}, \quad \text{RMS}(x) = \sqrt{\frac{1}{d}\sum x_i^2} \]

🎯 시험 핵심 키워드 (빈출)

  • √d_k: Var(q·k) = d_k → softmax saturation 방지
  • Multi-Head: CNN의 multiple filters와 유사, 각 head가 다른 sub-task
  • Teacher Forcing: 학습 시 ground truth 사용 → 병렬화 + 에러 전파 없음
  • Causal Mask: 미래 토큰 차단, -∞ → softmax 후 0
  • Label Smoothing: 과신 방지, ε=0.1 → 정답 0.9
  • Residual Connection: ResNet에서 차용, gradient 직접 흐름
  • PE 진화: Sinusoidal → Learned → ALiBi → RoPE (절대→상대)
  • ALiBi: attention score에 선형 bias, "Train Short Test Long"
  • RoPE: Q/K 회전, 내적이 상대 위치에만 의존, 현재 표준
  • Layer Norm vs Batch Norm: feature 축 vs batch 축
  • Post-Norm vs Pre-Norm: Pre-Norm이 gradient flow 좋음, 현재 표준
  • RMSNorm: 평균 안 빼고 RMS만, β 없음, 더 효율적
  • 현대 LLM: Pre-Norm + RMSNorm + RoPE + GQA

⚠️ 자주 틀리는 포인트

  • d_ff = 2048 (1024 아님!). d_model의 4배
  • BLEU = Precision 기반, ROUGE = Recall 기반 (반대로 외우기 쉬움)
  • Perplexity는 낮을수록 좋다 (높을수록 아님!)
  • Cross-Attention: Q는 Decoder, K/V는 Encoder (반대 아님!)
  • ALiBi의 slope는 학습 안 함 (고정값)
  • RMSNorm에는 β(shift) 파라미터가 없다
  • Transformer 탄생 이유: TPU가 RNN을 지원 안 해서 (GPU 비용 아님!)
