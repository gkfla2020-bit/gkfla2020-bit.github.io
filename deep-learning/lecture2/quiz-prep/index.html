📐 Review — Position Embeddings 핵심 정리

🎯 시험 출제 패턴 분석

실제 시험은 "왜 필요한가?", "핵심 장점/한계는?", "방법 간 차이는?" 패턴으로 출제된다. 단순 수식 암기보다 각 방법의 motivation과 trade-off를 이해하는 것이 중요.

1. 왜 Position Embedding이 필요한가?

Self-Attention은 set operation이다. 입력 토큰의 순서를 바꿔도 출력이 같다 (permutation equivariant). "I love you"와 "You love I"가 동일한 결과를 낸다. 따라서 토큰의 위치 정보를 별도로 주입해야 한다.

핵심: Self-Attention은 순서를 모른다 → Position Embedding으로 순서 정보 주입

2. 다섯 가지 방법 비교

⚠️ 시험에서 헷갈리기 쉬운 포인트:

  • ALiBi vs RoPE: 둘 다 attention을 수정하지만 방법이 다름!
    • ALiBi = attention score에 linear bias를 더함 (이름 자체가 Attention with Linear Biases)
    • RoPE = Q, K 벡터를 rotate (이름 자체가 Rotary Position Embedding)
  • Sinusoidal의 장점: (1) 학습 불필요 (hardcoded), (2) 임의 길이 일반화, (3) 상대 위치를 선형 변환(회전 행렬)으로 표현 가능 — "phase differences"로 상대 위치 인코딩
  • Learned의 한계: L_max를 초과하는 시퀀스에 일반화 불가 — 이것이 핵심 한계

3. 절대 → 상대 위치의 진화

절대 위치(Sinusoidal, Learned)는 각 토큰에 고정된 위치 벡터를 부여한다. 상대 위치(ALiBi, RoPE)는 토큰 간의 거리에 기반한다. 상대 위치가 더 자연스러운 이유: 자연어에서 중요한 것은 "토큰 A가 위치 5에 있다"가 아니라 "토큰 A와 B가 3칸 떨어져 있다"이기 때문이다.

진화 방향: 절대 위치 → 상대 위치, 입력 수정 → attention 수정

Sinusoidal(2017) → Learned(2018) → T5 Bias(2020) → ALiBi(2021) → RoPE(2021, 현재 표준)

🔧 Review — Layer Normalization 핵심 정리

4. Layer Norm vs Batch Norm — 정규화 축의 차이

Batch Norm: 같은 feature의 모든 배치 샘플로 μ, σ 계산 → batch 축 정규화

Layer Norm: 한 샘플 내 모든 feature로 μ, σ 계산 → feature 축 정규화

\[ \text{LayerNorm}(x) = \gamma \odot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta \]

Transformer에서 Layer Norm을 쓰는 이유: (1) batch size에 독립적, (2) 가변 길이 시퀀스 자연스럽게 처리, (3) 추론 시 running statistics 불필요, (4) 원 논문이 LN으로 튜닝됨.

Layer Norm의 핵심 기여: "Improve training stability and convergence" — 학습 안정성과 수렴 속도 향상. 이것 없이는 깊은 Transformer 학습이 매우 어렵다.

5. Post-Norm vs Pre-Norm

Table 2. Post-Norm vs Pre-Norm
특성Post-Norm (Original)Pre-Norm (현재 표준)
수식LN(x + SubLayer(x))x + SubLayer(LN(x))
LN 위치Residual 뒤SubLayer 앞
Gradient flowLN이 residual path 차단Residual path 깨끗 유지

시험 포인트: Pre-Norm이 더 안정적인 이유 = residual path가 깨끗하게 유지되어 gradient가 직접 흐를 수 있기 때문. ResNet의 skip connection과 같은 원리.

6. RMSNorm — Layer Norm의 단순화

RMSNorm: Layer Norm에서 평균 빼기(re-centering)를 제거. RMS로만 정규화.

\[ \text{RMSNorm}(x) = \gamma \odot \frac{x}{\text{RMS}(x)}, \quad \text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2} \]

차이: (1) 평균 μ 계산 안 함, (2) shift 파라미터 β 없음, (3) 학습 파라미터 γ만 (d개 vs LN의 2d개)

성능은 동등하면서 계산이 더 빠름 → 현대 LLM의 표준

⚠️ 틀리기 쉬운 포인트 — 시험 직전 체크

🚨 네가 틀린 문제 복습

Q6. "Which method introduces a linear bias in attention scores?"

네 답: ① RoPE → ❌

정답: ② ALiBi

기억법: ALiBi = Attention with Linear Biases. 이름에 "linear bias"가 들어있다. RoPE는 "rotate".

혼동하기 쉬운 쌍:

  • "linear bias in attention scores" → ALiBi (score에 m·(-|i-j|) 더함)
  • "rotate query and key vectors" → RoPE (Q, K를 회전)
  • "hardcoded, generalizes to arbitrary lengths" → Sinusoidal PE
  • "cannot generalize beyond training length" → Learned PE
  • "encode relative positions through phase differences" → Sinusoidal (삼각함수 덧셈 정리)
  • "normalizes across feature dimensions" → Layer Norm (batch 아님!)
  • "better training stability for deep models" → Pre-Norm (Post 아님!)
  • "removes mean subtraction" → RMSNorm (Layer Norm 아님!)
📐 Position Embeddings — 기본 개념 (Q1-8)

Q1. Why are positional embeddings needed in Transformers? (10점)

A. To reduce model size
B. To encode token order information, since self-attention is permutation equivariant
C. To increase vocabulary size
D. To normalize embeddings
Answer: B
Self-Attention은 set operation — 순서를 모른다. "I love you"와 "You love I"가 같은 결과. PE로 순서 정보를 주입해야 한다.

Q2. What happens to a Transformer without positional encoding? (10점)

A. It cannot process text at all
B. It becomes faster
C. It treats input as an unordered set, losing sequential information
D. It only works for short sequences
Answer: C
PE 없으면 Self-Attention은 입력을 순서 없는 집합(set)으로 처리. 텍스트 처리 자체는 가능하지만 순서 정보가 완전히 사라진다.

Q3. In the original Transformer, how is positional information added to the input? (10점)

A. By adding sinusoidal vectors to token embeddings
B. By concatenating position vectors to token embeddings
C. By modifying attention scores directly
D. By using a separate position encoder network
Answer: A
Original Transformer: Input = TokenEmbed + SinusoidalPE. 더하기(addition)로 결합. Concatenation이 아님에 주의.

Q4. What is the main motivation for moving from absolute to relative positional encoding? (10점)

A. Reduce model parameters
B. Capture relationships between token positions more effectively
C. Improve tokenization
D. Remove attention mechanism
Answer: B
자연어에서 중요한 것은 "토큰이 위치 5에 있다"(절대)가 아니라 "두 토큰이 3칸 떨어져 있다"(상대). 상대 위치가 토큰 간 관계를 더 효과적으로 포착.

Q5. Which of the following is NOT a method of injecting positional information? (10점)

A. Adding position vectors to input embeddings
B. Adding bias terms to attention scores
C. Rotating query and key vectors
D. Modifying the feed-forward network weights
Answer: D
PE 주입 방법: (A) 입력에 더하기 (Sinusoidal, Learned), (B) attention score에 bias (ALiBi, T5), (C) Q/K 회전 (RoPE). FFN 수정은 PE 방법이 아님.

Q6. What is the evolution direction of positional encoding methods? (10점)

A. Relative → Absolute, Attention modification → Input modification
B. Complex → Simple, More parameters → Fewer parameters
C. Absolute → Relative, Input modification → Attention modification
D. Learned → Hardcoded exclusively
Answer: C
진화: 절대→상대, 입력 수정→attention 수정. Sinusoidal(절대, 입력) → Learned(절대, 입력) → ALiBi(상대, attention) → RoPE(상대, Q/K 회전).

Q7. Which positional encoding method is currently the standard for modern LLMs like LLaMA and Mistral? (10점)

A. Sinusoidal
B. Learned
C. ALiBi
D. RoPE (Rotary Position Embedding)
Answer: D
RoPE가 현재 표준. LLaMA, LLaMA 2, Mistral, PaLM 등 대부분의 최신 LLM이 사용. Q/K를 회전시켜 내적이 상대 위치에만 의존하게 함.

Q8. What does "permutation equivariant" mean in the context of self-attention? (10점)

A. The model can only process sorted sequences
B. Reordering input tokens produces correspondingly reordered outputs
C. The model is invariant to input length
D. Attention weights are always uniform
Answer: B
Permutation equivariant: 입력 순서를 바꾸면 출력도 같은 방식으로 바뀜. 즉 순서 자체의 정보를 활용하지 못함. 이것이 PE가 필요한 이유.
🔬 Position Embedding 방법론 상세 (Q9-18)

Q9. What is a key advantage of sinusoidal (hardcoded) positional embeddings? (10점)

A. They reduce training time
B. They eliminate the need for embeddings
C. They can generalize to arbitrary sequence lengths
D. They improve tokenization
Answer: C
Sinusoidal PE: sin/cos 함수로 생성되므로 학습 불필요(hardcoded), 임의 길이에 일반화 가능. Learned PE는 L_max 제한이 있지만 Sinusoidal은 없음.

Q10. In sinusoidal positional encoding, why are sine and cosine functions used? (10점)

A. To reduce computation
B. To ensure orthogonality between tokens
C. To encode relative positions through phase differences
D. To normalize embeddings
Answer: C
삼각함수 덧셈 정리: PE_{pos+k}를 PE_{pos}의 선형 변환(회전 행렬)으로 표현 가능. Phase difference를 통해 상대 위치를 인코딩.

Q11. Which of the following is a limitation of learned positional embeddings? (10점)

A. Cannot represent semantic meaning
B. Cannot be trained on GPUs
C. Cannot generalize to longer sequences than seen during training
D. Require labeled data
Answer: C
Learned PE는 위치별 embedding을 학습. L_max까지만 학습하므로 그 이상의 위치에는 embedding이 없다. BERT(512), GPT-2(1024).

Q12. Which method introduces a linear bias in attention scores to encode position? (10점)

A. RoPE
B. ALiBi
C. Word2Vec
D. CBOW
Answer: B
ALiBi = Attention with Linear Biases. score_ij = q_i^T k_j / √d_k + m·(-|i-j|). 거리에 비례하는 선형 bias를 attention score에 더함. RoPE는 Q/K를 회전시키는 방식.

Q13. What is the core idea behind RoPE (Rotary Position Embeddings)? (10점)

A. Add noise to embeddings
B. Rotate query and key vectors based on position so that their dot product depends only on relative position
C. Use convolution to encode positions
D. Replace attention with recurrence
Answer: B
RoPE: q̃_m = R(θ_m)·q, k̃_n = R(θ_n)·k. 내적 q̃^T k̃ = q^T R(θ_{n-m}) k → 상대 위치 (n-m)에만 의존. Long-term decay 성질도 가짐.

Q14. What does "Train Short, Test Long" refer to in the context of ALiBi? (10점)

A. Training with small batch sizes
B. Training on short sequences but generalizing to longer sequences at inference
C. Using shorter training epochs
D. Reducing model depth during training
Answer: B
ALiBi의 핵심 장점: 짧은 시퀀스로 학습해도 긴 시퀀스에서 잘 동작. 선형 bias가 자연스럽게 외삽(extrapolation)되기 때문.

Q15. In ALiBi, what is the role of different slopes (m) across attention heads? (10점)

A. All heads use the same slope
B. Slopes are learned during training
C. Larger slopes focus on nearby tokens, smaller slopes allow attending to distant tokens
D. Slopes control the learning rate
Answer: C
큰 slope → steep penalty → 가까운 토큰에 집중 (local). 작은 slope → gentle penalty → 먼 토큰도 참조 (global). Slope는 고정값(학습 안 함).

Q16. What is the "long-term decay" property of RoPE? (10점)

A. Model performance decreases over training
B. The upper bound of attention scores naturally decreases for more distant tokens
C. Learning rate decays over time
D. Sequence length decreases during training
Answer: B
RoPE에서 |q̃_m^T k̃_n| ≤ f(|m-n|), f는 감소 함수. 먼 토큰일수록 attention score 상한이 자연스럽게 감소 → 자연어의 locality와 일치.

Q17. How does RoPE differ from ALiBi in encoding positional information? (10점)

A. RoPE modifies input embeddings, ALiBi modifies attention scores
B. RoPE uses learned parameters, ALiBi uses fixed parameters
C. RoPE rotates Q/K vectors, ALiBi adds linear bias to attention scores
D. RoPE is absolute, ALiBi is relative
Answer: C
RoPE: Q, K 벡터를 위치에 따라 회전 → 내적이 상대 위치에 의존. ALiBi: attention score에 m·(-|i-j|) 선형 bias 추가. 둘 다 상대 위치, 둘 다 학습 파라미터 없음.

Q18. Which positional encoding method requires NO learnable parameters? (10점)

A. Learned PE only
B. T5 Relative Bias only
C. ALiBi only
D. Sinusoidal, ALiBi, and RoPE
Answer: D
학습 파라미터 없는 방법: Sinusoidal (sin/cos hardcoded), ALiBi (slope 고정), RoPE (회전 각도 hardcoded). Learned PE와 T5 Bias는 학습 파라미터 있음.
🔧 Layer Normalization (Q19-26)

Q19. What is the main benefit of Layer Normalization (LN) in Transformers? (10점)

A. Increase vocabulary size
B. Improve training stability and convergence
C. Reduce sequence length
D. Eliminate overfitting completely
Answer: B
"Helps with training stability and convergence." LN 없이는 깊은 Transformer 학습이 매우 어렵다. 각 레이어의 입력 분포를 안정화.

Q20. What distinguishes Layer Normalization from Batch Normalization? (10점)

A. LN normalizes across the batch dimension
B. LN normalizes across feature dimensions for each sample independently
C. LN requires large batch sizes
D. LN cannot be used in Transformers
Answer: B
LN: feature 축 정규화 (한 샘플 내 모든 feature). BN: batch 축 정규화 (같은 feature의 모든 배치). LN은 batch size에 독립적.

Q21. Why is Layer Norm preferred over Batch Norm in Transformers? (10점)

A. Layer Norm is always faster
B. Batch Norm cannot be implemented
C. Layer Norm uses fewer parameters
D. Layer Norm is independent of batch size and handles variable-length sequences naturally
Answer: D
LN 장점: (1) batch size 독립, (2) 가변 길이 자연스럽게 처리, (3) 추론 시 running stats 불필요. 또한 원 논문이 LN으로 튜닝됨.

Q22. Which statement about Pre-Norm vs Post-Norm Transformers is correct? (10점)

A. Post-Norm is more stable for deep models
B. Pre-Norm is commonly used today due to better training stability
C. Both are identical in performance
D. Pre-Norm removes the need for residual connections
Answer: B
Pre-Norm: x + SubLayer(LN(x)). Residual path가 깨끗 → gradient 직접 흐름 → 학습 안정적. GPT-2+, LLaMA, Mistral 등 현대 LLM의 표준.

Q23. Why does Pre-Norm provide better gradient flow than Post-Norm? (10점)

A. Pre-Norm uses fewer parameters
B. Pre-Norm eliminates the need for attention
C. In Pre-Norm, the residual path is clean (not blocked by LayerNorm), allowing gradients to flow directly
D. Pre-Norm uses a different activation function
Answer: C
Post-Norm: LN(x + SubLayer(x)) → LN이 residual path를 차단. Pre-Norm: x + SubLayer(LN(x)) → residual path가 깨끗. ResNet의 skip connection과 같은 원리.

Q24. What is RMSNorm and how does it differ from standard Layer Normalization? (10점)

A. RMSNorm adds an extra bias term
B. RMSNorm removes mean subtraction (re-centering) and the shift parameter β, using only RMS for normalization
C. RMSNorm normalizes across the batch dimension
D. RMSNorm requires more computation than Layer Norm
Answer: B
RMSNorm(x) = γ · x/RMS(x). 평균 빼기(re-centering) 제거, β 파라미터 없음. 학습 파라미터 γ만 (d개 vs LN의 2d개). 성능 동등, 계산 더 빠름.

Q25. What normalization combination do modern LLMs (LLaMA, Mistral) typically use? (10점)

A. Post-Norm with Batch Normalization
B. Post-Norm with Layer Normalization
C. Pre-Norm with RMSNorm
D. Pre-Norm with Batch Normalization
Answer: C
현대 LLM 표준: Pre-Norm (학습 안정성) + RMSNorm (효율성). 진화: Post-LN(2017) → Pre-LN(GPT-2) → Pre-RMSNorm(LLaMA).

Q26. What are the learnable parameters γ and β in Layer Normalization? (10점)

A. They control the learning rate
B. They determine the batch size
C. γ (scale) and β (shift) restore representational power after normalization
D. They are fixed constants
Answer: C
단순 정규화만 하면 표현력 감소. γ(scale)와 β(shift)로 모델이 필요하면 정규화를 "되돌릴" 수 있다. γ=σ, β=μ로 학습하면 원래 분포 복원 가능.
🧩 응용 & 종합 문제 (Q27-30)

Q27. A model trained with max sequence length 512 fails on a 2048-token input. Which PE method was most likely used? (10점)

A. Sinusoidal
B. Learned positional embeddings
C. ALiBi
D. RoPE
Answer: B
Learned PE는 L_max까지만 학습. 512로 학습하면 513번째 위치부터 embedding이 없어 실패. Sinusoidal, ALiBi, RoPE는 길이 일반화 가능.

Q28. Which combination of techniques does Mistral 7B use? (10점)

A. Learned PE + Post-Norm + Layer Norm
B. Sinusoidal PE + Pre-Norm + Batch Norm
C. ALiBi + Post-Norm + RMSNorm
D. RoPE + Pre-Norm + RMSNorm
Answer: D
Mistral 7B: RoPE (위치 인코딩) + Pre-Norm (학습 안정성) + RMSNorm (효율성) + SWA + GQA. 현대 LLM의 표준 조합.

Q29. If you need a model that can be trained on 1K tokens but must work on 8K tokens at inference, which PE is best? (10점)

A. Learned PE
B. T5 Relative Bias
C. ALiBi (designed for "Train Short, Test Long")
D. No positional encoding needed
Answer: C
ALiBi는 "Train Short, Test Long"을 위해 설계됨. 선형 bias가 자연스럽게 외삽되어 학습 시보다 긴 시퀀스에서도 잘 동작. RoPE도 가능하지만 ALiBi가 이 목적에 가장 직접적.

Q30. Which of the following correctly describes the relationship between normalization and training depth? (10점)

A. Deeper models don't need normalization
B. Batch Norm is better for deeper Transformers
C. Post-Norm enables training of deeper models than Pre-Norm
D. Pre-Norm with RMSNorm enables stable training of very deep Transformers by maintaining clean gradient flow
Answer: D
Pre-Norm: residual path 깨끗 → gradient 직접 흐름 → 깊은 모델 학습 가능. RMSNorm: 효율적이면서 동등한 성능. 이 조합이 현대 LLM의 표준.
