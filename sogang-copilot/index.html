1. 왜 이 문제인가

1-1. 문제의 실체

서강대 졸업요건은 단일 규정이 아니라 학번 × 전공 × 이중·복수전공 × 교양체계 개편 연도의 조합으로 파생되는 수십 가지 변종이다. 학생이 혼자 학칙 PDF, 요람, 수강편람, 교양과정 안내서, 전공 홈페이지 공지 네 종류를 동시에 펼치고 계산해야 한다.

이건 단순 FAQ 챗봇으로는 못 푼다. "교양필수 영어A를 들었는데, 교환학생 가서 받아온 영문학 과목이 영어전공강의 카운트가 되나요?" — 이 질문의 답은 학번, 대체 인정 규정, 해당 학기 교무회의 결정문, 학생의 이수 이력 네 가지를 동시에 추론해야 나온다. 룰 엔진으로는 기하급수적 예외를 관리할 수 없고, LLM 단독으로는 환각을 일으킨다. RAG + 구조화된 이수이력 + 도메인 제약의 교차 추론이 본질적으로 필요한 문제다.

2. 해결 방법 — 시스템 개요

핵심 파이프라인은 3단 추론이다. ① 이수이력 구조화 ② 규정 검색·제약 추출 ③ 시나리오 최적화. LLM은 ①의 파싱과 ③의 자연어 설명에만 쓰고, ②의 계산은 결정론적 룰로 검증해 환각을 봉쇄한다.

※ 핵심 설계 원칙: 계산은 코드가, 설명은 LLM이. 역할 라우터가 학부·편입·대학원 페르소나에 따라 CSP 제약집합과 출력 포맷을 스위칭한다.

2-1. ① 이수이력 파서

  • SAINT 과목이수표 PDF/PNG 업로드 → OCR + 구조화 LLM 호출로 JSON 변환
  • 과목코드·학점·성적·이수구분(전필/전선/교필/교선/자선)·이수학기 7-필드 스키마
  • 파싱 결과는 사용자에게 확인 UI 제공 (오인식 수동 수정)

2-2. ② 학칙 RAG

  • 코퍼스: 학번별 졸업요건 고시(2019~2026), 교양과정 안내(연도별), 전공별 이수체계도, 교무회의 유권해석 요약
  • 청크 단위: 조·항 기준 + 메타데이터(학번 적용 범위, 전공, 유효기간)
  • 검색 시 메타데이터 필터링 필수: 22학번 학생에게 20학번 규정 섞이지 않도록 강제
  • 모든 답변에 규정 원문 인용(조항 번호 + 페이지) 필수 — 환각 방지

2-3. ③ 시나리오 엔진

핵심 난제. 단순 그래프 탐색으로는 "영어강의 3과목 + 공통선택 4영역 + 중복인정 최대 9학점"처럼 다중 제약이 걸리므로, CSP(Constraint Satisfaction Problem) 구조로 풀고 LLM은 결과를 자연어로 설명하는 역할만 맡긴다.

# 의사코드
constraints = [
    Requirement("전공필수", subject_codes=[..], min_credits=X),
    Requirement("공통선택", areas={"A":1, "B":1, "C":1, "D":1}),
    Requirement("영어전공강의", min_count=3),
    OverlapRule("주전공-복수전공 중복인정", max_credits=9),
    ...
]
taken = parse_saint(pdf)
remaining = [c for c in constraints if not c.is_satisfied(taken)]
plan = csp_solver.solve(
    remaining=remaining,
    offered_courses=course_catalog,
    semesters_left=user.semesters_left,
    user_interests=user.tags,
)
explanation = llm.explain(plan, style="한국어 3줄 요약 + 학기별 표")

4. 핵심 기능 상세

4-1. 현재 이수 현황 진단

단순 체크리스트가 아니라 "왜 부족한지"의 원인까지 추적한다. 예: "공통선택 C영역 0과목 — 22학번 기준 C영역은 인문·사회탐구 섹션이며, 최소 1과목 필요."

4-2. 학기별 최적 수강 시나리오

남은 학기에 부하를 균등 분산하고, 선수과목 그래프를 존중한다. 과목 개설 학기(홀/짝)까지 반영.

4-3. 복수전공 What-if 시뮬레이터

킬러 기능. 복전 후보 전공을 클릭하면 즉석에서 "이 복전 추가 시 졸업까지 +1학기 / +18학점 / 여름계절 2과목" 같은 정량 비교표를 띄운다. 중복인정 9학점 규정, 복전에서 면제되는 교양 규정까지 자동 반영.

4-4. 자연어 Q&A — 규정 인용 필수

모든 답변은 규정 원문 링크와 함께 출력. "~일 것입니다" 류 추측성 표현은 프롬프트 레벨에서 차단.

질문시스템 응답 형식
교환학생 학점이 영어전공강의로 인정되나요? 원칙 답변 + 인용 학칙 제○조 ○항 + 예외 교무회의 2024-03호 + "확정은 교무팀 확인" 디스클레이머

4-5. 환각 방지 3중 안전장치

  1. 메타데이터 하드 필터 — 내 학번에 적용되지 않는 규정은 검색 결과에서 원천 배제
  2. 규정 원문 인용 강제 — 인용 없으면 응답 자체를 막는 output guardrail
  3. 디스클레이머 자동 삽입 — 졸업사정은 최종적으로 교무팀 확인이 필요하다는 문구를 모든 진단 리포트 하단에 박음

5. 왜 이 팀이 1등인가 — 심사기준 대응

심사기준본 제안의 근거
혁신성 단순 챗봇이 아닌 이수이력 × 학칙 × CSP 최적화의 3중 추론. 국내 대학 어디에도 "학번별 학칙을 메타데이터로 분리한 RAG" 제품은 없음.
문제 해결 타당성 서강대 복전 비율이 타 대학 대비 높다는 구조적 특성과 정합. 종합봉사실 문의 감소라는 행정 비용 절감까지 정량화 가능.
구현 가능성 풀스택 + LLM 경험 보유. 본선 30만원 개발비 + 마인드로직 LLM 크레딧 지원 범위 내 완결. 9주 로드맵 제시.
AI 활용 적절성 "AI가 없으면 안 풀리는 문제" 요건 충족. 단일 룰엔진으로는 학번별 × 복전 × 예외의 조합폭발 감당 불가.
실제 도입 가능성 SAINT 연동 플러그인 형태로 설계. 학교가 원하면 그대로 붙일 수 있도록 API 스펙을 OpenAPI 3.0 문서로 동봉 제출 예정.
