## 이 글이 다루는 것

대회 사무국이 명시한 자산군·세부자산·종목 3단계 상한을 동시에 만족하는 비중을 즉시 산출하는 엔진. Streamlit 대시보드로 매일 운용 상태를 확인한다.

## 모듈 구조

- **config.py** — 자산군·세부자산·종목 상한을 그대로 매핑한 설정 파일. 대회 규정이 바뀌면 이 파일만 수정.
- **allocator.py** — 후보 ETF/종목과 목표 비중을 받아 상한 위반 여부를 검증·완화·재배분.
- **market_data.py** — 종목 정적 데이터(시총·유동성)와 가격 시계열 조회.
- **claude_report.py** — 일자별 포지션·손익·집중 리스크를 LLM이 한국어 운용 메모로 요약.
- **app.py** — Streamlit UI. 자산군 상한 게이지 + 일별 손익 라인 + 종목 비중 테이블.

## 핵심 의사결정

- 상한을 "하드 제약"으로 둠. 옵티마이저가 상한을 깨면 경고 대신 거절.
- 국내 8억 / 해외 2억 분리 회계. 한 함수로 처리하면 환산 오류가 잠복할 수 있어 의도적으로 분리.
- 일별 리포트 생성기와 자산배분 엔진을 같은 데이터 객체로 공유 — 포지션 일관성 보장.

## 적용 효과

- 주문 직전에 "이 한 종목이 들어가면 어떤 상한을 깨는지" 즉시 확인.
- 일일 리포트가 자동 생성되어 운용 일지가 끊기지 않음.
- 대회 종료 후 회고용 데이터가 그대로 남아 다음 대회 출전 시 직접 비교 가능.
