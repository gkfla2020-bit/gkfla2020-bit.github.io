▲ 맨위 Day1 Day2 Day3 총정리

정보처리기사 필기 · 3일 컷 기출 완전정복

2021~2025년 15개 기출(1~3회) 빈출 개념을 분석하여 3일 안에 합격 가능하도록 구성한 흑백 공부용 자료입니다. 5과목 × 20문항 = 총 100문항, 과목당 40점 이상 + 평균 60점 이상이 합격선입니다. 기출 반복 키워드에 HOT 표시.

3일 운영전략
Day 1 — 소프트웨어 설계 + 소프트웨어 개발 (과목1+2) · 집중 4~5h
Day 2 — 데이터베이스 구축 + 프로그래밍 언어 활용 (과목3+4) · 집중 5~6h
Day 3 — 정보시스템 구축관리 (과목5) + 총정리 암기 + 2회분 풀기 · 4h

목차

  1. Day 1 — 과목1 소프트웨어 설계 / 과목2 소프트웨어 개발
  2. Day 2 — 과목3 데이터베이스 구축 / 과목4 프로그래밍 언어 활용
  3. Day 3 — 과목5 정보시스템 구축관리 + 총정리
  4. 최종 요약·암기카드 (시험 직전용)
  5. 자주 틀리는 함정 포인트
합격 공부법 (교육방법론 적용)
  • 간격 반복(Spaced Repetition) — 각 섹션의 "체크포인트"를 Day가 끝나고 한 번, 다음 날 아침 한 번, 시험 직전 한 번 복습.
  • 능동회상(Active Recall) — 개념을 읽고 바로 책 덮고 키워드만 떠올려본 뒤 다시 확인.
  • Chunking — 영문 약어는 알파벳별 풀네임으로 3~5개씩 묶어서 암기.
  • Feynman — 한 줄로 스스로 설명 가능한지 점검. 설명 못 하면 다시 읽기.
  • 기출 우선 — 개념보다 기출문제 유형을 먼저 노출하면 개념이 빠르게 고정됨.
DAY 1

과목 1 · 소프트웨어 설계 & 과목 2 · 소프트웨어 개발

총 분량 4~5시간 · 핵심 키워드 80개

제 1 과목 · 소프트웨어 설계

1-1. 소프트웨어 생명주기 모형 HOT

  • 폭포수(Waterfall) — 고전적, 단계적 진행, 변경 어려움. "타당성→요구→설계→구현→테스트→유지보수".
  • 프로토타입(Prototyping) — 시제품으로 요구사항 명확화. 최종 결과물 일부를 먼저 의뢰자에게 보여주는 모델. 요구사항 반영 가능(기출 함정: "반영 불가능" = 틀림).
  • 나선형(Spiral) — Boehm 제안. 위험분석이 핵심. 계획→위험분석→개발→평가를 반복.
  • V-모델 — 폭포수 확장, 각 단계별 검증·확인(테스트) 대응.
  • 애자일(Agile) — 변화 대응, 반복적·점진적.
    애자일 4대 가치 : 개인&상호작용 > 프로세스/도구, 작동 SW > 문서, 고객협업 > 계약협상, 변화대응 > 계획준수. (기출: "문서 > 작동 SW" 는 틀림)
생명주기 = "폭프나 V 애" → 폭포수·프로토타입·나선형·V모델·애자일

1-2. 애자일 기법 — XP, Scrum, Lean

  • XP(eXtreme Programming) 5가지 핵심가치: 의사소통(Communication)·단순성(Simplicity)·용기(Courage)·존중(Respect)·피드백(Feedback). 기법: Pair Programming, TDD, 리팩토링, 사용자 스토리, 지속적 통합(CI).
  • Scrum — Sprint(2~4주), 제품책임자(PO)·스크럼마스터(SM)·개발팀. 일일 스크럼, 백로그.
  • Lean — 7원칙: 낭비 제거, 품질 내재화, 지식 창출, 늦은 결정, 빠른 인도, 사람 존중, 전체 최적화.
XP는 "테스트를 수행하지 않는다" → 절대 틀린 말. XP는 TDD(테스트 주도 개발) 기반!

1-3. 요구사항 분석

  • 기능 요구 — 시스템이 수행해야 할 기능 (예: 조회·입금·송금).
  • 비기능 요구 — 성능, 품질, 보안, 안전성, 응답시간 등. 시험 단골: "화면이 3초 내 보여야 한다" = 비기능.
  • 분석 도구: 자료흐름도(DFD), 자료사전(DD), 소단위명세서(Mini-Spec).

1-4. 자료흐름도(DFD) · 자료사전(DD) HOT

DFD 4요소: Process(원)·Data Flow(화살표)·Data Store(평행선 두 줄)·Terminator(사각형). (함정: Data Store를 "삼각형"이라 하면 틀림)

기호의미예시
=정의(is composed of)A = B + C
+연결(and)성명 + 나이
( )생략 가능(중간이름)
{ }반복(iteration){항목}
[ ]선택(or)[남 | 여]
**주석**설명**
DFD = "원·선·창·박" = Process·Flow·Store·Terminator

1-5. HIPO · N-S Chart

  • HIPO — 하향식 SW 개발 문서화 도구. 가시적 도표(Visual Table of Contents) / 총체적 도표(Overview) / 세부적 도표(Detail). (함정: "상향식" 이라고 하면 틀림)
  • N-S Chart(Nassi-Schneiderman) — 논리 중심의 도형식 표현. 화살표 사용 X, 박스 구조(연속·선택·반복)로 표현. "주로 화살표 사용" = 틀림.

1-6. UML (Unified Modeling Language) HOT

OMG 제정. 14종 다이어그램. 구조(정적) 7 / 행위(동적) 7.

구조(정적)행위(동적)
Class, Object, Component, Deployment, Package, Composite Structure, Profile Use case, Sequence, Activity, State, Communication(Collaboration), Interaction Overview, Timing
  • Class Diagram — 클래스·속성·연산·관계(연관/일반화/의존/실체화).
  • Use Case Diagram — 사용자(Actor) 관점 기능. 관계: 포함(include)·확장(extend)·일반화.
  • Sequence Diagram — 객체 간 메시지 순서(시간 흐름). 구성: 생명선(Lifeline)·실행(Activation)·메시지·객체. (함정: "확장"은 구성요소 아님)
  • State Diagram — 한 객체의 상태 변화.
  • Activity Diagram — 처리 흐름, 워크플로.
기출 함정: "State Diagram이 객체 간 메시지 교환" = 틀림 (그건 Sequence임).

1-7. 럼바우(Rumbaugh) 객체지향 분석 HOT

순서: 객체 → 동적 → 기능 (OMT)
객체 모델링(Object) — 객체 다이어그램, 정보 모델링
동적 모델링(Dynamic) — 상태 다이어그램, 시간 흐름
기능 모델링(Functional) — DFD, 자료 흐름
"객동기" = 객체→동적→기능 (축구감독 이름처럼 외우기)

1-8. 객체지향 · 캡슐화·상속·다형성

  • 캡슐화(Encapsulation) — 데이터 + 연산 묶음. 정보 은닉의 핵심. 변경 시 영향 최소화.
  • 상속(Inheritance) — 상위 속성/연산 물려받기.
  • 다형성(Polymorphism) — 같은 메시지에 다른 반응(오버로딩/오버라이딩).
  • 추상화(Abstraction) — 복잡도 감소, 공통 특성만 표현.
  • 메시지(Message) — 객체에 행위를 지시하는 명령.

1-9. SOLID 객체지향 설계 원칙 HOT

  • SRP 단일책임 — 클래스는 하나의 책임만.
  • OCP 개방폐쇄 — 확장엔 열리고 수정엔 닫힘.
  • LSP 리스코프치환 — 서브타입은 상위타입 자리에 교체 가능.
  • ISP 인터페이스분리 — 사용하지 않는 인터페이스 의존 금지. "클라이언트가 사용하지 않는 메소드와 의존관계 X".
  • DIP 의존역전 — 추상에 의존, 구체에 의존 X. "추상성 낮은 클래스 → 높은 클래스에 의존".
SOLID = S·O·L·I·D 순서대로

1-10. 결합도 & 응집도 HOT

결합도(Coupling): 약할수록 좋음
자료(Data) < 스탬프(Stamp) < 제어(Control) < 외부(External) < 공통(Common) < 내용(Content)
응집도(Cohesion): 강할수록 좋음
기능(Functional) > 순차(Sequential) > 교환/통신(Communicational) > 절차(Procedural) > 시간(Temporal) > 논리(Logical) > 우연(Coincidental)
결합도 약→강: "자스제외공내" · 응집도 강→약: "기순교절시논우"
  • 좋은 모듈 = 결합도 낮고 응집도 높음.
  • Cohesion(응집도), Coupling(결합도) 영단어도 바로 구분 가능하게.

1-11. 디자인 패턴 (GoF) HOT

분류패턴
생성(Creational) 5 Singleton, Factory Method, Abstract Factory, Builder, Prototype
구조(Structural) 7 Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy
행위(Behavioral) 11 Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor
빈출 함정: Observer·Iterator·State는 행위 패턴. (구조 아님!) · Adapter·Decorator·Composite·Bridge는 구조. Prototype은 생성.
생성 5 = "싱 팩 추 빌 프", 구조 7 = "어 브 컴 데 파 플 프", 행위 = 나머지

제 2 과목 · 소프트웨어 개발

2-1. 자료구조 — 선형/비선형 HOT

  • 선형: 배열·연결리스트·스택·큐·데크.
  • 비선형: 트리·그래프.
  • 스택(Stack) LIFO — 후위표기·재귀·undo.
  • 큐(Queue) FIFO — 스케줄링·버퍼.

함정: "큐는 비선형" = 틀림.

2-2. 트리(Tree) HOT

  • 차수(Degree): 노드가 가진 자식 수. 트리의 차수 = 전체 노드 중 최대 차수.
  • 단말(Leaf): 자식 없는 노드.
  • 레벨·높이(깊이): 루트=1부터.
  • 이진트리 순회
    • 전위(Preorder): 루트→좌→우 (ROOT-LEFT-RIGHT)
    • 중위(Inorder): 좌→루트→우 (LEFT-ROOT-RIGHT)
    • 후위(Postorder): 좌→우→루트
수식 표기법 변환
중위: A+B·C → 후위: ABC·+ → 전위: +A·BC
(피연산자 순서 유지, 연산자만 이동)

2-3. 정렬 알고리즘

알고리즘평균최악특징
버블O(n²)O(n²)인접 비교 교환
선택O(n²)O(n²)최솟값 선택
삽입O(n²)O(n²)거의 정렬 시 빠름
퀵O(n log n)O(n²)분할정복, 피벗
병합O(n log n)O(n log n)안정 정렬
힙O(n log n)O(n log n)힙 구조 이용
쉘O(n^1.5)-삽입정렬 개선
"버선삽 / 퀵병힙쉘" — 앞은 느리고 뒤는 빠름

2-4. 해싱(Hashing) 함수

  • 제산법(Division) · 제곱법 · 폴딩법(Folding, 여러 부분 더하기/XOR) · 기수변환 · 숫자분석 · 무작위.
  • 충돌 해결: 개방주소(선형·이차·이중해싱), 체이닝.
  • 버킷·슬롯·동거자(Synonym)·충돌·오버플로.

2-5. 테스트 기법 HOT

구분화이트박스블랙박스
관점내부 구조·로직외부 명세·기능
기법문장/분기/조건/경로 커버리지, 기초경로, 제어흐름, 데이터흐름, 루프동등분할(Equivalence Partitioning), 경계값 분석(BVA), 원인-결과, 오류추측, 비교

2-6. McCabe Cyclomatic Complexity

공식: V(G) = E - N + 2 (E=간선, N=노드)
혹은 V(G) = P + 1 (P=결정점 수)

기출 전형: 제어흐름 그래프 주고 V(G) 계산.

2-7. 테스트 전략 — V·통합 HOT

  • 단위→통합→시스템→인수 순.
  • 상향식(Bottom-up): 하위 모듈부터 · 드라이버(Driver) 필요.
  • 하향식(Top-down): 상위부터 · 스텁(Stub) 필요.
  • 빅뱅 / 샌드위치(혼합).
  • 회귀 테스트(Regression): 변경 후 기존 기능 검증.
  • α 테스트: 개발자 앞 사용자 / β 테스트: 실제 사용 환경.
상향=드라이버(아래서 위로 운전), 하향=스텁(위에서 아래로 끊어쓰기)

2-8. 리뷰 기법

  • 워크스루(Walkthrough) — 비공식 리뷰, 작성자 주도, 미리 자료 배포.
  • 인스펙션(Inspection) — 공식, 제3자 주도, 결함 찾기 체크리스트 기반.
  • 동료검토(Peer review).

2-9. 형상관리(SCM) HOT

제 3 과목 · 데이터베이스 구축

3-1. 데이터베이스 설계 단계 HOT

단계수행 작업
1. 요구조건 분석요구사항 명세서 작성
2. 개념적 설계E-R 모델, 트랜잭션 모델링, DBMS 독립
3. 논리적 설계목표 DBMS에 맞는 논리 스키마, 정규화, 테이블 설계
4. 물리적 설계저장 구조·인덱스·접근 경로 설계
5. 구현DDL로 스키마 작성, 응용 프로그램 작성
함정: "요구분석 - 트랜잭션 모델링" → 트랜잭션 모델링은 개념적 설계. / "논리적 설계 - DBMS 독립" → 논리는 DBMS 종속.

3-2. E-R 모델

  • 개체(Entity) = 사각형, 속성(Attribute) = 타원, 관계(Relationship) = 마름모, 연결 = 선.
  • 개체·속성·관계 타입을 기본 요소로 현실세계를 표현.

3-3. 관계형 데이터베이스 용어 HOT

  • 릴레이션(Relation) = 테이블.
  • 튜플(Tuple) = 행(Row) = 레코드. 카디널리티(Cardinality) = 튜플 수.
  • 속성(Attribute) = 열(Column) = 필드. 차수(Degree) = 속성 수.
  • 도메인(Domain) = 속성이 취할 수 있는 값의 집합.

3-4. 키(Key) 종류

  • 후보키(Candidate) — 유일성 + 최소성 만족.
  • 기본키(Primary) — 후보키 중 하나 선택, NULL 불가.
  • 대체키(Alternate) — 후보키 중 기본키 제외.
  • 슈퍼키(Super) — 유일성만 만족(최소성 X).
  • 외래키(Foreign) — 다른 릴레이션의 기본키 참조.
참조 무결성: 외래키 값은 참조 테이블의 기본키에 존재하거나 NULL.

3-5. 무결성 제약 HOT

  • 개체 무결성 — 기본키 NULL 불가, 중복 불가.
  • 참조 무결성 — 외래키 ↔ 참조 릴레이션 기본키 대응.
  • 도메인 무결성 — 속성 값이 도메인에 속함.
  • 사용자 정의 무결성 — 업무규칙.
  • NULL 무결성 / Key(유일성) 무결성.
"보안 무결성·정보 무결성" 같은 용어는 기출에서 가짜 선지!

3-6. 정규화(Normalization) HOT

정규형조건
1NF모든 속성이 원자값 (도메인 분할 불가)
2NF1NF + 기본키에 완전 함수 종속 (부분 종속 제거)
3NF2NF + 이행 종속 제거
BCNF3NF + 모든 결정자가 후보키
4NF다치 종속 제거
5NF조인 종속 제거
"도부이결다조" = 도메인원자·부분종속제거·이행종속제거·결정자후보키·다치·조인

역정규화(Denormalization) — 성능을 위해 일부러 중복 허용.

3-7. 관계 대수 & 관계 해석

  • 관계 대수: 절차적 언어. 원하는 결과를 얻기 위한 연산 명시.
  • 관계 해석: 비절차적, 서술적.

순수 관계 연산: SELECT(σ) · PROJECT(π) · JOIN(⋈) · DIVISION(÷)

일반 집합 연산: 합집합(∪)·교집합(∩)·차집합(−)·곱집합(×)

3-8. SQL — DDL/DML/DCL HOT

분류명령어
DDL(정의)CREATE · ALTER · DROP · TRUNCATE · RENAME
DML(조작)SELECT · INSERT · UPDATE · DELETE
DCL(제어)GRANT · REVOKE · COMMIT · ROLLBACK · SAVEPOINT
기출: "DML에 해당? INSERT·GRANT" → GRANT는 DCL!

3-9. SQL 문법 핵심

-- 기본
SELECT 컬럼 FROM 테이블 WHERE 조건
GROUP BY 그룹 HAVING 집계조건
ORDER BY 정렬 [ASC|DESC];

-- UPDATE (SET 위치 주의)
UPDATE 사원 SET 사원코드=사원코드+1000
WHERE 부서='인사' AND 사원코드<=3000;

-- LIKE 패턴
WHERE 이름 LIKE '김%'   -- 김으로 시작
WHERE 이름 LIKE '%사'   -- 사로 끝남
WHERE 이름 LIKE '_길동' -- 한 글자+길동

-- NULL 체크
WHERE 점수 IS NULL
WHERE 점수 IS NOT NULL

-- JOIN
SELECT * FROM A INNER JOIN B ON A.id = B.id;
SELECT * FROM A LEFT OUTER JOIN B ON A.id = B.id;

-- 집계
COUNT(*), SUM(col), AVG(col), MAX(col), MIN(col)

-- 뷰
CREATE VIEW V1 AS SELECT ... FROM ...;
DROP VIEW V1;

3-10. VIEW(뷰) 특징

  • CREATE VIEW로 정의, DROP으로 제거.
  • 논리적 독립성 제공, 물리적으로 존재하지 않음(가상 테이블).
  • 보안 측면에서 DBA가 활용 가능(특정 컬럼만 노출).
  • ALTER 불가(수정은 DROP 후 재생성).
함정: "뷰는 저장장치 내에 물리적으로 존재한다" = 틀림.

3-11. 트랜잭션 ACID HOT

  • Atomicity 원자성 — 모두 반영 or 모두 취소.
  • Consistency 일관성 — 실행 전후 DB 일관된 상태.
  • Isolation 고립성 — 실행 중 중간 결과 다른 트랜잭션 접근 X.
  • Durability 지속성 — 완료 후 영구 반영.

트랜잭션 상태: 활동(Active)→부분완료→완료(Committed)/실패→철회(Aborted, Rollback).

3-12. 병행제어(Concurrency) · 로킹

  • 로킹 단위가 작으면: 병행성↑, 오버헤드↑.
  • 단위가 크면: 공유도↓, 오버헤드↓.
  • 2PL(Two-Phase Locking): 확장(Lock만) → 축소(Unlock만) 단계.
  • 타임스탬프 순서 기법, 낙관적(Optimistic), 다중 버전 기법(MVCC).
함정: "로킹 단위 커지면 공유도 증가" = 틀림 (공유도 감소).

3-13. 트리거(Trigger)

  • INSERT/UPDATE/DELETE 이벤트에 자동 실행되는 절차형 SQL.
  • 무결성 유지, 로그 기록 용도.
  • CREATE TRIGGER 로 생성.

3-14. 파티셔닝(Partitioning)

  • 하나의 테이블을 작은 논리적 단위로 분할.
  • 가용성·성능 향상.
  • 종류: Range·List·Hash·Composite.

3-15. 인덱스(Index)

  • B-Tree, B+Tree, Hash, Bitmap 인덱스.
  • 검색 속도 향상, 갱신 시 오버헤드.
  • 클러스터드(데이터 정렬 저장) vs 논클러스터드.

3-16. 회복(Recovery) 기법

  • 로그 기반: 지연 갱신, 즉시 갱신.
  • 검사점(Checkpoint) 기법.
  • 그림자 페이징(Shadow Paging).
  • REDO: 커밋된 트랜잭션 재실행. UNDO: 미완료 트랜잭션 취소.

3-17. 분산 데이터베이스 목표

투명성: 위치·중복·병행·장애 투명성.

과목 3 체크: 정규화 순서 / 키 5종 / ACID / DDL·DML·DCL 분류 / 뷰 특징. 손으로 써보기.

4-12. TCP / UDP / IP HOT

SSH 포트 = 22 (기출 선지 "220번" = 틀림).

4-14. 라우팅 프로토콜

  • RIP — 거리 벡터, 홉 카운트 최대 15.
  • OSPF — 링크 상태(Link State), 최단경로(Dijkstra). 네트워크 변화 신속 대처.
  • BGP — 외부 게이트웨이, AS 간.
  • EIGRP — 하이브리드(Cisco).
OSPF는 링크 상태. "거리 벡터"라고 하면 틀림(그건 RIP).

4-15. LAN & MAC HOT

  • IEEE 802.3 — Ethernet, CSMA/CD 방식.
  • IEEE 802.11 — Wi-Fi, CSMA/CA 방식 (무선: 매체 비어있음 확인 후 전송).
  • IEEE 802.15 — Bluetooth/WPAN.
  • IEEE 802.16 — WiMAX.
  • 토폴로지: 버스(Bus)·링(Ring)·성(Star)·트리(Tree)·망(Mesh).

4-16. 네트워크 장비

  • Physical: 리피터, 허브.
  • Data Link: 브리지, L2 스위치.
  • Network: 라우터, L3 스위치, 게이트웨이 (서로 다른 네트워크 연결).

5-4. 일정 관리 — PERT / CPM / 간트

5-8. 접근통제 정책·모델

5-9. 암호화 알고리즘 HOT

5-10. IPSec · SSL/TLS

  • IPSec은 공개키·해시·대칭키 모두 사용. (함정: "일방향 암호화만 지원" = 틀림)

  2. 과목당 40점 미만 = 과락. 한 과목에 4문제 이상 맞히는 게 안전선.
  3. 평균 60점 = 100문제 중 60개 이상 정답.
  4. "틀린 것은?" 문제는 선지 4개 중 1개만 골라 빨간줄 표시. 나머지는 맞는 설명 = 추가 공부.
  5. 계산문제는 뒤로 미루고 지식문제 먼저 마무리.
  6. 2개까지 후보 좁히면 찍어도 50% 확률 = 포기 금지.
