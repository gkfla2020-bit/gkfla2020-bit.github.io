18) Plaintext 한 글자 변경 시 Ciphertext 변화 개수

1.5점 (각 0.25점)

각 암호 방식에서 plaintext의 한 문자를 변경했을 때, ciphertext에서 몇 개의 문자가 변하는지 답하시오.

a. Additive Cipher (Caesar)
정답: 1
Ci = (Pi + k) mod 26. Ci는 Pi에만 의존하므로 한 글자만 변함.
채점: 정답 1 → 0.25점. "1글자" 또는 "only one"이면 정답. 설명 없어도 답만 맞으면 OK.
b. Multiplicative Cipher
정답: 1
Ci = (Pi × k) mod 26. Ci는 Pi에만 의존하므로 한 글자만 변함.
채점: 정답 1 → 0.25점.
c. Affine Cipher
정답: 1
Ci = (Pi × k1 + k2) mod 26. Ci는 Pi에만 의존.
채점: 정답 1 → 0.25점.
d. Vigenere Cipher
정답: 1
Ci = (Pi + ki) mod 26. ki는 위치에 따라 달라지지만 다른 plaintext 문자에 의존하지 않음. 한 글자만 변함.
채점: 정답 1 → 0.25점.
e. Autokey Cipher
⚠️ 정답: 2 (답지 원본 수정됨)
Ci = (Pi + ki) mod 26 = (Pi + Pi-1) mod 26.
한 글자를 바꾸면 해당 위치의 ciphertext + 바로 다음 위치의 ciphertext, 총 2개가 변함.
그 이후의 ciphertext는 영향 없음 (key가 원래 plaintext 값으로 돌아가므로).
채점: 정답 2 → 0.25점. 답지 원본에 오류가 있었으나 정정됨. "all characters after"라고 답한 경우 0점.
f. One-Time Pad
정답: 1
키 스트림은 한 번만 사용. 각 ciphertext 문자는 해당 위치의 plaintext에만 의존. 한 글자만 변함.
채점: 정답 1 → 0.25점.

19) Transposition Cipher에서 Plaintext 한 글자 변경 시

0.5점 (각 0.25점)
a. Single Transposition
정답: 1
Single transposition은 문자의 순서만 재배열. 문자 자체는 변하지 않으므로 한 글자만 영향.
채점: 정답 1 → 0.25점.
b. Double Transposition
정답: 1
Double transposition도 순서만 두 번 재배열. 문자 값 자체는 변하지 않으므로 한 글자만 영향.
채점: 정답 1 → 0.25점.

27) Transposition Cipher 공격 유형 및 컬럼 수

1.0점 (각 0.5점)
a. 공격 유형
정답: Chosen-plaintext attack
Eve가 Alice의 컴퓨터에 접근하여 원하는 plaintext를 암호화할 수 있으므로 chosen-plaintext attack.
채점: "chosen-plaintext attack" 명시 → 0.5점. "known-plaintext"라고 쓰면 0점.
b. 컬럼 수 추정
정답: 2 또는 5
메시지 길이 10 = 2 × 5. 가능한 컬럼 수: 1, 2, 5, 10.
1이나 10은 transposition의 의미가 없으므로 비현실적 → 2 또는 5.
채점: "2 또는 5" (또는 "2 or 5") → 0.5점. 1, 2, 5, 10 전부 나열만 하고 필터링 안 하면 0.25점. 약수만 구하고 끝내면 0.25점.

29) Affine Cipher 해독

2.0점

Known-plaintext: "ab" → "GL"

풀이 과정:

1. 문자 → 숫자 변환: a=00, b=01, G=06, L=11

2. Affine cipher: C = (P × k1 + k2) mod 26

3. 연립방정식 수립:
    00 × k₁ + k₂ ≡ 06 (mod 26) → k₂ = 6
    01 × k₁ + k₂ ≡ 11 (mod 26) → k₁ + 6 ≡ 11 → k₁ = 5

4. 키: k₁ = 5, k₂ = 6

5. 복호화 공식:
    P = ((C - k₂) × k₁⁻¹) mod 26 = ((C - 6) × 21) mod 26 = ((C + 20) × 21) mod 26
    (k₁⁻¹ = 5⁻¹ mod 26 = 21, 왜냐하면 5 × 21 = 105 ≡ 1 mod 26)

6. Ciphertext: XPALASXYFGFUKPXUSOGEUTKCDGFXANMGNVS

7. Plaintext: the best of a fight is making up afterwards
채점 기준 (2.0점):
• 연립방정식 수립 및 k₁=5, k₂=6 도출: 0.8점
• k₁⁻¹ = 21 계산 및 복호화 공식 유도: 0.6점
• 최종 plaintext 정답: 0.6점
※ 최종 답만 맞고 과정 없으면 1.0점. 과정은 맞는데 계산 실수로 plaintext 틀리면 1.4점.

39) Hill Cipher의 Chosen-Plaintext Attack 취약성

1.0점
정답:
Hill cipher에서 C = P × K.
Plaintext로 identity matrix I를 선택하면 C = I × K = K.
즉, ciphertext가 곧 key가 됨.

따라서 Eve가 Alice의 컴퓨터에 접근하여 identity matrix를 plaintext로 암호화하면 key를 바로 알아낼 수 있음.
→ Hill cipher는 chosen-plaintext attack에 매우 취약.
채점 기준 (1.0점):
• C = P × K에서 P = I이면 C = K라는 핵심 논리: 0.6점
• chosen-plaintext attack에 취약하다는 결론: 0.4점
※ 결론만 쓰고 이유 없으면 0.4점.

22) GF(2⁴)에서 덧셈과 뺄셈

0.5점 (각 0.25점)

지수의 modulus: 2n - 1 = 15. GF(2)에서 덧셈 = 뺄셈 = XOR.

a. 덧셈 예시
정답:
• g³ + g¹² = g³ + (g + 1) = g³ + g + 1 → (1011) = (1000) + (0011)
• g¹⁰ + g¹² = (g³ + g) + (g + 1) = g³ + 1 → (1001) = (1010) + (0011)
채점: 덧셈 예시 2개 중 최소 1개 정확 → 0.25점. 둘 다 틀리면 0점.
b. 뺄셈 예시
정답:
• g³ - g⁹ = g³ - (g² + 1) = g³ + g² + 1 → (1101) = (1000) - (0101)
• g¹⁰ - g⁴ = (g³ + g) - (g³ + 1) = g + 1 → (0011) = (1010) - (1001)
채점: 뺄셈 예시 2개 중 최소 1개 정확 → 0.25점. 둘 다 틀리면 0점.
참고: 22번은 21번 결과를 활용하는 문제. 학생이 다른 예시를 사용해도 계산이 맞으면 정답 처리.

30) GF(2)에서 Degree 3 Irreducible Polynomial 찾기

1.0점
풀이:
Degree 3 다항식이 reducible이면 degree 1 + degree 2로 인수분해 가능.
GF(2)에서 degree 1 다항식: x, (x+1) 뿐.

모든 degree 3 다항식 검사:
  • x³ = x × x² → reducible
  • x³ + 1 = (x+1)(x²+x+1) → reducible
  • x³ + x = x(x²+1) → reducible
  • x³ + x + 1 → irreducible (x=0: 1≠0, x=1: 1≠0)
  • x³ + x² = x²(x+1) → reducible
  • x³ + x² + 1 → irreducible (x=0: 1≠0, x=1: 1≠0)
  • x³ + x² + x = x(x²+x+1) → reducible
  • x³ + x² + x + 1 = (x+1)(x²+x+1) → reducible
정답: f₁(x) = x³ + x + 1, f₂(x) = x³ + x² + 1
채점 기준 (1.0점):
• 두 irreducible polynomial 모두 정확히 찾음: 1.0점
• 하나만 찾음: 0.5점
• 방법은 맞지만 계산 실수: 0.5점
• 전체 다항식 나열 + 인수분해 과정 보여주면 가산점 없음 (기본)
