왜 이 조합인가 (Rationale)

① 문제 정의

서강대 수강신청 시즌마다 반복되는 풍경 — 학생은 강의계획서 PDF 수백 건을 여러 탭에 띄워놓고 전공필수·선수과목·시간표 충돌을 수기로 맞추느라 하루를 보냅니다. 특히 트랙제·융합전공·복수전공 학생은 "내 커리어에 맞는 4년 경로"를 그릴 도구가 없어 선배의 구전 조언이나 SNS 카더라에 의존합니다.

페인 포인트 3가지

  • 분산된 정보: 강의계획서·졸업요건·트랙 안내가 3개 이상의 시스템에 흩어져 있음
  • 정적 안내: 학과 홈페이지의 커리큘럼은 평면 PDF — 내 학점·관심사로 필터링 불가
  • 개인화 부재: ChatGPT는 서강대 규정을 모르고, 학사시스템은 추천 기능이 없음

② 솔루션 — "역산" 접근

학생이 목표(트랙·관심사)와 제약(학년·학점·스케줄)을 입력하면, 에이전트가 졸업까지의 경로를 역순 계획(backward chaining)으로 풀어냅니다.

  1. 요건 수집 — 해당 학과의 졸업 요건과 트랙 코어과목을 RAG로 불러옴
  2. 필수 배치 — 전공필수를 개설학기 조건에 맞게 우선 할당
  3. 선수과목 역추적 — 관심 상급과목(예: 딥러닝)에서 기계학습 → 자료구조 → 프로그래밍입문 체인을 자동 확장
  4. 제약 풀이 — 시간표 충돌·학기 최대학점·자유 제약(예: 금요일 오후 회피)을 SAT처럼 검증
  5. 설명 생성 — LLM이 "왜 이 조합인가"를 자연어로 풀이 + 출처 인용

③ 차별점

vs
ChatGPT 범용
→
서강 규정 RAG
vs
학사시스템
→
LLM 개인화
  • 범용 LLM은 서강대 학칙·선수과목·트랙을 모름 → 도메인 RAG로 해결
  • 학사시스템은 조회만 제공 → 본 서비스는 역산 플래닝 + 설명 생성
  • 에타 선배 조언은 개인화·검증 불가 → 출처 인용 + 확신도 표시로 신뢰 확보

④ 2개월 MVP 범위

  • 학과 2개: 경영학과 · 컴퓨터공학 (타 전공 확장 구조 확보)
  • 강의계획서 200건 인덱싱 (PDF 텍스트 추출 + 벡터 임베딩)
  • 로드맵 생성기: 관심사 → 학기별 추천 + Rationale
  • 시간표 충돌 검증기: 제약 기반 필터
  • 웹 UI: 이 데모 수준 완성도 + 실제 LLM 백엔드 연결

이후 확장 (본선 이후)

  • 교환학생 학점인정 매칭
  • 복수·부전공·마이크로디그리 시뮬레이션
  • 동료 수강 이력 협업 필터링 (프라이버시 옵트인)

RAG 파이프라인 상세

인덱싱

  1. 수집: 서강대 공개 강의계획서 PDF · 학칙 PDF · 학과 공지사항
  2. 파싱: LlamaParse로 표·수식 보존, 과목 코드·시간·선수과목을 구조화 메타로 추출
  3. 청킹: 과목 단위(semantic chunk) + 500토큰 슬라이딩 윈도우
  4. 임베딩: Voyage-3 한국어 최적, pgvector 저장
  5. 메타 필터: 학과 / 학년 / 학기로 사전 필터 → 검색 정밀도 향상

검색 (쿼리 타임)

  1. 사용자 질의를 쿼리 재작성 LLM이 3가지 변형으로 확장
  2. Hybrid: BM25 + Dense 가중 결합
  3. Rerank: Cohere rerank-3으로 상위 20→5 정제
  4. LLM에 컨텍스트 전달 시 출처 인용 강제(구조화 출력 스키마)

할루시네이션 방어

  • Citation required: LLM 출력 JSON 스키마에 citations[] 필수 필드. 빈 배열이면 재생성
  • Ground truth 체크: 추천된 과목 코드가 실제 DB에 존재하는지 프로그램적 검증
  • 선수과목 확정성: 그래프 탐색은 결정론적. LLM이 선수과목을 "상상"할 여지 없음
  • 확신도 표시: UI에 "이 부분은 규정 조회, 이 부분은 AI 추천"을 시각적으로 분리
