01 — 쉽게 설명하면

수강신청을 "내비게이션" 처럼.

지금은 목적지(졸업)만 찍혀 있고, 학생이 직접 지도를 펼쳐서 길을 그립니다. 우리는 그 지도를 읽고 "이 길로 가면 돼요" 라고 말해주는 AI를 만듭니다.

🎯

목적지를 말하면

"AI+경영에 관심 있어요" 한 줄이면 충분합니다. 복잡한 학점 계산은 필요 없습니다.

🗺️

길을 그려줍니다

학기별로 어떤 과목을 들어야 하는지, 순서대로. 선수과목은 자동으로 앞에 배치됩니다.

🚧

막히는 구간은 우회

"금요일 오후 알바" 같은 제약을 입력하면 그 시간의 수업은 자동으로 뺍니다.

💬

이유까지 설명

"왜 이 과목을 먼저 듣는지" AI가 자연어로 풀이해줍니다. 학칙 조항까지 인용해서.

🧭

비유하자면 — 구글맵이 "경복궁까지 어떻게 가요?"를 물으면 교통·시간·도로상황을 종합해 최적 경로를 그리듯, 본 서비스는 "퀀트 펀드 취업까지 어떻게 가요?"에 학점·선수과목·시간표·강의평을 종합해 4년 수강 경로를 그립니다.

02 — 왜 지금 필요한가

서강대 학생의 매 학기 고통, 구체적으로.

강의계획서는 PDF, 졸업요건은 학과 홈페이지, 트랙 안내는 별도 문서. ChatGPT는 서강대 규정을 모르고, 학사시스템은 추천 기능이 없습니다.

BEFORE · 현재

탭 20개 띄워놓고 수기로 맞추기

  • 강의계획서 PDF를 여러 탭에 펼침
  • 졸업요건 엑셀과 대조하며 남은 학점 계산
  • 선수과목 체인을 손으로 역추적 (딥러닝 ← 기계학습 ← 자료구조 ← …)
  • 시간표 충돌은 수강신청 직전에야 발견
  • 선배 카더라 · 에타 글 파편 수집
  • 학기당 4~6시간 소요, 결과물은 스프레드시트 한 장
AFTER · 내비게이터 투입

관심사 한 줄 → 4년 로드맵 30초

  • 전공·학년·이수학점·트랙을 폼으로 입력
  • 자유 제약은 자연어로 ("금요일 오후 알바")
  • AI가 졸업요건 RAG로 검색 후 역산
  • 선수과목 그래프를 자동 탐색해 순서 배치
  • 시간표 충돌·학점 상한·개인 제약까지 검증
  • "왜 이 조합인가"를 학칙 출처 인용하며 설명
03 — 어떻게 작동하는가

4개의 전문 AI 에이전트가 릴레이합니다.

단일 챗봇이 아니라, 각자 다른 일을 잘하는 에이전트 4개가 순차적으로 문제를 넘깁니다. 오케스트레이터가 지휘하고, 각 에이전트는 RAG / 그래프 / 제약풀이 / 설명생성을 맡습니다.

01

RequirementRAG

졸업요건 검색자
학칙·시행세칙·학과별 졸업요건 PDF가 벡터DB에 들어있습니다. 학생 전공이 입력되면 관련 조항을 출처와 함께 꺼내옵니다.
pgvector · Voyage-3 · BM25+dense
02

PrereqGraph

선수과목 탐색자
모든 과목을 노드로, 선수관계를 간선으로 둔 DAG. 목표 과목에서 뒤로 DFS 탐색해 반드시 들어야 할 기초과목을 자동 수집.
NetworkX · Topological sort
03

TimetableSAT

제약 풀이자
시간표 충돌, 학기 최대학점, 자유 제약("금요일 오후 회피")을 제약충족문제로 변환해 풀이. 해가 없으면 완화.
Z3 SAT · Or-tools
04

Explainer

설명 생성자
왜 이 조합인가를 자연어로 풀이. 구조화 출력 스키마에 citations[] 필수 필드로 할루시네이션 차단.
Claude · JSON Schema

할루시네이션 방어 4중 장치

AI가 없는 과목을 상상하거나, 엉뚱한 규정을 지어내는 걸 원천 차단합니다.

🛡️ 출처 인용 강제

LLM 응답 JSON 스키마에 citations[] 필수. 빈 배열이면 자동 재생성.

✅ Ground truth 검증

추천 과목 코드가 실제 DB에 존재하는지 프로그램적으로 확인. 존재하지 않으면 제거.

🔗 결정론적 그래프

선수과목 탐색은 LLM이 아닌 NetworkX DFS. AI가 "상상한 선수과목"이 끼어들 여지 없음.

📊 확신도 분리

UI에서 "규정 조회"(초록)와 "AI 추천"(노랑)을 시각적으로 구분. 사용자 신뢰 확보.

05 — 심사 관점 자가평가

혁신성 ★★★★☆ 역산 플래닝 + RAG + 그래프 탐색의 결합은 국내 대학 서비스 중 최초

AI 활용 적절성 ★★★★☆ RAG로 할루시네이션 제어, LLM이 없으면 불가능한 자연어 설명 + 도메인 특화 필요성 명확
