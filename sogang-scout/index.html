서강대 학생은 매 학기 자기가 받을 수 있는 돈을 스스로 검색해야 한다. 교내 장학금·근로장학·단대별 특별장학·기업협력 장학·공모전 지원금이 수십 개 채널에 흩어져 있고, 자격 조건은 자연어 문장으로 묻혀 있다. 결과: 놓치는 사람이 너무 많다. AI가 자격 조건을 읽고 내 프로필과 매칭해주면 끝나는 문제를 학생이 혼자 하고 있다. 이걸 끝낸다.

1. 왜 이 문제인가

1-1. 문제의 실체

서강의 장학·지원 공고는 최소 다섯 개의 서로 다른 공간에 올라온다. 디지털정보처 공지, 각 단과대 사무실 홈페이지, 학과 게시판, 동문회·재단 공고, 기업협력팀의 산학 장학. 학생이 이 모두를 매일 확인할 수는 없다.

게다가 자격 조건은 하나같이 자연어 문장으로 쓰여 있다. "직전 학기 12학점 이상 이수 + 학과 상위 30% + 소득분위 8 이하 + 비수도권 출신 우대" 같은 조합. 자기 프로필을 각 공고에 일일이 대입해야 한다. 그래서 포기한다.

이게 왜 AI 아니면 못 푸는가: 자격 조건을 해석하고(→ 예: "학과 상위 30%"를 학생의 전공·학번·성적과 매칭), 애매한 표현을 추론하고(→ "관련 분야 관심이 있는 자"), 누락된 조건을 역추적(→ 조건 9개 중 7개 충족, 남은 2개가 뭔지 안내)해야 한다. 정규식·키워드 매칭으로는 안 된다.

2. 핵심 기능

자동 공고 수집

5개 이상 채널을 매일 크롤링·중복 제거. 새 공고는 자동 요약·태깅.

자격 자동 매칭

자연어 조건을 구조화된 제약으로 변환 후 내 프로필과 교차 매칭.

마감 D-day 알림

카톡·이메일로 마감 3일·1일 전 리마인드. 서류 체크리스트 포함.

놓친 돈 계산기

지난 학기 놓친 장학 금액을 합산해 보여줌. "이번 학기엔 놓치지 말자" 동기 부여.

지원서 초안 생성

공고 핵심 키워드 + 내 활동 이력을 엮어 자기소개서 초안을 뽑는다.

규정 인용 가드

매칭 결과마다 공고 원문 링크와 조건 근거를 표시. 환각 차단.

3. 시스템 아키텍처

파이프라인은 4-레이어. 수집 → 구조화 → 매칭 → 전달. LLM은 공고 파싱과 매칭 설명에만 쓰고, 자격 판별은 결정론적 룰 엔진으로 검증한다.

※ 설계 원칙: LLM은 공고 해석·설명에만, 자격 판별은 룰엔진. 모든 매칭 결과에 공고 원문 인용 필수 — 환각을 제품 레벨에서 봉쇄한다.

    new_posts = crawler.fetch(source)
    for post in new_posts:
        structured = llm.parse(post, schema=SCHOLARSHIP_SCHEMA)
        # {title, amount, deadline, eligibility: {grade, gpa, income, region, major, tags}}
        qdrant.upsert(structured, embed=bge_m3(post.text))

# 매칭 (사용자 요청 시)
def match(profile):
    candidates = qdrant.search(profile.vector, filter=profile.meta)
    for c in candidates:
        score, reasons = rule_engine.eval(c.eligibility, profile)
        if score >= THRESHOLD:
            explanation = llm.explain(c, profile, reasons)
            yield MatchCard(c, score, reasons, explanation)

7. 보안·윤리

  • 소득분위·가족관계 등 민감 정보는 기본 로컬 저장. 서버 매칭 원하면 동의 후 암호화 전송.
  • 사용자가 지원서 초안 생성 시 LLM 호출은 마인드로직 허브 경유로 외부 모델에 평문 노출 차단.
  • "자격 있음"이 아닌 "자격 요건 충족 가능성 높음"으로만 표현. 최종 판정은 장학팀에 있음을 모든 카드에 명시.
  • 광고·제휴 장학 등 상업적 공고는 별도 태그로 구분.
